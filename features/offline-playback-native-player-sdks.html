    Reproducción sin conexión con los SDK del reproductor nativo

    En este tema, aprenderá cómo funciona la reproducción sin conexión con los SDK de Brightcove Native Player.

    Resumen

    La reproducción sin conexión con los SDK de Brightcove Native Player para iOS y Android permite a los editores llegar a sus espectadores de formas completamente nuevas, al tiempo que garantizan que su contenido permanezca seguro. Nuestros SDK ofrecen soluciones poderosamente sencillas para problemas complejos de reproducción sin conexión, incluyendo gestión de descargas, gestión de catálogos sin conexión, análisis, DRM y, por supuesto, reproducción.

    Al utilizar la reproducción sin conexión, los usuarios pueden descargar contenido de vídeo protegido por DRM y claro (no cifrado) en sus dispositivos y verlo cuando no están conectados.

    Requisitos

    Los SDK de Brightcove Native Player admiten la reproducción sin conexión con las siguientes versiones:

    Versión nativa del SDK de Brightcove

    • SDK nativo para iOS 6.0.1+
    • SDK nativo para Android 5.0.3+

    Versión del sistema operativo de dispositivo

    • iOS 10.0+, pero se recomienda 10.3+
    • Android 5.0+

    Configuración

    Para comenzar, haga lo siguiente:

    • Póngase en contacto con el administrador de su cuenta para habilitar su cuenta para la reproducción sin conexión.
    • Asegúrate de usar Entrega dinámica al ingerir tus vídeos.
    • Prepare cada vídeo que desee habilitar para su descarga.

    Envío de contenido

    La transmisión de vídeo aprovecha la función de transmisión de múltiples velocidades de bits de Video Cloud, que detecta el ancho de banda en el origen y ofrece la mejor calidad para el dispositivo. La función de reproducción sin conexión admite los siguientes tipos de transmisión:

    • Android: Transmisión adaptativa dinámica a través de HTTP (DASH)
    • iOS: Apple HTTP Live Streaming (HLS)

    Seguridad

    La protección del contenido descargado es lo más importante con nuestra solución Offline. Los SDK de Brightcove Native Player ofrecen la siguiente funcionalidad para garantizar que el contenido se almacene de forma segura en los dispositivos de los espectadores:

    Contenido protegido por DRM

    • Para el contenido protegido por DRM, los SDK nativos utilizan las siguientes tecnologías:
      • Android: DASH con Widevine Modular

      • iOS: HLS con FairPlay

    • El contenido de vídeo solo se puede ver en el dispositivo y la aplicación que ha descargado el contenido.

    • El contenido de vídeo solo se puede ver hasta que caduque una fecha y hora específicas.

      Los SDK nativos permiten a los usuarios alquilar o comprar vídeos. La caducidad se aplica a alquileres y se establece en la licencia DRM utilizando la API de SDK nativos para descargar vídeos. Por el contrario, la caducidad establecida en el token Fastly se aplica al contenido de streaming.

    • Las URL de vídeo devueltas al cliente caducarán para que otros no puedan usarlas.

    Todo el contenido

    • Cuando se elimina la aplicación, se elimina todo el contenido descargado.
      Para Android, esto es cierto siempre y cuando no se reemplace la ubicación de almacenamiento predeterminada.
    • Tu aplicación puede eliminar contenido almacenado localmente en cualquier momento.

    Gestión de descargas

    Los SDK de Brightcove Native Player manejan las complejidades de descargar contenido en el dispositivo de un espectador, lo que simplifica enormemente el proceso de las aplicaciones nativas.

    • Los editores pueden elegir la calidad de copia del contenido que desea descargar.
    • Los subtítulos se pueden descargar y mostrar sin conexión.
    • Los SDK nativos admiten varias pistas de audio. Para obtener más información sobre cómo descargar pistas adicionales, consulte lo siguiente:
      • Varias pistas de audio para reproducción sin conexión con el SDK nativo para Android
      • Guía del desarrollador de aplicaciones iOS para la reproducción sin conexión.

    Funcionalidad

    Las siguientes funciones se pueden agregar a las aplicaciones para admitir la administración de descargas:

    • Mostrar contenido
    • Comprobar el tamaño de descarga
    • Mostrar estado de descarga
    • Administrar descargas
    • Mostrar espacio de almacenamiento local
    • Pausar/Reanudar/Cancelar descarga

    Mostrar contenido

    Muestra una lista de contenido disponible para streaming o descarga.

    Cuando establece la offline_enabled propiedad de un vídeo en true , los SDK nativos marcarán este vídeo como descargable.

    Preparar vídeos

    Para marcar un video como descargable, debe hacer lo siguiente:

    1. Si aún no lo has hecho, ponte en contacto con tu administrador de cuentas para habilitar la reproducción sin conexión de tu cuenta.

    2. Para cada vídeo que desee descargar, configure la opción sin conexión activada mediante una de las siguientes acciones:

      Usar Video Cloud Studio

      Utilice Video Cloud Studio para habilitar la reproducción sin conexión.

      Usar la API de CMS

      También puede utilizar la API de CMS para establecer el campo offline_enabled en un valor verdadero.

      Aquí hay un ejemplo usando curl:

        curl --header "Authorization: Bearer $oauth_token" --request PATCH --data '
  {"offline_enabled" : true}
  ' https://cms.api.brightcove.com/v1/accounts/your account id/videos/your video id

    Cuando trabaje con contenido claro, consulte la sección Prácticas recomendadas para obtener sugerencias sobre cómo mostrar el contenido.

    Comprobar el tamaño de descarga

    Brightcove recomienda que antes de solicitar una descarga, asegúrese de que hay suficiente espacio en el almacenamiento del dispositivo para la descarga. Para obtener una estimación del tamaño total de la descarga de vídeo, siga estos pasos:

    • Para iOS, revisa la información en la sección Comprobar el tamaño de la descarga de la Guía del desarrollador de aplicaciones iOS para la reproducción sin conexión.

    • Para Android, puede obtener una estimación en la devolución de llamada OnDownloadStarted desde DownloadEventListener. Aquí hay un ejemplo de cómo configurar el detector de eventos:

        MediaDownloadable.DownloadEventListener downloadEventListener = new MediaDownloadable.DownloadEventListener() {
  //Your implementation.
  }
  OfflineCatalog catalog = new OfflineCatalog(context, eventEmitter, accountId, policyKey);
  catalog.addDownloadEventListener(downloadEventListener);

     

    Mostrar espacio de almacenamiento local

    Informar al usuario cuando no hay suficiente almacenamiento local para completar la descarga del elemento actual y los elementos de la cola.

    Los SDK proporcionan el tamaño estimado y el progreso para que pueda determinar si hay suficiente espacio para las descargas.

    Mostrar estado de descarga

    Muestra la descarga actual junto con su estado.

    Los SDK devolverán el tamaño total estimado, el estado actual de la descarga y el porcentaje de información sobre el progreso de la descarga.

    Para dispositivos Android, el estado de descarga también se muestra en el área de notificación. El título de la notificación se establece en el título del vídeo. La notificación se elimina si la descarga está en pausa o se cancela. Para obtener más información, consulte los fragmentos de código utilizando el SDK nativo para Android documento.

    Administrar descargas

    Descarga de varios vídeos

    Los SDK nativos permitirán poner en cola varios vídeos para descargas. Por ejemplo, un usuario puede descargar varios episodios de una temporada y los SDK nativos administrarán el proceso proporcionando el estado de descarga para cada activo.

    Si tu aplicación está orientada a iOS 11.0 - 11.2, no recomendamos descargas simultáneas porque el comportamiento de pausa y reanudación no es confiable con varias descargas simultáneas. El problema se ha solucionado con iOS 11.3.

    Descarga de pistas secundarias (iOS)

    Con iOS 13, Apple ha cambiado el funcionamiento de las descargas. Para obtener más información, consulta la guía Guía del desarrollador de aplicaciones para iOS para descargar vídeo y reproducción sin conexión .

    Debido a este cambio, la aplicación de ejemplo Descarga sin conexión de iOS puede aparecer para volver a descargar vídeos con varias pistas de audio. Esto se debe a que el progreso de la descarga se da para cada pista. Por lo tanto, la barra de progreso comenzará de nuevo desde cero para cada pista de audio adicional.

    Aquí hay una breve descripción general de AVMediaSelectionOptions, subtítulos y pistas de audio alternativas: Añadir subtítulos y pistas de audio alternativas

    Determinar la descarga total

    ¿Hay alguna manera de didProgressTo obtener la descarga total de un token de video?

    No. Para un token de vídeo sin conexión, no hay un valor agregado de progreso de descarga. El desarrollador de la aplicación determina cuántas descargas de activos se producirán. La primera descarga de activos es el vídeo principal y las pistas de audio y texto preferidas. Por lo general, es la descarga más grande y el progreso avanzará de 0.00 a 100,00.

    Si una solicitud de descarga incluye selecciones de medios adicionales (que el desarrollador de la aplicación solicita explícitamente), el tiempo total de descarga es simplemente el del vídeo principal y la selección de medios preferidos, más el de cada selección de medios adicionales. Al representar el progreso total en la interfaz de usuario, elija un valor de escala para cada descarga de modo que la suma de todas las descargas totalizará el 100% y sumará los tiempos de descarga escalados de cada selección de medios.

    Identificar tipos de objeto

    ¿Hay alguna manera de identificar a qué tipo de objeto (vídeo o pista de texto) está relacionado con cada didProgressTo evento?

    Sí. Las descargas adicionales son de objetos Opción de selección de medios de Apple (AVMediaSelectionOption), no pistas. El objeto de selección de medios que se está descargando se identifica en la devolución de llamada de progreso. (BCOVOfflineVideOmanagerDelegate.h)

    Apple proporciona métodos para examinar los atributos de una opción de selección de medios. Para obtener más información, consulte AVMediaSelectionOption deApple.

    Puede ver los valores de metadatos en la consola del depurador de Xcode cuando ejecuta la aplicación de ejemplo OfflinePlayer (no se deje engañar por las dos opciones en inglés, es solo una aplicación de muestra) 

    OfflinePlayer[523:355259] AVMediaSelection option 0 | legible display name: English
OfflinePlayer[523:355259] AVMediaSelection option 0 | audible display name: English
OfflinePlayer[523:355259] AVMediaSelection option 1 | legible display name: English
OfflinePlayer[523:355259] AVMediaSelection option 1 | audible display name: English

    Puede ver cómo se hace esto en el código DownloadManager.

    Pausar/Reanudar/Cancelar descarga

    Permitir al usuario pausar, reanudar o cancelar la descarga de contenido.

    La funcionalidad de descarga se reanudará donde lo dejó si el usuario pausa o reanuda o si la descarga se interrumpe. Por ejemplo, si pierde la conexión Wi-Fi durante la descarga, se reanudará donde lo dejó cuando se vuelva a conectar.

    Si cancela una descarga de vídeo, cualquier progreso realizado se borrará.

    Gestión de catálogos locales

    Los SDK de Brightcove Native Player simplifican el proceso de administración del contenido local descargado.

    Se puede agregar la siguiente funcionalidad a las aplicaciones para admitir la administración de catálogos locales:

    • Mostrar contenido descargado
    • Reproducir contenido local
    • Eliminar contenido local

    Mostrar contenido descargado

    Muestra todos los episodios descargados para cada show o temporada.

    Los SDK devuelven metadatos y detalles de reproducción para todo el contenido descargado. Estos metadatos incluyen el título del vídeo, descripción, miniaturas, subtítulos disponibles, pistas de audio disponibles, etc.

    Existen nuevas propiedades de metadatos específicas para la reproducción sin conexión, entre las que se incluyen las siguientes:

    Contenido protegido por DRM

    • Identificador de token
    • Fecha de vencimiento de licencia

    Todo el contenido

    • Descargar hora de inicio y finalización
    • Nombre de miniatura descargado junto con su URL de archivo
    • Nombre del póster descargado junto con la URL del archivo
    • URL de archivo relativa y derivada del paquete de vídeo

    A continuación, puede filtrar los resultados para su implementación específica.

    Podrá ver el tiempo restante de una licencia de alquiler.

    Eliminar contenido local

    Permitir a los usuarios eliminar contenido del almacenamiento local.

    Los SDK exponen un método para eliminar un vídeo junto con sus metadatos y activos relacionados.

    Reproducción de contenido

    Los SDK nativos le permiten reproducir contenido en las siguientes situaciones:

    • Reproduce un vídeo mientras se descarga.
    • Reproduzca un vídeo después de que se haya descargado. El contenido descargado se puede reproducir localmente cuando el dispositivo está fuera de línea o sin conexión. El vídeo sin conexión solo se puede reproducir desde el almacenamiento local.
    • Si decide transmitir una versión en línea del vídeo, debe recuperar un nuevo objeto de vídeo del Servicio de reproducción.

    Analytics

    Se han introducido análisis para medir datos relacionados con el flujo fuera de línea. Para obtener más información, consulte el documento Analytics Related to Offline Playback with the Native SDK .

    La tabla siguiente contiene vínculos a ejemplos de código para mostrarle cómo empezar a reproducir sin conexión:

    Reproducción sin conexión de Android Descarga y reproduce vídeos de DASH, incluidos aquellos protegidos con cifrado Widevine.

    Notas

    Las siguientes notas se aplican a la función de reproducción sin conexión admitida por los SDK nativos:

    Todos

    Publicidad

    Ni la publicidad del lado del cliente ni del lado del servidor es compatible con la reproducción sin conexión.

    HLSe

    Actualmente, el contenido HLSE no es compatible con la reproducción sin conexión mediante el SDK nativo de Brightcove para Android, iOS o tvOS.

    Modo de baja potencia

    El modo de bajo consumo de energía en los dispositivos puede afectar a ciertas funciones, incluida la descarga. Para obtener más información, consulte lo siguiente:

    • Gestión de energía Android 9 (API nivel 28)
    • Modo de bajo consumo en el iPhone
    • Reaccionar al modo de bajo consumo en iPhones

    Vídeos compartidos

    Los vídeos compartidos solo se pueden descargar para su visualización sin conexión si el vídeo original puede serlo. La propiedad se hereda del vídeo original y no se puede cambiar sólo para el vídeo compartido.

    Solo audio

    Los SDK nativos admiten contenido de solo audio con reproducción sin conexión. Usted controla lo que ven los espectadores, ya sea una imagen de póster o una pantalla personalizada.

    El SDK nativo para Android requiere un VideoView, y el SDK nativo para iOS requiere un UIView. Debido a esto, el audio de fondo no es compatible. Esto significa que el audio no se seguirá reproduciendo cuando un usuario envía la aplicación al fondo.

    Android

    Eliminación de aplicaciones

    Cuando se elimina la aplicación, se elimina todo el contenido descargado. Esto es cierto siempre que no se reemplace la ubicación de almacenamiento predeterminada.

    iOS

    AirPlay

    Como función de seguridad, no puede transmitir un vídeo HLS sin conexión a un dispositivo AirPlay para su reproducción. Apple confirma que esto es una AVFoundation limitación.

    Preguntas frecuentes (FAQ)

    A continuación encontrará respuestas a algunas preguntas generales.

    Contenido protegido por DRM

    ¿Puede el usuario cambiar el reloj del sistema para omitir la fecha de caducidad?

    Android: La caducidad absoluta determina la fecha y la hora hasta la que una licencia es válida. Esto se aplica mediante Widevine y la capa DRM de Android. Teóricamente, la capa DRM en Android debe evitar que el usuario evite el período de caducidad. Cualquier problema que podamos haber encontrado se describe en las notas de la versión del sistema operativo Android.

    iOS: La caducidad de la licencia es forzada por FairPlay. iOS maneja todas las comprobaciones de cambios de reloj, etc. Una aplicación tiene la opción de comprobar la caducidad de la licencia para alertar al usuario de que la licencia ha caducado. Esto puede ser preferible a simplemente reproducir un vídeo caducado y luego recibir un error de reproducción.


    ¿La licencia y los vídeos sin conexión seguirán cargándose mientras la aplicación está en segundo plano?

    Android: Sí. Normalmente, la licencia se descarga inmediatamente, pero incluso la licencia se puede descargar mientras la aplicación está en segundo plano.

    iOS: La carga de licencias debe realizarse en primer plano. Tenemos un método para precargar la licencia. La descarga del contenido principal del vídeo puede ocurrir mientras la aplicación está en segundo plano.

    Todo el contenido

    ¿Cuál es el formato de vídeo utilizado para descargar contenido sin conexión?

    Android: DASH protegido por DRM o DASH no cifrado.

    iOS: HLS vapor. Puede especificar una velocidad de bits para que la descarga utilice una variante específica.


    ¿Los vídeos descargados se reproducirán localmente o en streaming cuando el usuario esté en línea?

    Un vídeo descargado se puede reproducir en línea o sin conexión, pero siempre utilizará el vídeo descargado. Si decide reproducir una versión en línea del vídeo, debe recuperar un nuevo objeto de vídeo del Servicio de reproducción. El vídeo sin conexión solo se puede reproducir desde el almacenamiento local.


    Mejores prácticas

    Las siguientes directrices te ayudarán a desarrollar tu aplicación para la reproducción sin conexión.

    Visualización de botones de pausa/cancelación

    Los botones de pausa y cancelación deben mostrarse cuando se haya iniciado la descarga de vídeo.

    Android: Debe mostrar los botones de pausa y cancelación como parte de la llamada de onDownloadStarted devolución de llamada en DownloadEventListener. Para obtener más información, consulta la aplicación de ejemplo Reproducción sin conexión.

    iOS: Para obtener más información sobre cómo crear una aplicación para la reproducción sin conexión, consulta la Guía del desarrollador de aplicaciones de iOS.

    Visualización de contenido claro

    Si su cuenta no está habilitada para DRM o está trabajando con contenido claro, los SDK nativos de Brightcove no ejecutarán una solicitud de licencia del lado del servidor. Por lo tanto, el contenido claro no se considera disponible para alquiler o compra. En la mayoría de los casos, para un contenido claro, se recomienda ocultar los botones de alquiler y compra y mostrar un solo botón de descarga en su lugar.

    Habilitación de la función

    Póngase en contacto con el administrador de su cuenta para obtener información sobre los precios y activar la función Reproducción sin conexión para su cuenta.

    Identificación de problemas

    Los siguientes consejos pueden ayudarle a investigar problemas al descargar vídeos.

    Puede producirse un error cuando un usuario intenta descargar un activo a una velocidad de bits superior a la que puede manejar el dispositivo debido a una red lenta.

    Para evitar esto, puede agregar código para comprobar las capacidades del dispositivo e impedir que los dispositivos con conexiones lentas intenten descargar activos de alta velocidad de bits. Puede probar con un dispositivo real mientras se bloquea la conexión y ver el tráfico de red con la herramienta proxy de Charles .

    Para obtener más información sobre el manejo de errores, consulte el documento Gestión de errores con el SDK nativo para iOS .

    Cuando el código de reproducción sin conexión recibe un error, lo detectamos y lanzamos un NSERror. Para obtener el error subyacente, puede probar esto:

      NSError *underlyingError = initialError.userInfo[NSUnderlyingErrorKey];

    Para obtener más información, consulta la Guía para desarrolladores de aplicaciones iOS para la reproducción sin conexión.
