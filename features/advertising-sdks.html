Descripción general

Los SDK nativos de Brightcove ofrecen un conjunto de opciones para ayudarlo a monetizar su contenido. Los SDK nativos y sus integraciones permiten a los editores decidir cómo optimizar mejor la publicidad en su reproductor con funciones como anuncios previos, intermedios y posteriores al video, así como anuncios complementarios.

Notas:

  • Los SDK nativos de Brightcove NO admiten anuncios VPAID.
  • Para Android MediaPlayer , existen limitaciones conocidas con el uso de anuncios mid-roll mientras se reproducen videos HLS. El uso del SDK de reproductor nativo de Brightcove resuelve muchos de estos problemas. Aprender más sobre Reproducción de HLS con el SDK nativo para Android.

Puntos de referencia

Las URL publicitarias se pueden definir con puntos de referencia de anuncios o dentro de una URL de etiqueta publicitaria VMAP:

  • Con los puntos de referencia de anuncios, cada uno debe configurarse con una URL de etiqueta de anuncio VAST.
  • Sin puntos de referencia de anuncios, puede utilizar una URL de etiqueta de anuncio VMAP para definir cuándo se reproducirán los anuncios.
La URL de VMAP es una lista de reproducción de anuncios, donde una URL de VAST define un anuncio individual. Para obtener información general, consulte el Guía de aprendizaje: Publicidad en video documento.

Complementos del lado del cliente

La publicidad del lado del cliente es el enfoque tradicional de realizar solicitudes de anuncios e inserción de anuncios en el dispositivo del cliente. Con este método, el video principal se reproduce hasta que se alcanza un punto de referencia de anuncio específico. En este punto, se realiza una solicitud al servidor de anuncios configurado, que responde con los detalles del anuncio adecuados. A continuación, el reproductor comienza a reproducir el anuncio solicitado.

Las integraciones de Brightcove simplifican este proceso al manejar el trabajo de realizar las solicitudes de anuncios, interpretar los detalles publicitarios y mostrar los anuncios.

Anuncios de medios interactivos de Google (IMA)

Brightcove fue la primera plataforma de video en línea en asociarse con Google DoubleClick y desarrollar apoyo para SDK de IMA de Google en plataformas web (Flash / HTML5) y móviles (iOS / Android). Nuestra estrecha relación con Google nos permite estar al día con las próximas funciones y correcciones de errores.

Los editores pueden utilizar el SDK de IMA de Brightcove para solicitar y mostrar anuncios publicados por DoubleClick for Publishers (DFP), la red de Google AdSense. También pueden usarlo para analizar su propia lista de reproducción VMAP siempre que se adhiera al estándar VMAP según lo interpretado por Brightcove IMA SDK.

Al utilizar el SDK de IMA de Google para iOS y Android, ofrecemos una integración fácil de usar para ambas plataformas, lo que garantiza que se mantendrá actualizado con las actualizaciones de Google.

Inserción de anuncios en el servidor (SSAI)

La unión de anuncios del lado del servidor es un enfoque innovador para entregar publicidad a sus espectadores. Al proporcionar un flujo continuo único, los anuncios están incrustados en el contenido. Esto elimina las solicitudes de anuncios y reduce los tiempos de carga. También evita que los espectadores bloqueen anuncios. Los SDK de Brightcove Native Player proporcionan la lógica y la interfaz de usuario publicitarias adecuadas para garantizar que la experiencia de reproducción al cambiar entre anuncios y contenido sea más fluida que nunca.
