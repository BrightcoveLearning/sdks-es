    Controles de reproductores nativos de Brightcove

    En este tema, aprenderá a usar y personalizar los controles del reproductor con los SDK nativos de Brightcove.

    Introducción

    Combinando la estética del reproductor web Brightcove Web Player con los estándares de las plataformas iOS y Android, los controles de reproductores nativos de Brightcove proporcionan a la vez elegancia y un potente conjunto de funciones sin necesidad de programación adicional.

    Funciones disponibles con los controles de reproductores nativos de Brightcove:

    • interfaz de usuario dinámica
    • Personalización
    • Accesibilidad

    Interfaz de usuario dinámica (Dynamic UI)

    Los controles de reproductores nativos de Brightcove son altamente adaptables, por lo que permiten visualizar sus contenidos en una atractiva interfaz de usuario independientemente de las condiciones de reproducción.

    Tamaño de pantalla adaptativo

    Para cualquier aplicación móvil resulta esencial garantizar que el tamaño de los elementos de la interfaz de usuario sea suficiente como para verlos claramente y utilizarlos cómodamente. Los controles del reproductor nativo de Brightcove contienen una lógica para determinar cuánto espacio hay disponible para la interfaz de usuario. Si el ancho cae por debajo de un determinado umbral, los controles cambiarán automáticamente para usar una configuración de dos líneas.

    Android

    El SDK nativo para Android comprueba el tamaño de la pantalla del dispositivo para establecer el diseño de los controles del reproductor.

    • Cuando el ancho es 480dp o mayor, se utiliza el diseño de una línea.
    • Cuando el ancho es menor que 480dp, se utiliza el diseño compacto de dos líneas.

    Para obtener más información sobre los píxeles independientes de densidad (dp), consulte el documento Admite diferentes densidades de píxeles de Android.

    iOS

    El SDK nativo para iOS comprueba el tamaño de la pantalla del dispositivo para establecer el diseño de los controles del reproductor.

    • Cuando el ancho es 450px o mayor, se utiliza el diseño de una línea.
    • Cuando el ancho es menor que 450px, se utiliza el diseño compacto de dos líneas.

    Modos de reproducción

    A lo largo de su ciclo de vida, el reproductor puede contener diferentes tipos de contenidos, cada uno con su propia interfaz de usuario. Los controles de reproductores nativos de Brightcove contienen la lógica necesaria para determinar automáticamente si el tipo de contenido es VoD, en directo o Live DVR y adaptar la interfaz de usuario en consecuencia, incluso aunque la lista de reproducción contenga diferentes tipos de contenidos.

    Vídeo a la carta (VoD)

    La interfaz de vídeo a la carta "VoD" se utiliza en la mayoría de experiencias de reproducción, y es la raíz de todos los diseños de interfaz de usuario de reproductores.

    Algunas de las funciones más notables de los controles de reproductores de Brightcove:

    • Botón Reproducción/Pausa
      • Permite reproducir el contenido y detener la reproducción
    • Botón Saltar atrás
      • Retrocede 10 segundos en el contenido
    • Barra de posicionamiento avanzada
      • Muestra el contenido reproducido en color rosa
      • Muestra el contenido almacenado en el búfer en color gris claro
      • Permite utilizar marcadores para anuncios u otros fines (Android)
    • Botón Subtítulos
      • Solo aparece cuando se detectan subtítulos. Al pulsar este botón, aparece el menú de ajustes de subtítulos.
    • Botón de pantalla completa
      • Permite al espectador activar la reproducción a pantalla completa o volver a la reproducción integrada.
    • Botón Airplay (solo iOS)
      • Solo aparece cuando se detecta algún dispositivo Airplay. Esta función permite al espectador configurar la retransmisión a través de Airplay.

    En directo

    La interfaz de usuario de Live oculta determinados elementos que un usuario no debe utilizar. Los elementos eliminados son el botón de salto atrás y la barra de posicionamiento. Sin embargo, se añade una nueva etiqueta «LIVE», que cuando está verde indica que el contenido que se está viendo está activo.

    Live DVR

    El formato Live DVR ofrece la misma funcionalidad que el diseño VoD, además de la etiqueta adicional LIVE, que adopta el color verde cuando el contenido es en directo, y el color gris cuando el espectador visualiza contenidos más antiguos. La etiqueta LIVE también actúa como botón que se puede pulsar en cualquier momento para volver a posicionarse en el momento en directo más reciente.

    Personalización

    ¿Desea cambiar los colores o la piel para que coincida con los temas de colores de su aplicación? ¡No hay problema! Los controles de reproductores nativos de Brightcove han sido diseñados para ser fácilmente personalizables. A continuación detallamos algunas de las numerosas posibilidades de personalización de los controles:

    • Intercambia colores y usa tus propios botones personalizados para crear temas personalizados. El SDK nativo para Android le permite incluir/excluir fácilmente una amplia variedad de botones comunes. Aquí, los botones de control se han cambiado a naranja, y la barra de búsqueda tiene una imagen personalizada.
    • Aunque el SDK nativo para iOS también le permite elegir entre un conjunto de botones y diseños comunes, también puede crear su propio diseño personalizado. Aquí, hay un ejemplo de un diseño personalizado.
    • Alternativamente, puede eliminar botones para crear un diseño simple y limpio.

    Accesibilidad

    Las funciones de accesibilidad vienen integradas en nuestra interfaz de usuario, sin necesidad de configuración adicional.

    Personas con discapacidad auditiva

    Los controles de reproductores nativos de Brightcove incorporan un botón de subtítulos en todos los diseños. Este botón garantiza que, siempre que haya subtítulos disponibles, estos se puedan personalizar fácilmente. Para obtener más información sobre los subtítulos opcionales con los SDK, consulte nuestra sección Formatos y subtítulos de vídeo .
