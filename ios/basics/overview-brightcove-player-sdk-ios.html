    Resumen: SDK de reproductores de Brightcove para iOS

    Bienvenido a la documentación del desarrollador de Player SDK para iOS. Aquí encontrarás la información para crear aplicaciones nativas para reproducir contenido de vídeo en dispositivos iOS.

    Estructura de documentación

    En Brightcove, nos esforzamos por mantener la documentación estructurada, limpia e intuitiva. Como tal, aprovechamos los ReadMes de github para obtener detalles sobre todos nuestros complementos e integraciones. Para su comodidad, hemos enumerado y vinculado a estos complementos a continuación.

    Versiones admitidas

    El SDK de Native Player para iOS ofrece compatibilidad con las siguientes versiones de iOS:

    Soporte activo

    Brightcove proporciona compatibilidad activa con el SDK de iOS más reciente en las siguientes versiones de iOS:

    • iOS 14
    • iOS 13
    • iOS 12.4.1

    Soporte pasivo

    Brightcove proporciona soporte pasivo para las siguientes versiones de iOS:

    • iOS 11.4.1

    Entender los niveles de soporte

    El soporte se define en términos de la [versión principal].[ versión menor].[ versión del parche] de un sistema operativo (por ejemplo: iOS 11.4.1). Para aprovechar las últimas mejoras tecnológicas de Apple, ofrecemos dos niveles de soporte:

    • Activo: Brightcove probará y corregirá errores en estas versiones del sistema operativo.
    • Pasivo: Brightcove comprobará errores en esas versiones de sistema operativo, pero solo los corregirá caso por caso.

    Por regla general, Brightcove ofrece servicio técnico activo para las versiones del sistema operativo utilizadas por un 80 % de los usuarios, y servicio pasivo para otras versiones que sean utilizadas por al menos un 5 % de la base de usuarios.

    Introducción

    Antes de comenzar el desarrollo con el Player SDK para iOS, hay algunos conceptos sobre los que debe aprender:

    1. Medios

      La versión actual del SDK de Native Player admite la reproducción de contenido de su cuenta de Video Cloud a través de la API de reproducción o como activos remotos con URL accesibles a Internet.

    2. Analytics

      Con el SDK de Native Player, los clientes de Video Cloud obtienen automáticamente datos analíticos sólidos reportados en Video Cloud.

    3. SDK principal y dependencias

      El SDK de Brightcove Player para iOS y sus complementos se pueden agregar a su proyecto utilizando el administrador de dependencias de CocoaPods. Los podspecs de todos los componentes del SDK de Brightcove Player están alojados en el repositorio de BrightCovespecs.

      Se requiere la versión 1.0 o posterior de CocoaPods.
    4. Referencia de SDK nativa

      Para ayudarle a comprender el SDK, puede encontrar un desglose detallado de la arquitectura en Player SDK para iOS. Descubra cómo interactúan los componentes críticos para manejar las complejidades de controlar el reproductor de vídeo nativo. Instalación, inicio rápido y arquitectura de Player SDK para iOS

      Aquí, también encontrará la versión beta más reciente del SDK, si existe.

    5. Descripción general del SDK nativo

      Para obtener información general rápida, obtenga información sobre cómo interactúan los componentes del SDK de Brightcove Player con los componentes nativos de iOS. Descripción general de Player SDK para iOS

    6. Preguntas y Respuestas

      Únase a nosotros en el foro de desarrolladores de SDK, donde los ingenieros de Brightcove y la comunidad responden a las preguntas relacionadas con SDK. Busque preguntas previamente respondidas o publique una nueva. Como grupo público, puede obtener acceso proporcionando su dirección de correo electrónico.

    7. Formación en vídeo

      Vea una serie de vídeos de formación que le guiará a través de los pasos de crear una aplicación Swift en Xcode que reproduzca un vídeo de su biblioteca de Video Cloud.

      • Introducción a los SDK de Brightcove Native Player

    Consejos:

    • Las vistas y controles de PlayerUI (incluidos los controles publicitarios) utilizan la guía iOS Safe Layout para evitar la colocación incorrecta en esquinas, pasaje de muesca o debajo del indicador de inicio.
    • El componente Subtítulos de Sidecar admite la reproducción a través de AirPlay con subtítulos. Para obtener más información sobre cómo mantener sincronizados los subtítulos, consulte el documento Sincronizar subtítulos WebVTT .

    Notas:

    • Para mejorar el cumplimiento de CocoaPods, el SDK nativo para iOS 6.7.7 ha cambiado el nombre de algunos archivos Podspec. Para obtener más información, consulte la Referencia del SDK nativo.
    • El objetivo de implementación mínimo para todos los entornos nativos de Brightcove para iOS es ahora iOS 11.0 y tvOS 11.0
    • El SDK nativo de Brightcove para iOS 6.5.0+ requiere Xcode 11.0 o superior.
    • El recuento automático de referencias (ARC) es necesario en el proyecto. Este es el valor predeterminado actual para los proyectos Xcode.
    • Con el lanzamiento de iOS 9 y Xcode 7, se introdujo una nueva característica llamada Seguridad de transporte de aplicaciones (ATS). Más información sobre cómo trabajar con seguridad de transporte de aplicaciones.
    • Un problema conocido de Apple con el contenido de streaming de FairPlay es que no se renderizará en el modo Mirroring AirPlay. Para obtener más detalles, consulte la presentación WWDC: Protección de contenido para HLS (desplácese aproximadamente a la mitad de la transcripción).

    Integraciones

    La siguiente tabla contiene enlaces para obtener complementos integrados con su implementación de SDK. Los complementos se pueden descargar desde sus respectivos repositorios de Github, y cada repositorio contiene un LÉAME detallado con instrucciones sobre cómo configurar el plugin.

    Las aplicaciones de ejemplo demuestran la funcionalidad básica de cada plugin integrado con el SDK. Estas aplicaciones de ejemplo son una buena manera de aprender acerca de cada plugin, y sirven como punto de referencia al depurar.

    La versión actual del SDK nativo para iOS incluye todos los componentes con el mismo número de versión. Al integrar componentes, el SDK y los números de versión del componente deben coincidir. Por ejemplo, al utilizar el componente de complemento IMA 6.7.0, debe utilizar el SDK nativo 6.7.0.

    Las siguientes notas se refieren a las integraciones de la tabla anterior.

    [ 1] La versión actual del complemento IMA es compatible con el SDK v3 de IMA iOS de Google, que admite las versiones 9.0+ de iOS. Para obtener más información, consulta la documentación de Google.

    Los SDK nativos de Brightcove admiten plantillas XML de VMAP que contienen el protocolo VAST, incluido VAST 3.0, y compatibilidad limitada con VAST 4.0. Para obtener más información sobre la compatibilidad de versiones VAST, consulte el documento de compatibilidad y soporte de Google para el SDK de IMA para iOS. Los SDK nativos también admiten el SDK de medición abierta de IAB.

    [ 2] Si está utilizando Swift, debe agregar manualmente las bibliotecas IMA de Google, en lugar de usar CocoaPods. Esto se debe a que la biblioteca IMA de Google no admite marcos dinámicos, y Swift no admite la creación de bibliotecas estáticas. Para obtener más información, consulte la sección Manual de la documentación del complemento IMA para Brightcove Player SDK para iOS . Aunque el complemento IMA de Brightcove utiliza un marco dinámico, la biblioteca IMA de Google solo se proporciona como un marco estático.

    Combinación de complementos

    Con el SDK nativo para iOS, los BCOVSessionProviders de cada plugin se crean y encadenan juntos. Esta cadena se utiliza entonces para crear el archivo BCOVPlaybackController . Para obtener más información, consulte la sección Combinación de complementos de la Referencia del SDK.

    Publicidad

    Aquí hay información adicional que puede ser útil cuando trabaje con los plugins de Publicidad.

    Complemento IMA

    La ubicación de los anuncios VAST suele especificarse mediante un documento VMAP. VAST describe un anuncio mientras que VMAP describe la ubicación de los anuncios en un vídeo. Para obtener más información sobre el código, consulte el ejemplo de BasicimaPlayer iOS .

    Cuando se trabaja con anuncios de IMA, es recomendable comprobar que los anuncios funcionan fuera del SDK antes de implementarlos. Los siguientes enlaces serán útiles para esto:

    • Validador de respuesta de anuncios de Google VAST
    • Guía de lista de reproducción de anuncios múltiples de vídeo digital (VMAP)
    • Foro técnico de Google IMA
