Estructura de la documentación

En Brightcove, nos esforzamos por mantener la documentación estructurada, limpia e intuitiva. Como tal, aprovechamos los archivos READMEs de github para obtener detalles sobre todos nuestros complementos e integraciones. Para su conveniencia, hemos enumerado y vinculado a estos complementos a continuación.

Versiones admitidas

Native Player SDK para iOS ofrece soporte para las siguientes versiones de iOS:

Apoyo activo

Brightcove proporciona soporte activo para el último SDK de iOS en la última versión pública de las siguientes versiones de iOS:

  • iOS 14
  • iOS 13.7
  • iOS 12.5.3

Soporte pasivo

Brightcove proporciona soporte pasivo para las siguientes versiones de iOS:

  • iOS 11.4.1

Comprender los niveles de soporte

El soporte se define en términos de [versión principal].[versión menor].[versión de parche] de un sistema operativo (por ejemplo: iOS 11.4.1). Para aprovechar las últimas mejoras tecnológicas de Apple, ofrecemos dos niveles de soporte:

  • Activo: Brightcove probará y corregirá errores en estas versiones del sistema operativo.
  • Pasivo: Brightcove probará en estas versiones del sistema operativo, pero las correcciones de errores se manejarán caso por caso.

Generalmente, Brightcove proporciona soporte activo para las versiones del sistema operativo que utiliza el 80% de la base de usuarios y soporte pasivo para otras versiones utilizadas por al menos el 5% de la base de usuarios.

Empezando

Antes de comenzar el desarrollo con Player SDK para iOS, hay algunos conceptos que debe conocer:

  1. Medios de comunicación

    La versión actual de Native Player SDK admite la reproducción de contenido de su cuenta de Video Cloud a través de la API de reproducción o como activos remotos con URL accesibles a través de Internet.

  2. Analítica

    Con Native Player SDK, los clientes de Video Cloud obtienen automáticamente datos de análisis sólidos en Video Cloud.

  3. SDK principal y dependencias

    Brightcove Player SDK para iOS y sus complementos se pueden agregar a su proyecto usando el CocoaPods administrador de dependencias. Las podspecs para todos los componentes de Brightcove Player SDK están alojadas en el Repositorio de BrightcoveSpecs.

    Se requiere la versión 1.0 o posterior de CocoaPods.
  4. Referencia de SDK nativo

    Para ayudarlo a comprender el SDK, puede encontrar un desglose detallado de la arquitectura en Player SDK para iOS. Descubra cómo interactúan los componentes críticos para manejar las complejidades de controlar el reproductor de video nativo. Player SDK para instalación, inicio rápido y arquitectura de iOS

    Aquí también encontrará la última versión beta del SDK, si existe.

  5. Descripción general del SDK nativo

    Para obtener una descripción general rápida, aprenda cómo interactúan los componentes de Brightcove Player SDK con los componentes nativos de iOS. Descripción general de Player SDK para iOS

  6. Preguntas y respuestas

    Únase a nosotros en el Foro de desarrolladores de SDK donde los ingenieros de Brightcove y la comunidad responden a las preguntas relacionadas con el SDK. Busque preguntas previamente respondidas o publique una nueva. Como grupo público, puede obtener acceso proporcionando su dirección de correo electrónico.

  7. Entrenamiento en video

    Vea una serie de videos de capacitación que lo guiarán a través de los pasos para crear una aplicación Swift en Xcode que reproduzca un video de su biblioteca de Video Cloud.

Consejos:

  • Las vistas y los controles de PlayerUI (incluidos los controles de publicidad) utilizan la guía Diseño seguro de iOS para evitar una ubicación incorrecta en las esquinas, la región de la muesca o debajo del indicador de inicio.
  • El componente Sidecar Subtitles admite la reproducción a través de AirPlay con subtítulos. Para obtener detalles sobre cómo mantener los subtítulos sincronizados, consulte la Sincronización de subtítulos de WebVTT documento.

Notas:

  • Para mejorar el cumplimiento de CocoaPods, el SDK nativo para iOS 6.7.7 ha cambiado el nombre de algunos archivos Podspec. Para obtener más detalles, consulte la Referencia de SDK nativo.
  • El objetivo de implementación mínimo para todos los marcos de Brightcove Native Player para iOS es ahora iOS 11.0 y tvOS 11.0
  • Brightcove Native SDK para iOS 6.5.0+ requiere Xcode 11.0 o superior.
  • El recuento automático de referencias (ARC) es necesario en su proyecto. Este es el valor predeterminado actual para los proyectos de Xcode.
  • Con el lanzamiento de iOS 9 y Xcode 7, se introdujo una nueva función llamada App Transport Security (ATS). Aprender más sobre Trabajar con App Transport Security.
  • Un problema conocido de Apple con el contenido de transmisión de FairPlay es que no se procesará en el modo AirPlay Mirroring. Para obtener más detalles, consulte la Presentación WWDC: Protección de contenido para HLS (desplácese hasta la mitad de la transcripción).

La siguiente tabla contiene enlaces para integrar complementos con su implementación de SDK. Los complementos se pueden descargar de sus respectivos repositorios de Github, y cada repositorio contiene un archivo README detallado con instrucciones sobre cómo configurar el complemento.

Las aplicaciones de muestra demuestran la funcionalidad básica de cada complemento integrado con el SDK. Estas aplicaciones de muestra son una buena manera de aprender sobre cada complemento y sirven como punto de referencia al depurar.

La versión actual del SDK nativo para iOS incluye todos los componentes con el mismo número de versión. Al integrar componentes, el SDK y los números de versión del componente deben coincidir. Por ejemplo, al usar el componente de complemento IMA 6.7.0, debe usar el SDK nativo 6.7.0.

Aunque el complemento IMA de Brightcove utiliza un marco dinámico, la biblioteca IMA de Google solo se proporciona como marco estático.

Combinando complementos

Con el SDK nativo para iOS, el BCOVSessionProviders de cada complemento se crean y encadenan. Esta cadena se utiliza luego para crear el BCOVPlaybackController . Para obtener más detalles, consulte la Sección de combinación de complementos de la referencia SDK.

Publicidad con el complemento IMA

La ubicación de los anuncios VAST generalmente se especifica mediante un documento VMAP. VAST describe un anuncio, mientras que VMAP describe la ubicación de anuncios en un video. Para obtener detalles sobre el código, consulte la BasicIMAPlayer iOS muestra.

Al trabajar con anuncios IMA, se recomienda comprobar que sus anuncios funcionan fuera del SDK antes de implementarlos. Los siguientes enlaces serán útiles para esto:

  • Validador de respuesta a anuncios de Google VAST
  • Guía de listas de reproducción de anuncios múltiples de video digital (VMAP)
  • Foro técnico de IMA de Google

Notas de la plataforma

Las siguientes notas están relacionadas con la plataforma iOS.

Reflejando tu pantalla con Airplay

La funcionalidad AirPlay funciona de inmediato con el SDK nativo para iOS, lo que permite a los usuarios transmitir video a pantallas de alta definición con Apple TV.

Puede utilizar los controles de Brightcove o utilizar sus propios controles personalizados para AirPlay. Para obtener más detalles, consulte la Controles personalizados para AirPlay fragmento de código.

El uso de AirPlay con dispositivos más antiguos (gen 3) puede provocar un comportamiento inesperado. Es posible que el dispositivo de destino no sepa qué pistas de texto se seleccionaron en el dispositivo remitente.
