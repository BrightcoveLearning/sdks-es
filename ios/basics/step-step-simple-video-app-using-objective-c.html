    Paso a paso: Aplicación de vídeo simple usando Objective-C

    En este tema, aprenderá a crear una aplicación de reproducción de vídeo simple con el SDK de Brightcove Player para iOS. Lo construirá utilizando el lenguaje de programación Objective-C.

    Resumen

    En este ejemplo se reproduce un vídeo devuelto por la API de reproducción de Brightcove, que es la API más reciente y recomendada para recuperar contenido de la biblioteca de Video Cloud.

    Para esta opción, necesitará una clave de directiva. Si no está familiarizado con las claves de directiva, consulte Introducción a la API de directivas.

    Este ejemplo está escrito en Objective-C, pero también puede experimentar con Swift. Ver lo siguiente:
    • Paso a paso። Aplicación de vídeo simple usando Swift
    • Versión rápida del VideoCloudBasicPlayer

    ¡Póngase en marcha!

    En este ejemplo se reproducen vídeos de una matriz interna de archivos de vídeo.

    Crear el proyecto

    Configurar un proyecto en Xcode. A continuación, agregue el SDK de Brightcove Player junto con cualquier dependencia al proyecto.

    Configurar el proyecto Xcode

    Crea un nuevo proyecto de Xcode para la aplicación.

    1. Comience la creación de un nuevo proyecto iOS en Xcode. Seleccione Crear un nuevo proyecto de Xcode. Alternativamente, en el menú superior de Xcode, puede seleccionar Archivo, Nuevo, Proyecto.

    2. Elija Aplicación de vista única para la plantilla y seleccione Siguiente.

    3. Establezca la información del proyecto de la siguiente manera:

      • Nombre del producto: Simple-Reproducción de Vídeo
      • Equipo: ninguno

        El campo Equipo es opcional. Si quieres distribuir tu aplicación en el App Store, tendrás que seleccionar un equipo al que perteneces a través del Programa para desarrolladores de Apple. El nombre del equipo es necesario para la firma de código. En este ejemplo, seleccionaremos ninguno.

      • Nombre de la organización: nombre de su empresa
      • Identificador de la organización: com.example-company Esto hace que el nombre del producto sea único para la tienda de aplicaciones.

    4. Elija la ubicación en la que desea guardar el proyecto y créelo.

    5. Ahora cierre el proyecto (sí, ciérrelo — esto es importante!)

    Agregue el SDK y sus dependencias a su proyecto

    La forma más fácil de agregar el SDK y sus dependencias a su proyecto es usar CocoaPods.

    CocoaPods es un gestor de dependencias que agrega bibliotecas a su proyecto. No es necesario, pero facilita la instalación. Para instalar CocoaPods, consulte las instrucciones en el sitio de CocoaPods.

    • Se requiere la versión 1.0 o posterior de CocoaPods.
    • El uso de CocoaPods (la herramienta) en un proyecto Xcode existente modificará el archivo del proyecto.pod Es posible que desee hacer una copia de seguridad antes de hacer esto.

    1. En la carpeta del proyecto, cree un archivo de texto sin formato llamado Podfile (sin extensión de archivo).

    2. Con un editor de texto, agregue las siguientes líneas de código al Podfile y guárdelo. Este código hace lo siguiente:

      • Línea 1: Apunta a la ubicación de GitHub para las especificaciones del pod de Brightcove.
      • Línea 3: Establece Pods para usar marcos en lugar de bibliotecas estáticas.
      • Línea 5: Define la versión de la plataforma iOS.

      • Líneas 7-9: Instale el SDK de Brightcove Native Player.

        1. Obtenga la versión actual del SDK nativo desde el documento Descripción general del SDK nativo de Brightcove para iOS .
        2. Reemplace el valor por el número de versión actual del SDK. '6.3.11'

        3. Para obtener más información acerca de la administración de dependencias, consulte el artículo How To Manage Dependencies with CocoaPods .

         
        La propiedad se utiliza para CocoaPods versión 1.0 y posterior.target
      source 'https://github.com/brightcove/BrightcoveSpecs.git'

use_frameworks!

platform :ios, '11.0' # (or whatever version you want)

target 'Simple-Video-Playback' do
  pod 'Brightcove-Player-Core', '6.3.11'
end

    3. Abra una sesión de Terminal y vaya a su carpeta de proyecto Simple-Video-Playback Xcode.

    4. En la sesión de Terminal, escriba el comando siguiente:

      pod install

      y presione Retorno para ejecutar el it. Debería ver una serie de mensajes en el terminal indicando que el SDK de Brightcove Player se ha agregado al proyecto.

    5. Observe la última línea, que es importante: a partir de este punto, debe abrir el archivo Simple-Video-Playback.xcWorkspace en Xcode, no el archivo Simple-Video-Playback.xcodeproj. El comando crea este archivo para el proyecto.pod install.xcworkspace

    • Si crea y edita archivos Podfiles de forma regular, puede intentar usar la aplicación CocoaPods para completar los pasos anteriores.
    • Si no desea utilizar CocoaPods, puede descargar la última versión del SDK y agregarla manualmente a su proyecto.

    Establecer el comportamiento de audio de la aplicación

    La sesión de audio controla el comportamiento de audio en el nivel de la aplicación. Obtenga más información sobre la clase AvAudioSession.

    Puedes elegir entre varias categorías y ajustes de sesión de audio para personalizar el comportamiento de audio de tu aplicación. Elige la mejor categoría de sesión de audio para tu aplicación.

    Para esta muestra, vamos a utilizar AVAudioSessionCategoryPlayback. Reproduce audio incluso cuando la pantalla está bloqueada y con el interruptor de anilla/silencioso configurado en silencioso. Para que sea simple, pondremos el código para esto en el Delegado de la Aplicación.

    1. En su proyecto, abra el archivo de implementación de App Delegate (AppDelegate.m).

    2. En el didFinishLaunchingWithOptions método, agregue código para establecer la categoría de sesión de audio. Asegúrese de importar el marco.AVFoundation

      • Necesitamos el siguiente código para asegurarnos de que el audio se reproduzca cuando esperamos que lo haga. Por ejemplo, sin configurar este código, no oiremos el vídeo cuando el interruptor de silencio esté encendido.
      • Para simplificar la muestra, estamos poniendo esto en el delegado de la aplicación. Para obtener más información acerca de cómo usarlo en su propia aplicación, consulte la Guía de programación de sesiones de audio.
      //  AppDelegate.m
//  Simple-Video-Playback
//
//  Copyright © 2018 Brightcove. All rights reserved.
//

#import <AVFoundation/AVFoundation.h>

- (BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions {

NSError *categoryError = nil;
BOOL success = [[AVAudioSession sharedInstance] setCategory:AVAudioSessionCategoryPlayback error:&categoryError];

if (!success)
{
		NSLog(@"AppDelegate Debug - Error setting AVAudioSession category.  Because of this, there may be no sound. `%@`", categoryError);
}
return YES;
}

  [super viewDidLoad];
  // Do any additional setup after loading the view, typically from a nib.

  // Set up our player view. Create with a standard VOD layout.
  BCOVPUIPlayerView *playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:self.playbackController options:nil controlsView:[BCOVPUIBasicControlView basicControlViewWithVODLayout] ];

  _videoContainer addSubview:playerView];
  playerView.translatesAutoresizingMaskIntoConstraints = NO;
  [NSLayoutConstraint activateConstraints:@[
      [playerView.topAnchor constraintEqualToAnchor:_videoContainer.topAnchor],
      [playerView.rightAnchor constraintEqualToAnchor:_videoContainer.rightAnchor],
      [playerView.leftAnchor constraintEqualToAnchor:_videoContainer.leftAnchor],
      [playerView.bottomAnchor constraintEqualToAnchor:_videoContainer.bottomAnchor],
  ]];
  _playerView = playerView;

  // Associate the playerView with the playback controller.
  _playerView.playbackController = _playbackController;

  [self requestContentFromPlaybackService];
}

    Solicitar contenido de la biblioteca de Brightcove

    Para reproducir contenido de vídeo, solicitará un vídeo desde su biblioteca de Video Cloud.

    1. Debajo de la viewDidLoad función, cree una función llamada de requestContentFromPlaybackService la siguiente manera:

      - (void)requestContentFromPlaybackService
{
[self.playbackService findVideoWithVideoID:kViewControllerVideoID parameters:nil completion:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) {

	if (video)
	{
		[self.playbackController setVideos:@[ video ]];
	}
	else
	{
		NSLog(@"ViewController Debug - Error retrieving video: `%@`", error);
	}

}];
}

    Administrar seguridad de medios

    La Seguridad de transporte de aplicaciones (ATS) impone conexiones seguras entre la aplicación y los servicios web. Con el lanzamiento de Apple del SDK de iOS 9, se agregó una nueva característica llamada Seguridad de transporte de aplicaciones (ATS).

    1. Una de las siguientes situaciones se aplicará a tu aplicación:

      • De forma predeterminada, Brightcove Native SDK para iOS utiliza una directiva de selección de fuentes para elegir fuentes HTTPS en lugar de HTTP, de modo que pueda crear sus aplicaciones con ATS habilitado.

        Eso es todo, estás listo para ejecutar tu aplicación.

         

      • Si tus activos remotos usan HTTPS en lugar de fuentes HTTP, puedes crear tus aplicaciones con ATS habilitado.

        Eso es todo, estás listo para ejecutar tu aplicación.

         

      • Si está utilizando fuentes HTTP o tiene otras llamadas HTTP en su aplicación, puede aparecer el siguiente mensaje de error:

        App Transport Security has blocked a cleartext HTTP (http://) resource load since it is insecure.
Temporary exceptions can be configured via your app's Info.plist file.

        Esto significa que ATS está habilitado, pero el sistema no está configurado para cumplir con los requisitos de ATS. Para solucionar esta situación, consulte el documento Trabajar con seguridad de transporte de aplicaciones (ATS).
