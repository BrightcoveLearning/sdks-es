    Controles personalizados para AirPlay

    Si utiliza los controles de Brightcove PlayerUI, está todo configurado. La funcionalidad AirPlay funciona de forma inmediata, lo que permite a los usuarios transmitir vídeo a pantallas de alta definición con Apple TV.

    Si utiliza controles personalizados, puede seguir estos pasos:

    1. Obtén información sobre AirPlay en la documentación para desarrolladores de Apple para AirPlay.

    2. Utilice la allowsExternalPlayback propiedad del controlador de reproducción para establecer la allowsExternalPlayback propiedad de AVPlayer:

      _playbackController.allowsExternalPlayback = YES;

    3. Configure un control de enrutador AirPlay y gestione su selección. Para obtener más información, consulta el documento Descripción general de AirPlay de Apple .

    Personalización del botón de subtítulos opcionales

    El código de ejemplo BCOVPlayerUI muestra cómo personalizar el reproductor de Brightcove cuando se utiliza el SDK nativo para iOS. Para obtener más información, consulte la sección Personalización de controles de PlayerUI del documento de referencia Native SDK.

    Para personalizar el botón de subtítulos opcionales mediante el PlayerUI, siga estos pasos:

    1. Comience con la aplicación de reproducción de vídeo básica.

    2. Reemplace los valores de la clave de política, el ID de cuenta y el ID de vídeo. Selecciona un vídeo de tu cuenta que tenga pistas de texto.

    3. Configure la vista del reproductor con un diseño VOD estándar:

      // Set up our player view. Create with a standard VOD layout.
BCOVPUIPlayerView *playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController:self.playbackController options:nil controlsView:
[BCOVPUIBasicControlView basicControlViewWithVODLayout] ];

    4. El closedCaptionButton se declara como a BCOVPUIButton , que es una subclase de UIButton y agrega tres métodos adicionales para la personalización. Siempre que personalice los controles de BCOVPlayerUI, debe usar las API de Native Player siempre que estén disponibles. Su código personalizado debería ser similar a este:

      // Customize the CC button.
BCOVPUIButton *ccButton = playerView.controlsView.closedCaptionButton;
ccButton.titleLabel.font = [UIFont systemFontOfSize:14.];
ccButton.primaryTitle = @"CC";
[ccButton showPrimaryTitle:YES];

    Visualización del contenido de FairPlay en una pantalla externa

    Cuando una pantalla externa está conectada a un dispositivo iOS mediante un adaptador AV y un cable HDMI, el comportamiento predeterminado es reflejar la pantalla de iOS. La excepción a esto es cuando usa vídeo protegido por FairPlay, que Apple impide duplicar (WWDC 2015, sesión 502).

    Para mostrar vídeos protegidos por FairPlay, establezca las propiedades de AVPlayer expuestas a través del Controlador de reproducción de Brightcove para permitir que el vídeo de FairPlay se reproduzca en una pantalla externa. El vídeo se reproduce en modo de pantalla completa. Aquí hay un ejemplo de configuración de estas propiedades:

    playbackController.allowsExternalPlayback = YES;
playbackController.usesExternalPlaybackWhileExternalScreenIsActive = YES;

    Google Analytics

    Si utiliza el reproductor de Brightcove y la clase de catálogo, los análisis de vídeo se recopilarán automáticamente y aparecerán en el módulo Analytics de Video Cloud. Para obtener métricas adicionales, puedes agregar Google Analytics a tu aplicación.

    Para integrar Google Analytics con tu aplicación, sigue estos pasos:

    1. Revisa el documento de Google para añadir Analytics a tu aplicación iOS.

    2. Aquí hay una forma de usar Google Analytics para realizar un seguimiento de la reproducción de video usando el SDK de Firebase de Google:

      // This snippet shows one way to track video playback
// using the Firebase SDK from Google Analytics with
// the lifecycle event playback controller delegate method.
- (void)playbackController:(id<BCOVPlaybackController>)controller playbackSession:(id<BCOVPlaybackSession>)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent
{
// Common parameters
NSString *video_name = session.video.properties[kBCOVVideoPropertyKeyName];
NSString *video_ID = session.video.properties[kBCOVVideoPropertyKeyId];

// Session is ready to play
if ([kBCOVPlaybackSessionLifecycleEventReady isEqualToString:lifecycleEvent.eventType])
{
	[FIRAnalytics logEventWithName:@"bcov_video_ready"
    parameters:@{
        @"bcov_video_name": video_name,
        @"bcov_video_id": video_ID
    }];
}

// Session encountered an error
if ([kBCOVPlaybackSessionLifecycleEventError isEqualToString:lifecycleEvent.eventType])
{
	NSError *error = lifecycleEvent.properties[kBCOVPlaybackSessionEventKeyError];
	int error_code = error.code;
	[FIRAnalytics logEventWithName:@"bcov_video_playback_error"
    parameters:@{
        @"bcov_video_name": video_name,
        @"bcov_video_id": video_ID,
        @"bcov_video_error_code": @(error_code)
    }];
}

// Session has completed
if ([kBCOVPlaybackSessionLifecycleEventTerminate isEqualToString:lifecycleEvent.eventType])
{
	[FIRAnalytics logEventWithName:@"bcov_video_terminate"
    parameters:@{
        @"bcov_video_name": video_name,
        @"bcov_video_id": video_ID
     }];
}
}

     

    Limitar la velocidad de bits

    No puede controlar qué fuente (copia) en el manifiesto HLS es seleccionado por el AVPlayer , pero puede poner un límite de velocidad de bits en la reproducción. Esto impide que el reproductor utilice fuentes (copias) con una velocidad de bits sobre la velocidad de bits especificada.

    preferredPeakBitRate Establezca el límite deseado, en bits por segundo, del consumo de ancho de banda de red para el dado AVPlayerItem .

    Utilice una de las siguientes declaraciones:

    // Swift
var preferredPeakBitRate: Double
// Obj-C
@property(nonatomic) double preferredPeakBitRate

    Looping de un vídeo

    En algunos casos, es posible que desee que se reproduzca automáticamente un vídeo. Para hacer esto, puede obtener el evento del ciclo de vida de «fin del video», buscar hasta el principio y volver a jugar.

    Este código asume que ha establecido el delegado del PlaybackController en el objeto con este método:

    - (void)playbackController:(id<BCOVPlaybackController>)controller playbackSession:(id<BCOVPlaybackSession>)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent
{
  if ([kBCOVPlaybackSessionLifecycleEventEnd isEqualToString:lifecycleEvent.eventType])
  {
    [controller seekToTime:kCMTimeZero completionHandler:^(BOOL finished) {
    if (finished)
    	{
        [controller play];
    	}
    }];
  }
}

    Administrar vídeos en una lista de reproducción

    Una forma de administrar una lista de reproducción de vídeos es almacenar los objetos de vídeo en una tabla. Cuando el usuario selecciona un vídeo de la tabla, la fila de la tabla contendrá el objeto de vídeo.

    Aquí hay una descripción general de cómo funciona:

    1. Recupera una lista de reproducción de tu cuenta.

      - (void)retrievePlaylist
{
[self.refreshControl beginRefreshing];

// Retrieve a playlist through the BCOVPlaybackService
BCOVPlaybackServiceRequestFactory *playbackServiceRequestFactory = [[BCOVPlaybackServiceRequestFactory alloc]         initWithAccountId:kDynamicDeliveryAccountID
		policyKey:kDynamicDeliveryPolicyKey];
BCOVPlaybackService *playbackService = [[BCOVPlaybackService alloc] initWithRequestFactory:playbackServiceRequestFactory];

[playbackService findPlaylistWithReferenceID:kDynamicDeliveryPlaylistRefID
		parameters:nil
		completion:^(BCOVPlaylist *playlist, NSDictionary *jsonResponse, NSError *error)
{
	 [self.refreshControl endRefreshing];

	 NSLog(@"JSON Response:\n%@", jsonResponse);

	 if (playlist)
	 {
      self.currentVideos = playlist.videos.mutableCopy;
      self.currentPlaylistTitle = playlist.properties[@"name"];
      self.currentPlaylistDescription = playlist.properties[@"description"];

      NSLog(@"Retrieved playlist containing %d videos", (int)self.currentVideos.count);

      [self usePlaylist:self.currentVideos];
	 }
	 else
	 {
      NSLog(@"No playlist for ID %@ was found.", kDynamicDeliveryPlaylistRefID);
	 }

}];
}

    2. Vuelva a inicializar los contenedores que almacenan información relacionada con los vídeos de la lista de reproducción actual.

    3. Cuando se selecciona la vista de tabla, el índice de la fila se utiliza para crear una nueva videoDictionary. A continuación, pregunte al diccionario para el video. Si el video no es nulo, cargue el video en el playbackController.

      // Play the video in this row when selected
- (IBAction)tableView:(UITableView *)tableView didSelectRowAtIndexPath:(nonnull NSIndexPath *)indexPath
{
NSDictionary *videoDictionary = self.videosTableViewData[ (int)indexPath.row ];
BCOVVideo *video = videoDictionary[@"video"];

  if (video != nil)
  {
  	[self.playbackController setVideos:@[ video ]];
  }
}

    Para trabajar con listas de reproducción, puede almacenar la lista de reproducción en otro objeto, como una tabla. En función de la interacción del usuario, puede navegar por los índices del objeto y seleccionar el vídeo apropiado.

    Para obtener un ejemplo de código completo, consulte el ejemplo del Reproductor sin conexión .

    Valores de progreso de medios

    Durante la reproducción multimedia, los valores notificados al método delegado de progreso del Player SDK pueden incluir un valor inicial de infinito negativo y un valor final de infinito positivo. Estos valores se utilizan al procesar anuncios de pre-roll y post-roll.

    Si estos valores no son importantes para usted o interfieren con su propio seguimiento del progreso, pueden ignorarse fácilmente con una declaración condicional como esta:

    - (void)playbackController:(id<BCOVPlaybackController>)controller playbackSession:(id<BCOVPlaybackSession>)session didProgressTo:(NSTimeInterval)progress
{
if (progress < 0.0 || progress >= INFINITY)
return;
// Your code here
}

    Paginación con la API de reproducción

    Al recuperar el contenido de Video Cloud desde la API de reproducción, puede implementar la paginación de una lista de reproducción.

    Para realizar una página a través de un conjunto de vídeos de una lista de reproducción, utilice los siguientes parámetros de URL de solicitud:

    • limit - define el número de vídeos que se van a devolver desde la API de reproducción

    Agregar puntos de referencia mediante programación

    Los clientes de Video Cloud pueden agregar puntos de referencia a un vídeo mediante Video Cloud Studio, tal y como se muestra en el documento Agregar puntos de referencia a vídeos .

    También puede agregar puntos de referencia a su vídeo mediante programación. El siguiente código agrega puntos de referencia de intervalos trimestrales al vídeo devuelto desde la API de reproducción:

    // programmatically add cue points to a video
- (void)requestContentFromPlaybackService
{
[self.service findVideoWithVideoID:kViewControllerVideoID parameters:nil completion:^(BCOVVideo *video, NSDictionary *jsonResponse, NSError *error) {
if (video)
{
// Get the video duration from the properties dictionary
NSNumber *durationNumber = video.properties[@"duration"]; // milliseconds
float duration = durationNumber.floatValue / 1000.0; // convert to seconds
video = [video update:^(id<BCOVMutableVideo> mutableVideo)
{
// Add quarterly interval cue points of your own type
BCOVCuePoint *cp1 = [[BCOVCuePoint alloc] initWithType:@"your cue point type" position:CMTimeMake(duration * 250, 1000)];
BCOVCuePoint *cp2 = [[BCOVCuePoint alloc] initWithType:@"your cue point type" position:CMTimeMake(duration * 500, 1000)];
BCOVCuePoint *cp3 = [[BCOVCuePoint alloc] initWithType:@"your cue point type" position:CMTimeMake(duration * 750, 1000)];
BCOVCuePoint *cp4 = [[BCOVCuePoint alloc] initWithType:@"your cue point type" position:CMTimeMake(duration * 1000, 1000)];
// Create new cue point collection using existing cue points and new cue points
NSMutableArray *newCuePoints = [[NSMutableArray alloc] initWithArray:mutableVideo.cuePoints.array];
[newCuePoints addObject:cp1];
[newCuePoints addObject:cp2];
[newCuePoints addObject:cp3];
[newCuePoints addObject:cp4];
mutableVideo.cuePoints = [[BCOVCuePointCollection alloc] initWithArray:newCuePoints];
}];
[self.playbackController setVideos:@[ video ]];
}
else
{
NSLog(@"ViewController Debug - Error retrieving video: `%@`", error);
}
}];
}

    Tenga en cuenta que el valor de your cue point type puede ser cualquier valor de cadena que desee, siempre y cuando no esté utilizando ninguno de los complementos de iOS. Para obtener más información, consulte el documento de referencia del protocolo BCOvcuePoint .

    Si utiliza puntos de referencia con el complemento IMA, obtenga más información al respecto en la sección VAST y vMap/Server Lado Ad Rules del complemento IMA para las notas Native SDK para iOS. La aplicación de ejemplo de IMA muestra el valor necesario para los puntos de referencia de anuncio de IMA.

    El siguiente código escucha sus puntos de referencia y muestra un mensaje:

    // listen for cue points and display them
-(void)playbackController:(id<BCOVPlaybackController>)controller playbackSession:(id<BCOVPlaybackSession>)session didPassCuePoints:(NSDictionary *)cuePointInfo {
BCOVCuePointCollection *cpc = cuePointInfo[@"kBCOVPlaybackSessionEventKeyCuePoints"];
for (BCOVCuePoint *cp in cpc) {
  if ([cp.type isEqualToString:@"your cue point type"])
    {
    NSLog(@"Found your cue point at %f", CMTimeGetSeconds(cp.position));
    }
  }
}

    Extracción del reproductor

    Puede haber casos en los que desee eliminar el reproductor y la vista.

    La desasignación del controlador de vista que tiene la propiedad de un BCOVPlaybackController desasignará el controlador de reproducción también. Para ello, elimine la vista del reproductor de su supervista y configure el puntero del controlador de reproducción en nil.

    Aquí hay un ejemplo de código: 

    playerView?.removeFromSuperview()
playerView = nil
playbackController = nil

    Ajuste de la velocidad de reproducción

    Para controlar la velocidad de reproducción, puede establecer la rate propiedad en la AVPlayer clase expuesta en la sesión.

    Por defecto, la velocidad de reproducción sólo se puede ajustar en intervalos regulares (0.50, 0.67, 1.0, 1.25, 1.50 y 2.0). Al establecer el AudioTimePitchAlgorithm, puede usar valores de velocidad más granulares (como 1.7). Para obtener más detalles, consulte esta discusión de stackoverflow.

    avPlayerItem.audioTimePitchAlgorithm = AVAudioTimePitchAlgorithmVarispeed;

    Aunque este ejemplo utiliza el tipo de varispeed propiedad, la clase AvaudioTimePitchAlgorithm tiene 4 tipos de propiedad que puede usar:

    • LowQualityZeroLatency
    • espectral
    • TimeDomain
    • varispeado

    Observe que la varispeed opción no proporciona corrección de tono. Esto puede hacer que el audio sea anormalmente alto con una velocidad de reproducción alta, como 1.7. Elija la opción de propiedad que mejor se adapte a sus necesidades.

    Para un BCOVPlaybackSession , su código se vería similar a esto:

    - (void)playbackController:(id<BCOVPlaybackController>)controller
  playbackSession:(id<BCOVPlaybackSession>)session didReceiveLifecycleEvent:(BCOVPlaybackSessionLifecycleEvent *)lifecycleEvent {

  if ( [lifecycleEvent.eventType isEqualToString:kBCOVPlaybackSessionLifecycleEventReady] ) {
    NSLog(@"%@ at time %f", kBCOVPlaybackSessionLifecycleEventReady, CMTimeGetSeconds([session.player currentTime]));
    AVPlayerItem * avPlayerItem = [session.player currentItem];
    avPlayerItem.audioTimePitchAlgorithm = AVAudioTimePitchAlgorithmVarispeed;
    session.player.rate = 1.7;
  }
}
    El SDK nativo para iOS no admite valores negativos para la velocidad de reproducción (es decir, no se admite la reproducción de vídeos al revés).

    Ajuste del modo de gafas de realidad virtual para vídeos de 360°

    Al reproducir un vídeo de 360°, los usuarios pueden seleccionar el botón Video 360 de la barra de control para cambiar al modo de gafas de realidad virtual. También es posible que desee hacer esto mediante programación, antes de que comience la reproducción. Puede hacer esto actualizando el BCOVPlaybackController protocolo viewProjection propiedad de la siguiente manera:

    - (void)playbackController:(id<BCOVPlaybackController>)controller didAdvanceToPlaybackSession:(id<BCOVPlaybackSession>)session
{
BCOVVideo360ViewProjection *viewProjection = [self.playbackController.viewProjection copy];
[viewProjection setProjectionStyle:BCOVVideo360ProjectionStyleVRGoggles];
[self.playbackController setViewProjection:viewProjection];
}
    Tenga en cuenta que debe establecer el estilo de proyección del controlador en el método didAdvanceToPlaybackSession delegado.

    Cambio del color de fondo

    Al reproducir un vídeo en modo retrato, es posible que observe un borde negro en la parte superior y por debajo del reproductor. La vista del reproductor es del tamaño de la pantalla, pero el vídeo solo ocupa una pequeña parte del centro de la vista del reproductor. Las partes visibles alrededor del vídeo son el fondo de la capa del reproductor.

    Esto es un AVPlayer comportamiento normal. Se reduce el vídeo para caber dentro de la capa del reproductor, y el resto es el fondo de la capa del reproductor.

    Puede cambiar el fondo de la capa del reproductor con el siguiente código:

    (void)playbackController:(id<BCOVPlaybackController>)controller didAdvanceToPlaybackSession:(id<BCOVPlaybackSession>)session
{
session.playerLayer.backgroundColor = UIColor.whiteColor.CGColor;
}
