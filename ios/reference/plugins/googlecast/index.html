Referencia de Brightcove Native Player para iOS

Complemento de Google Cast para Brightcove Player SDK para iOS, versión 6.9.0.1697

Requisitos

  • Xcode 11.0+
  • ARCO

Plataformas compatibles

iOS 11.0 y superior.

Instalación

El complemento Google Cast para Brightcove Player SDK proporciona un marco de biblioteca estático para la instalación.

El complemento de Google Cast es compatible con la versión 4.5.0 del SDK de Google Cast para iOS. Las notas de la versión de Google Cast se pueden encontrar aquí.

CocoaPods

Puedes usar CocoaPods para agregar el complemento Google Cast para Brightcove Player SDK a su proyecto. Puede encontrar las últimas especificaciones de pods de Brightcove-Player-GoogleCast aquí. El pod incorporará la versión correcta de GoogleCast automáticamente. Se requiere CocoaPods 1.0 o más reciente.

Ejemplo de CocoaPod Podfile:

`` fuente bash ‘https://github.com/CocoaPods/Specs ' fuente ’https://github.com/brightcove/BrightcoveSpecs.git '

plataforma: ios, ‘11,0’

use_frameworks!

objetivo ‘ExampleApp’ hacer vaina ‘Brightcove-Player-GoogleCast’ final

La instalación de Brightcove-Player-GoogleCast instala automáticamente la dependencia, Brightcove-Player-Core. Según sus necesidades, puede optar por incluir el marco dinámico Brightcove-Player-Core o el estático. De forma predeterminada, se instala el marco Core dinámico. Para elegir el marco Core estático, su Podfile se vería así:

`` fuente bash ‘https://github.com/CocoaPods/Specs’ fuente ‘https://github.com/brightcove/BrightcoveSpecs.git’

plataforma: ios, ‘11,0’

use_frameworks!

objetivo ‘ExampleApp’ hacer vaina ‘Brightcove-Player-GoogleCast-static’ final

Al actualizar su instalación,’Es una buena idea actualizar la copia local de su repositorio BrightcoveSpecs para asegurarse de tener las últimas especificaciones de podspecs localmente, al igual que actualizaría su repositorio principal de CococaPods. Usar actualización del repositorio de pod para hacerlo.

Manual

Para agregar el complemento Google Cast para Brightcove Player SDK a su proyecto manualmente:

  1. Siga el Configuración manual del SDK de Google Cast guía.
  2. Siga el Instalación manual de Brightcove Player SDK guía.
  3. Descargar el Complemento de Google Cast para Brightcove Player SDK estructura.
  4. Sobre el “General” pestaña del destino de su aplicación, agregue BrightcoveGoogleCast.framework del complemento Google Cast para la descarga del SDK de Brightcove Player a la lista de Frameworks, bibliotecas, contenido incrustado.
  5. Sobre el “Configuración de compilación” pestaña del destino de su aplicación, asegúrese de que el “Rutas de búsqueda del marco” incluir las rutas a los marcos. Esto debería haberse hecho automáticamente a menos que el marco esté almacenado en un directorio raíz diferente al de su proyecto.

Importaciones

El complemento Google Cast para Brightcove Player SDK se puede importar en código de diferentes formas; @import BrightcoveGoogleCast; , #import <BrightcoveGoogleCast / BrightcoveGoogleCast.h> o #import <BrightcoveGoogleCast / [clase específica] .h>. Puede importar el GoogleCast y BrightcovePlayerSDK módulos de manera similar.

Antes de que empieces

Antes de intentar utilizar este complemento, ya debe estar familiarizado con lo siguiente:

  • Registre su solicitud. Esto lo guiará a través de la creación de su aplicación de receptor y le otorgará una identificación de aplicación. Comenzando, puede usar la ID de la aplicación de demostración ‘4F8B3483’.
  • Configuración para desarrollar con Cast Application Framework (CAF) para iOS . Esto lo guiará a través de la configuración de los derechos necesarios para su aplicación.
  • Inicializar el contexto del reparto. Esto lo guiará a través de la inicialización del contexto de transmisión con el ID de la aplicación de su receptor.
  • Agregar mini controladores. Esto lo guiará a través de cómo aprovechar el SDK de Google Cast’s mini controles multimedia.

Inicio rápido

El complemento BrightcoveGoogleCast es un puente entre SDK de Google Cast para iOS y el SDK de Brightcove Player para iOS.

Este fragmento muestra su uso básico.

[1] self.googleCastManager = [BCOVGoogleCastManager nuevo]; [2] id <BCOVPlaybackController> playbackController = [BCOVPlayerSDKManager.sharedManager createPlaybackController]; [3] [playbackController addSessionConsumer: self.googleCastManager];

Desglosando el código en pasos:

  1. Cree una referencia al BCOVGoogleCastManager único.
  2. Cree una instancia de BCOVPlaybackController para usar.
  3. Agrega tu BCOVGoogleCastManager referencia como consumidor de sesión a su instancia de BCOVPlaybackController.

Receptor CAF Brightcove

El ID de la aplicación para el receptor CAF de Brightcove es 341387A3 y se asigna a la constante kBCOVCAFReceiverApplicationID. Puede verificar el ID de la aplicación marcando el CAF Receiver config.json.

Si está utilizando el receptor CAF de Brightcove,’Necesitaré inicializar estas variables de esta manera:

[1] BCOVReceiverAppConfig * receiverAppConfig = [BCOVReceiverAppConfig nuevo]; receptorAppConfig.accountId = kAccountID; receptorAppConfig.policyKey = kServicePolicyKey; [2] self.googleCastManager = [[BCOVGoogleCastManager alloc] initForBrightcoveReceiverApp: receiverAppConfig];

Las siguientes propiedades también están disponibles para configurar BCOVReceiverAppConfig según sea necesario:

  • splashScreen (para personalizar la imagen de la pantalla de bienvenida)
  • playerUrl (para usar un reproductor personalizado)
  • authToken (para usar con PAS / EPA)
  • watermarkingToken (para usar con la fabricación de agua forense)
  • adConfigId (para usar con SSAI)
  • userId (para usar con seguimiento analítico)
  • applicationId (para usar con seguimiento analítico)

Métodos delegados

BCOVGoogleCastManagerDelegate tiene métodos delegados que puede usar para recibir notificaciones cuando ocurran o estén a punto de ocurrir eventos importantes relacionados con el reparto, por ejemplo

  • - (void) SwitchedToLocalPlayback: (NSTimeInterval) lastKnownStreamPosition;
  • - (anulado) SwitchedToRemotePlayback;
  • - (void) currentCastedVideoDidComplete;
  • - (void) castedVideoFailedToPlay;
  • - (vacío) adecuadoSourceNotFound;
  • - (vacío) willBuildMediaInformationBuilder: (GCKMediaInformationBuilder * _Nonnull);
  • - (void) willSendMediaLoadOptions: (GCKMediaLoadOptions * _Nonnull);

Para obtener una lista completa con descripciones, consulte la Referencia de protocolo BCOVGoogeCastManagerDelegate.

Para aprovechar estos eventos, establezca un delegado en el BCOVGoogleCastManager único.

 self.googleCastManager.delegate = self;

Selección de fuente

los BCOVGoogleCastManager intentará encontrar una fuente adecuada para usar. Buscará fuentes en este orden:

  1. HTTPS HLS v3
  2. HTTPS DASH
  3. HTTPS MP4
  4. HTTP HLS v3
  5. HTTP DASH
  6. HTTP MP4

Si no se encuentra ninguno de estos, el método delegado adecuadoSourceNotFound sera llamado.

Su cuenta de Video Cloud deberá configurarse para admitir HLS v3, asegúrese de que DASH esté habilitado o tenga una fuente MP4 disponible para cada video.

Personalizaciones

Hay dos propiedades, además de la propiedad de delegado, que puede establecer en el BCOVGoogleCastManager clase. Estos son:

  • GCKImage * fallbackPosterImage: La GCKImage que se utilizará cuando no haya una imagen de póster disponible para un video.
  • Póster CGSizeImageSize: El alto y el ancho que desea usar para la imagen del objeto GCKImage que se crea. El valor predeterminado es 480h x 720w.

Limitaciones / problemas conocidos

Cuando se utiliza un receptor predeterminado sin modificar (incluido el receptor de demostración), se aplican las siguientes limitaciones:

  • DRM no es compatible.
  • No se admiten varias pistas de audio.
  • La publicidad del lado del cliente y del lado del servidor no es compatible.
  • No se admiten transmisiones de DVR en vivo y en vivo.

Al utilizar el receptor CAF de Brightcove, se aplican las siguientes limitaciones:

  • No se admite la publicidad del lado del cliente.
