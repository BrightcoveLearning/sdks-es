Complemento FreeWheel para Brightcove Player SDK para iOS, versión 6.9.0.1697

Requisitos

  • Xcode 11.0+
  • ARCO

Plataformas compatibles

iOS 11.0 y superior.

tvOS 11.0 y superior.

Instalación

El complemento FreeWheel para Brightcove Player SDK proporciona un marco de biblioteca estático para la instalación. Se agregará un marco dinámico en el futuro cuando FreeWheel lance una versión dylib.

El complemento FreeWheel para Brightcove Player SDK se ha integrado con la versión 6.38.0 del marco FreeWheel AdManager que está disponible en el Descargas de integración del SDK de AdManager página.

CocoaPods

Puedes usar CocoaPods para agregar el complemento FreeWheel para Brightcove Player SDK a su proyecto. Puedes encontrar lo último Brightcove-Player-FreeWheel podspec aquí. Para usar esta especificación, agregue lo siguiente al principio de Podfile: fuente 'https://github.com/brightcove/BrightcoveSpecs.git '. Se requiere CocoaPods versión 1.0 o superior.

El SDK de FreeWheel no es incluido en esta vaina. usted debe agregue manualmente el FreeWheel SDK AdManager.framework a su proyecto. Sin embargo, el pod agregará todas las bibliotecas requeridas por el marco de trabajo AdManager.framework.

Ejemplo de marco estático:

fuente 'https://github.com/brightcove/BrightcoveSpecs.git' pod 'Brightcove-player-freewheel'

Al actualizar su instalación,’Es una buena idea actualizar la copia local de su repositorio BrightcoveSpecs para que tenga las últimas podspecs localmente, al igual que actualizaría su repositorio principal de CococaPods.

Normalmente, si usa actualización de la vaina en Terminal esto sucederá automáticamente, o alternativamente puede actualizar explícitamente con actualización de repositorio de pod Brightcove. (Su repositorio de BrightcoveSpecs puede tener un nombre diferente si lo agregó explícitamente a su lista de repositorios de podspecs).

Manual

Para agregar el complemento FreeWheel para Brightcove Player SDK a su proyecto manualmente:

  1. Instale la última versión del SDK de Brightcove Player y siga las instrucciones para instalar el [Marco estático [https://github.com/brightcove/brightcove-player-sdk-ios/#Instalación manual] .
  2. Descargue la última versión comprimida del complemento de nuestra página de lanzamiento.
  3. Añade el ‘BrightcoveFW.framework’ a su proyecto. Puede hacer esto haciendo clic con el botón derecho en la carpeta Frameworks y elegir “Agregar archivos a” opción y seleccione BrightcoveFW.framework de la ruta donde está almacenado.
  4. Sobre el “Configuración de compilación” pestaña del destino de su aplicación, asegúrese de que el “Rutas de búsqueda del marco” incluir la ruta al marco. Esto debería haberse hecho automáticamente a menos que el marco esté almacenado en un directorio raíz diferente al de su proyecto.
  5. Sobre el “Fases de construcción” pestaña del destino de su aplicación, agregue lo siguiente a la “Vincular binarios con bibliotecas” fase:
    • BrightcoveFW.framework
  6. Sobre el “Configuración de compilación” pestaña del destino de su aplicación:
    • Asegurarse de que -ObjC ha sido agregado a la “Otras banderas del vinculador” entorno de construcción.
  7. Instale la biblioteca FreeWheel, que debe recuperar de su cuenta FreeWheel.

Importaciones

El complemento FreeWheel para Brightcove Player SDK se puede importar en código de diferentes formas; @import BrightcoveFW; , #importar <BrightcoveFW / BrightcoveFW.h> o #import <BrightcoveFW / [clase específica] .h>.

Reproducir y pausar

Brightcove FreeWheel Plugin implementa una lógica de pausa y reproducción personalizada para garantizar la experiencia publicitaria más fluida posible. Por lo tanto, deberá asegurarse de utilizar el método de reproducción en el BCOVPlaybackController o la [BCOVSessionProviderExtension fw_play] o [BCOVSessionProviderExtension fw_pause] (BCOVSessionProviderExtension y no el AVPlayer.

Como ejemplo, llamar al juego por primera vez en BCOVPlaybackController permite que Brightcove FreeWheel Plugin procese anuncios pre-roll sin que se reproduzca nada del contenido antes del pre-roll.

Uso de la PlayerUI incorporada

Si está utilizando la versión 5.1 o posterior de Brightcove Player SDK, puede aprovechar los controles de anuncios integrados con el complemento Brightcove FreeWheel.

Para usar PlayerUI, cree su controlador de reproducción como en el código de inicio rápido anterior.

Ranuras de precarga

Dependiendo de las condiciones de la red, puede haber un retraso en el tiempo que tardan en reproducirse los anuncios, una vez que se ha alcanzado la posición del anuncio. Para mejorar el rendimiento de los anuncios, es posible precargar las ranuras FreeWheel. Para activar la precarga, habilite preloadSlots sobre BCOVFWSessionProviderOptions.

Cuando está habilitado, las ranuras de pre-roll se cargarán lo antes posible. Para midrolls, las ranuras se precargarán una cantidad configurable de segundos, determinada por timeBeforeAdToPreload sobre BCOVFWSessionProviderOptions.

Usando una duración precisa

Dependiendo de su configuración de FreeWheel, proporcionar una duración precisa en las llamadas a setVideoId: es la diferencia entre recibir el número correcto de midrolls o no.

De forma predeterminada, el SDK esperará para llamar al BCOVFWSessionProviderAdContextPolicy hasta que AVPlayerItem tenga una duración precisa, y este valor se proporcionará a la llamada de bloqueo. Si no desea esperar hasta que la duración de AVPlayerItem esté lista y tenga esta información disponible de otra fuente (como las propiedades de BCOVVideo / BCOVSource o su propio CMS), waitForAVPlayerDuration se puede utilizar para deshabilitar la espera.

Manejo de Búsquedas

Cuando se buscan varios grupos de anuncios (como dos ranuras de midrolls, en diferentes posiciones), el SDK proporciona un mecanismo conveniente para determinar qué grupos de anuncios se reproducen. Esto se puede modificar cambiando el valor predeterminado BCOVCuePointProgressPolicy sobre BCOVFWSessionProviderOptions. Para obtener más información sobre BCOVCuePointProgressPolicy , consulte el BCOVFWSessionProvider.h expediente.

Seguimiento de errores de FreeWheel

Eventos de error de FreeWheel (FWAdErrorEvent ) encontrado a través del FWAdEventNotification se pasará a través de la playbackController: playbackSession: didReceiveLifecycleEvent: método delegado con un tipo de evento de kBCOVFWLifecycleEventAdError. Todo el objeto userInfo que el FreeWheel SDK envía junto con la notificación se incluye en las propiedades de los eventos y se puede acceder a él con el kBCOVFWLifecycleEventPropertyKeyAdError llave.

A continuación, se muestra un ejemplo de cómo detectar estos errores:

- (void) playbackController: (id <BCOVPlaybackController>) controller playbackSession: (id <BCOVPlaybackSession>) sesión didReceiveLifecycleEvent: (BCOVPlaybackSessionLifecycleEvent *) lifecycleEvent { if ([kBCOVFWLifecycleEventAdError isEqualToString:lifecycleEvent.eventType]) { NSDictionary *errorInfo = lifecycleEvent.properties[kBCOVFWLifecycleEventPropertyKeyAdError]; NSLog(@"FreeWheel Error Encountered: %@", errorInfo[FWInfoKeyErrorInfo]); } }
