Instalación

Pulse Plugin para Brightcove Player SDK proporciona un marco de biblioteca dinámico para la instalación.

El complemento Pulse es compatible con INVIDI Technologies Pulse SDK versión 2.5.20.1.0 para iOS y versión 2.5.19.9.0 para tvOS, Referencia del SDK de Pulse iOS y tvOS. Podspec para el complemento Pulse para referencias de Brightcove Player SDK Historial de versiones de Pulse SDK.

CocoaPods

Puedes usar CocoaPods para agregar Pulse Plugin para Brightcove Player SDK a su proyecto. Puedes encontrar lo último Brightcove-Player-Pulse podspec aquí. PulseSDK debe agregarse a su proyecto, descargue la última versión aquí. Se requiere CocoaPods 1.0 o más reciente.

Ejemplo de CocoaPod Podfile:

source 'https://github.com/CocoaPods/Specs' source' https://github.com/brightcove/BrightcoveSpecs.git' platform:ios, '11.0' use_frameworks! objetivo 'MyApp' do pod 'Brightcove-Player Pulse' end

Manual

Para agregar el complemento Pulse para Brightcove Player SDK a su proyecto manualmente:

  1. Descargue la última estructura comprimida del SDK de Brightcove Player desde la página de lanzamientos.
  2. Descargue la última versión comprimida de Pulse Plugin para Brightcove Player SDK framework desde el página de lanzamientos.
  3. Descargar el PulseSDK estructura.
  4. Sobre el “General” pestaña del destino de su aplicación, agregue el dinámica framework, BrightcovePlayerSDK.framework, desde la descarga de Brightcove Player SDK a la lista de Frameworks, bibliotecas y contenido incrustado. El marco dinámico, BrightcovePlayerSDK.framework, se encuentra en el directorio ios / dynamic de la descarga. El Empotrar El entorno para el marco debe ser “Insertar y firmar.”
  5. Sobre el “General” pestaña del destino de su aplicación, agregue BrightcovePulse.framework del Pulse Plugin para la descarga del SDK de Brightcove Player a la lista de Frameworks, bibliotecas y contenido incrustado. El Empotrar El entorno para el marco debe ser “Insertar y firmar.”
  6. Sobre el “General” pestaña de su aplicación objetivo, agregue PulseSDK.framework de la descarga de INVIDI Technologies a la lista de Frameworks, bibliotecas y contenido incrustado. El Empotrar El entorno para el marco debe ser “Insertar y firmar.”
  7. Sobre el “Configuración de compilación” pestaña del destino de su aplicación, asegúrese de que el “Rutas de búsqueda del marco” incluir las rutas a los marcos. Esto debería haberse hecho automáticamente a menos que el marco esté almacenado en un directorio raíz diferente al de su proyecto.
  8. Sobre el “Configuración de compilación” pestaña del destino de su aplicación:
    • Asegurarse de que -ObjC ha sido agregado a la “Otras banderas del vinculador” entorno de construcción.

Importaciones

Pulse Plugin para Brightcove Player SDK se puede importar en código de diferentes formas; @import BrightcovePulse; , #importar <BrightcovePulse / BrightcovePulse.h> o #import <BrightcovePulse / [clase específica] .h>. Puede importar el PulseSDK y BrightcovePlayerSDK módulos de manera similar.

Inicio rápido

El complemento BrightcovePulse es un puente entre PulseSDK y el SDK de Brightcove Player para iOS. Este fragmento muestra su uso básico con las reglas de anuncios del lado del servidor.

[1] OOContentMetadata * contentMetadata = [OOContentMetadata nuevo]; contentMetadata.category = <categoría>; contentMetadata.tags = @ [<tag1>, <tag2>, ..., <tagN>]; contentMetadata.flags = @ [<flag1>, <flag2>, ..., <flagN>]; OORequestSettings * requestSettings = [OORequestSettings nuevo]; requestSettings.linearPlaybackPositions = @ [@ (posición1), @ (posición2), ..., @ (posiciónN)]; NSString * pulseHost = <dominio de pulso>; [2] NSDictionary * pulseOptions = @{ kBCOVPulseOptionPulsePlaybackSessionDelegateKey: self, kBCOVPulseOptionPulsePersistentIdKey: [NSUUID.UUID UUIDString] } ; BCOVPlayerSDKManager * manager = [BCOVPlayerSDKManager sharedManager]; id <BCOVPlaybackController> controller = [3] [manager createPulsePlaybackControllerWithPulseHost: pulseHost contentMetadata: contentMetadata requestSettings: requestSettings adContainer: self.playerView.contentOverlayView companionSlots: nil viewStrategy: nil] opciones; controller.delegate = self; [videoContainerView addSubview: controller.view]; NSString * policyKey = <your-policy-key>; NSString * accountId = <your-ccount-id>; NSString * videoID = <your-video-id>; BCOVPlaybackService * playbackService = [[BCOVPlaybackService alloc] initWithAccountId: accountID policyKey: policyKey]; [playbackService findVideoWithVideoID: videoID parameters: nil completado: ^ (BCOVVideo * video, NSDictionary * jsonResponse, NSError * error) { [controller setVideos:@[ video ]]; [controller play]; }];

Desglosando el código en pasos:

  1. Crea lo mismo OOContentMetadata , OORequestSettings que crearía si estuviera usando INVIDI Technologies’s Pulse SDK de iOS directamente. Pulse Plugin requiere un dominio válido y un contenedor UIView para mostrar los anuncios.
  2. Opcionalmente, el complemento puede recibir un NSDictionary con propiedades que se utilizarán en el complemento. Las claves disponibles son kBCOVPulseOptionPulsePlaybackSessionDelegateKey , kBCOVPulseOptionPulseDeviceContainerKey y kBCOVPulseOptionPulsePersistentIdKey. Para anular la sesión de pulso, se debe implementar PulsePlaybackSessionDelegate, pasando el kBCOVPulseOptionPulsePlaybackSessionDelegateKey con la clase que implementa el método.
  3. BrightcovePulse agrega algunos métodos de categoría a BCOVPlaybackManager. El primero de estos es -createPulsePlaybackControllerWithPulseHost: contentMetadata: requestSettings: adContainer: companionSlots: viewStrategy: opciones:. Utilice este método para crear su controlador de reproducción.

Reproducir y pausar

Brightcove Pulse Plugin implementa una lógica de pausa y reproducción personalizada para garantizar la experiencia publicitaria más fluida posible. Por lo tanto, deberá asegurarse de utilizar el método de reproducción en el BCOVPlaybackController o la [BCOVSessionProviderExtension pulse_play] o [BCOVSessionProviderExtension pulse_pause] (BCOVSessionProviderExtension y no el AVPlayer.

Como ejemplo, llamar al juego por primera vez en BCOVPlaybackController permite a BCOVPulse procesar anuncios pre-roll sin que se reproduzca nada del contenido antes del pre-roll. Para obtener más información sobre cómo BCOVPulse anula el valor predeterminado BCOVPlaybackController métodos, por favor echa un vistazo BCOVSessionProviderExtension.

Uso de la PlayerUI incorporada

En tus UIViewController , crear un BCOVPUIPlayerView propiedad llamada vista de reproductor, para contener los controles de reproducción, la vista de contenido de video y una vista especial donde Pulse puede mostrar sus anuncios.

// Vista del reproductor de PlayerUI @property (no atómico) BCOVPUIPlayerView * playerView;

Luego crea tu vista de jugador; suministre un controlador de reproducción nulo que se agregará más adelante. Esta vista de reproductor contiene tanto la vista de contenido de video como la vista que muestra controles de reproducción y controles de anuncios. Esta configuración es la misma sin importar qué complemento esté utilizando. Configure la vista del reproductor para que coincida con el contenedor de video de su diseño (videoView ) cuando cambia de tamaño.

// Cree y configure la Vista de control. BCOVPUIBasicControlView * controlView = [BCOVPUIBasicControlView basicControlViewWithVODLayout]; // Crea la vista del reproductor con un controlador de reproducción nulo. self.playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController: ninguna opción: ninguna controlView: controlView]; // Agrega BCOVPUIPlayerView a tu vista de video. [self.videoView addSubview: self.playerView];

usted’Necesitará configurar el diseño para la vista del jugador, puede hacerlo con Auto Layout o con el método anterior Springs & Struts.

Resortes y puntales self.playerView.frame = self.videoView.bounds; self.playerView.autoresizingMask = UIViewAutoresizingFlexibleHeight | UIViewAutoresizingFlexibleWidth;

Diseño automático self.playerView.translatesAutoresizingMaskIntoConstraints = NO; [NSLayoutConstraint activeConstraints: @ [[self.playerView.topAnchor constrintEqualToAnchor: self.videoView.topAnchor], [self.playerView.rightAnchor constraintEqualToAnchor: self.videoView.rightAnchor], [self.playerView.leftAnchorvideoAnchorrest.quot. , [self.playerView.bottomAnchor constraintEqualToAnchor: self.videoView.bottomAnchor],]];

La creación del controlador de reproducción es específica de Pulse. Cree su controlador de reproducción como lo hizo anteriormente, pero en lugar de su vista de contenedor de video, pase el contentOverlayView desde la vista del jugador como tu adContainer. los contentOverlayView es una vista especial que se utiliza para superponer vistas en el contenido del video principal. También deberías usar nulo en lugar de [manager defaultControlsViewStrategy] si estuvieras usando eso como tu viewStrategy (este era el método más antiguo para usar controles integrados).

// Crea el controlador de reproducción. id <BCOVPlaybackController> controller = [manager createPulsePlaybackControllerWithPulseHost: pulseHost contentMetadata: contentMetadata requestSettings: requestSettings adContainer: self.playerView.contentOverlayView companionSlots: nil viewStrategy: nil opciones: nil]; controller.delegate = self; // Asignar un nuevo controlador de reproducción a la vista del reproductor. // Esto asocia la sesión del playerController con PlayerUI. // Puede mantener esta vista de reproductor y asignarle nuevos // controladores de reproducción a medida que se crean. self.playerView.playbackController = self.playbackController;

Ahora, cuando reproduzca videos con anuncios, verá los controles de PlayerUI mientras reproduce contenido de video, además de marcadores de anuncios en el depurador de la línea de tiempo.

PlayerUI es altamente personalizable. Para obtener más información y código de muestra, consulte Diseños personalizados sección en el archivo README.md del Repositorio de Brightcove Native Player SDK.

Personalización del comportamiento del complemento

Hay un par de puntos de configuración en BCOVPulse. Puede combinar BCOVPulse con otro complemento para Brightcove Player SDK para iOS, puede crear una estrategia de visualización personalizada y puede anular la solicitud de anuncios actual.

BCOVPulsePlaybackSessionDelegate

los BCOVPulsePlaybackSessionDelegate El protocolo proporciona una forma para que el complemento BrightcovePulse se comunique con la aplicación host. Este delegado le permite anular los metadatos del contenido y solicitar configuraciones antes de que comience la sesión.

los UIViewController necesita adoptar el BCOVPulsePlaybackSessionDelegate protocolo.

@interface ViewController () <BCOVPulsePlaybackSessionDelegate>

Cree un NSDictionary pasando el kBCOVPulseOptionPulsePlaybackSessionDelegateKey con la clase que implementará el método para anular la sesión.

NSDictionary * pulseOptions = @{ kBCOVPulseOptionPulsePlaybackSessionDelegateKey: self } ;

Crear un BCOVPlaybackSessionProvider o BCOVPlaybackController para Pulse utilizando los métodos en BCOVPlaybackManager , recuerde pasar el diccionario creado en el paso anterior. Este ejemplo usa un controlador de reproducción.

BCOVPlayerSDKManager * manager = [BCOVPlayerSDKManager sharedManager]; id <BCOVPlaybackController> controller = [manager createPulsePlaybackControllerWithPulseHost: pulseHost contentMetadata: [OOContentMetadata nuevo] requestSettings: [OORequestSettings nuevo] adContainer: self.playerView.contentOverlayView companionSlots: nil viewStrategy opciones:

los createSessionForVideo: withPulseHost: contentMetadata: requestSettings: El método proporciona el video actual, el dominio de host, metadatos de contenido y solicitar configuración para la sesión. En este ejemplo, los objetos anteriores estaban vacíos y se anularán con un nuevo categoría y posiciones de reproducción lineal formación.

 - (id<OOPulseSession>)createSessionForVideo:(BCOVVideo *)video withPulseHost:(NSString *)pulseHost contentMetadata:(OOContentMetadata *)contentMetadata requestSettings:(OORequestSettings *)requestSettings { // Override the content metadata. contentMetadata.category = @"new_category_for_midrolls"; // Override the request settings. requestSettings.linearPlaybackPositions = @[ @(15) ]; return [OOPulse sessionWithContentMetadata:contentMetadata requestSettings:requestSettings]; }

Ver estrategia

Puede proporcionar una estrategia de vista personalizada a BCOVPlaybackManager cuando está construyendo su controlador de reproducción o proveedor de sesión, en lugar de especificar defaultControlsViewStrategy directamente. Con una estrategia de vista personalizada, la vista del contenedor de anuncios y los espacios para anuncios complementarios se pueden vincular con la vista de contenido de video. Este es un ejemplo de estrategia de vista personalizada.

BCOVPlaybackControllerViewStrategy customViewStrategy = ^ UIView * (UIView * vista, id <BCOVPlaybackController> playbackController){ BCOVPlaybackControllerViewStrategy defaultControlsViewStrategy = [playbackManager defaultControlsViewStrategy]; UIView *contentAndDefaultControlsView = defaultControlsViewStrategy(view, playbackController); [<UIView of video container> addSubview:contentAndDefaultControlsView]; return <UIView of video container>; } ;

Proveedores de sesiones de composición

Si está utilizando más de un complemento para Brightcove Player SDK para iOS que necesita crear un controlador de reproducción personalizado, en su lugar debe componer una cadena de proveedores de sesión y pasar el proveedor de sesión final al [BCOVPlayerSDKManager createPlaybackControllerWithSessionProvider: viewStrategy:] método.

Al componer proveedores de sesiones, la precarga de sesiones se puede habilitar desde BCOVBasicSessionProvider.

Preguntas frecuentes

Puedo escuchar los anuncios, pero puedo’t verlos jugar.

Esto suele ocurrir cuando la vista del contenedor de anuncios no está en la jerarquía de vistas o cuando la vista del anuncio (que es una subvista de la vista del contenedor de anuncios) está cubierta por otras vistas.
