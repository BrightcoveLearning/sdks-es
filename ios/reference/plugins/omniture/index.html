Referencia de Brightcove Native Player para iOS

Complemento de Omniture (Adobe Marketing Cloud - Adobe Analytics) para Brightcove Player SDK para iOS, versión 6.9.0.1697

Requisitos

  • Xcode 11.0+
  • ARCO

Plataformas compatibles

iOS 11.0 y superior.

Instalación

El complemento Omniture es un marco de biblioteca estático que admite Adobe Marketing Cloud v4.14.1 y Adobe Video Heartbeat v2.0.1. Bibliotecas de Marketing Cloud y Video Heartbeat no son incluido con este SDK y debe agregarse manualmente a su proyecto. Las instrucciones para descargar las bibliotecas se proporcionan a continuación.

CocoaPods

Puedes usar CocoaPods versión 1.0 o superior para agregar el complemento Omniture a su proyecto. Puedes encontrar lo último Brightcove-Player-Omniture podspec aquí. Para usar esta especificación, agregue lo siguiente en la parte superior de su Podfile: fuente 'https://github.com/brightcove/BCOVSpecs.git '.

Ejemplo de marco estático:

`` fuente bash ‘https://github.com/brightcove/BrightcoveSpecs.git '

objetivo ‘MyOmniturePlayer’ hacer vaina ‘Brightcove-Player-Omniture’ final

Al actualizar su instalación,’Es una buena idea actualizar la copia local de su repositorio BrightcoveSpecs para que tenga las últimas podspecs localmente, al igual que actualizaría su repositorio principal de CococaPods.

Normalmente, si usa actualización de la vaina en Terminal esto sucederá automáticamente, o alternativamente puede actualizar explícitamente con actualización del repositorio de pod.

Manual

Para agregar el complemento Omniture para Brightcove Player SDK a su proyecto manualmente:

  1. Instale la última versión del SDK de Brightcove Player.
  2. Descargue la última versión comprimida del complemento desde el página de lanzamiento.
  3. Añade el BrightcoveAMC.framework proyecto. Puede hacer esto haciendo clic con el botón derecho en la carpeta Frameworks y elegir “Agregar archivos a” opción y seleccione la BrightcoveAMC.framework de la ruta donde se almacena.
  4. Sobre el “Configuración de compilación” pestaña del destino de su aplicación, asegúrese de que el “Rutas de búsqueda del marco” incluir la ruta al marco. Esto debería haberse hecho automáticamente a menos que el marco esté almacenado en un directorio raíz diferente al de su proyecto.
  5. Sobre el “Fases de construcción” pestaña del destino de su aplicación, agregue lo siguiente a la “Vincular binarios con bibliotecas” fase:
    • BrightcoveAMC.framework
  6. Sobre el “Configuración de compilación” pestaña del destino de su aplicación:
    • Asegurarse de que -ObjC ha sido agregado a la “Otras banderas del vinculador” entorno de construcción.
  7. Descargue Adobe Marketing Cloud siguiendo Adobe’s instrucciones y añadir AdobeMobileLibrary.a a su proyecto.
  8. Descargue Adobe Video Heartbeat siguiendo Adobe’s instrucciones y añadir VideoHeartbeat.a a su proyecto.
  9. Agregue las carpetas de encabezado de Marketing Cloud y Video Heartbeat a la configuración de Ruta de búsqueda de encabezado de su proyecto.

Inicio rápido

BrightcoveAMC es un puente entre el SDK de Brightcove Player para iOS y seguimiento de video de Adobe, que incluye Adobe Video Heartbeat v2 y Adobe Media Tracking v4. BrightcoveAMC realiza un seguimiento de los comportamientos de video básicos: video listo, video completado, reproducción de video y pausa de video.

Implementación del cliente

Para configurar Adobe Video Heartbeat, los clientes del complemento Omniture deben implementar instancias de BCOVAMCVideoHeartbeatConfigurationPolicy. Para configurar Adobe Media Tracking, se requiere una instancia de BCOVAMCMediaSettingPolicy. Estas politicas permiten la personalización de objetos ADBMediaHeartbeatConfig o ADBMediaSettings para cada sesión de reproducción.

Video Heartbeat v2

Este ejemplo usa el seguimiento de latidos de video.

[1] // Crea el objeto de política de configuración de VHB. BCOVAMCVideoHeartbeatConfigurationPolicy videoHeartbeatConfigurationPolicy = ^ ADBMediaHeartbeatConfig * (id <BCOVPlaybackSession> sesión) { ADBMediaHeartbeatConfig *configData = [[ADBMediaHeartbeatConfig alloc] init]; configData.trackingServer = <adobe-assigned_tracking_server>"; configData.channel = <cutomize_sample_channel>; configData.appVersion = <app_version>; configData.ovp = <online_video_platform>; configData.playerName = <player_name>; configData.ssl = <YES | NO>; // Set debugLogging to true to activate debug tracing. Remove it in production. configData.debugLogging = YES; return configData; } ; [2] // Cree el objeto de política de análisis AMC de Brightcove desde el objeto de política de configuración de VHB. BCOVAMCAnalyticsPolicy * heartbeatPolicy = [[BCOVAMCAnalyticsPolicy alloc] initWithHeartbeatConfigurationPolicy: videoHeartbeatConfigPolicy]; BCOVAMCSessionConsumer * sessionConsumer = [BCOVAMCSessionConsumer heartbeatAnalyticsConsumerWithPolicy: heartbeatPolicy delegate: self]; BCOVPlayerSDKManager * manager = [BCOVPlayerSDKManager sharedManager]; id <BCOVPlaybackController> controlador = [administrador createPlaybackController]; controller.delegate = self; [self.view addSubview: controller.view]; [3] // Agregue el consumidor de sesión Brightcove AMC al controlador de reproducción. [controlador addSessionConsumer: sessionConsumer]; [4] // Encuentra y reproduce un video. NSString * policyKey = <your-policy-key>; NSString * accountId = <your-ccount-id>; NSString * videoID = <your-video-id>; BCOVPlaybackService * service = [[BCOVPlaybackService alloc] initWithAccountId: accountId policyKey: policyKey]; [service findVideoWithVideoID: videoID parameters: nil completado: ^ (BCOVVideo * video, NSDictionary * jsonResponse, NSError * error) { [controller setVideos:@[ video ]]; [controller play]; }];
  1. Cree el bloque de política de configuración de latido de video que se llamará al inicio de cada sesión de reproducción. La política permite la personalización de los datos de configuración en función de la sesión actual.
  2. Utilice el bloque de política de configuración para crear e inicializar una instancia de BCOVAMCAnalyticsPolicy con [initWithHeartbeatConfigurationPolicy:]. El objeto BCOVAMCAnalyticsPolicy se utiliza para crear BCOVAMCSessionConsumer.
  3. Una vez creado el controlador de reproducción, llame al método de agregar consumidor de sesión, [addSessionConsumer:] , para agregar el consumidor de sesión AMC.

Nube de marketing

Este ejemplo usa seguimiento de medios.

[1] BCOVAMCMediaSettingPolicy mediaSettingPolicy = ^ ADBMediaSettings * (id <BCOVPlaybackSession> sesión) { ADBMediaSettings *settings = [ADBMobile mediaCreateSettingsWithName:<cutomize_setting_name> [2] length:0 playerName:<cutomize_player_name> playerID:<cutomize_player_ID>]; [3] settings.milestones = @"25,50,75"; // a customization. return settings; } ; [4] BBCOVAMCAnalyticsPolicy * mediaPolicy = [[BCOVAMCAnalyticsPolicy alloc] initWithMediaSettingsPolicy: mediaSettingPolicy]; BCOVAMCSessionConsumer * sessionConsumer = [BCOVAMCSessionConsumer mediaAnalyticsConsumerWithPolicy: mediaPolicy delegate: self]; BCOVPlayerSDKManager * manager = [BCOVPlayerSDKManager sharedManager]; id <BCOVPlaybackController> controlador = [administrador createPlaybackController]; controller.delegate = self; [self.view addSubview: controller.view]; [5] [controlador addSessionConsumer: sessionConsumer]; NSString * policyKey = <your-policy-key>; NSString * accountId = <your-ccount-id>; NSString * videoID = <your-video-id>; BCOVPlaybackService * service = [[BCOVPlaybackService alloc] initWithAccountId: accountId policyKey: policyKey]; [service findVideoWithVideoID: videoID parameters: nil completado: ^ (BCOVVideo * video, NSDictionary * jsonResponse, NSError * error) { [controller setVideos:@[ video ]]; [controller play]; }];
  1. Cree el bloque de políticas de configuración de medios que se llamará al inicio de cada sesión de reproducción. La política permite la personalización de la configuración de medios en función de la sesión actual.
  2. Al crear la instancia de ADBMediaSettings con un método de clase de ADBMobile + [mediaCreateSettingsWithName: length: playerName: playerID:] , puedes configurar duración del video para 0. El complemento de Omniture lo actualizará más tarde.
  3. Agregue configuraciones opcionales como hitos.
  4. Utilice el bloque de políticas de configuración de medios para crear e inicializar una instancia de BCOVAMCAnalyticsPolicy con [initWithMediaSettingsPolicy:]. El objeto de política se utiliza para crear un consumidor de sesión AMC para el seguimiento de medios de Adobe como + [mediaAnalyticsConsumerWithPolicy: delegate:].
  5. Una vez creado el controlador de reproducción, llame al método de agregar consumidor de sesión, [addSessionConsumer:] , para agregar el consumidor de sesión AMC.
