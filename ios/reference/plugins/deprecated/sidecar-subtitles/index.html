Referencia de subtítulos de BCOVSidecar

Complemento de subtítulos Sidecar para Brightcove Player SDK para iOS y tvOS, versión 2.1.6.129

Plataformas compatibles

iOS 8.0 y superior.

tvOS 9.0 y superior.

Instalación

El complemento Sidecar Subtitles para Brightcove Player SDK proporciona dos paquetes de instalación para iOS; un marco de biblioteca estático y un marco dinámico. Para tvOS, el complemento Sidecar Subtitles solo proporciona un marco dinámico.

CocoaPods

Puedes usar CocoaPods para agregar el complemento de subtítulos Sidecar para Brightcove Player SDK a su proyecto. Puedes encontrar lo último Brightcove-Player-SDK-SidecarSubtítulos podspec aquí.

Especificar el grupo predeterminado Brightcove-Player-SDK-SidecarSubtítulos instalará el marco de la biblioteca estática. Para instalar el marco dinámico, declare el pod con el dinámica subespecificación: Brightcove-Player-SDK-Sidecar Subtítulos / dinámico

Ejemplo de marco estático:

pod 'Brightcove - Player - SDK - Side Car Subtítulos'

Ejemplo de marco dinámico:

pod 'Brightcove-player-SDK-Sidecar subtítulos de car/dinámico'

Es necesario mantener un repositorio maestro de especificaciones de pods actualizado para asegurarse de que siempre está utilizando las últimas versiones del software Brightcove. A partir de CocoaPods 1.0.0, las actualizaciones de repositorios de podspec ya no son una función automática, por lo que para actualizar su repositorio principal, ejecute lo siguiente en la línea de comando:

actualización del repositorio de pod

Manual

Para agregar el complemento de subtítulos Sidecar para Brightcove Player SDK a su proyecto manualmente:

  1. Instale la última versión del SDK de Brightcove Player.
  2. Descargue la última versión comprimida del complemento BrightcoveSidecarSubtitles de nuestro página de lanzamiento.
  3. Agregar BrightcoveSidecarSubtitles.framework a su proyecto.
  4. Sobre el “Configuración de compilación” pestaña del destino de su aplicación, asegúrese de que el “Rutas de búsqueda del marco” incluir la ruta al marco. Esto debería haberse hecho automáticamente a menos que el marco esté almacenado en un directorio raíz diferente al de su proyecto.
  5. (Solo marco dinámico) En el “General” pestaña del destino de su aplicación, agregue ‘BrightcoveSidecarSubtitles.framework’ al “Binarios integrados” sección.
  6. (Solo marco dinámico) En el “Fases de construcción” pestaña, agregue un “Ejecutar guión” fase con el comando bash ${BUILT_PRODUCTS_DIR}PS{FRAMEWORKS_FOLDER_PATH}/BrightcoveSidecarSubtitles.framework/strip-frameworks.sh. Cheque “Ejecute el script solo al instalar”. Esto eliminará las arquitecturas innecesarias de la compilación, lo cual es importante para el envío de la App Store.
  7. (Solo marco estático) En el “Configuración de compilación” pestaña del destino de su aplicación, agregue -ObjC al “Otras banderas del vinculador” entorno de construcción.

Importaciones

El Brightcove Player SDK para iOS se puede importar en código de diferentes formas:

  • @import BrightcoveSidecarSubtitles;
  • #import <BrightcoveSidecarSubtitles / BrightcoveSidecarSubtitles.h>
  • #import <BrightcoveSidecarSubtitles / [clase específica] .h>.

Inicio rápido

 NSString * policyKey = <your-policy-key>; NSString * accountId = <your-ccount-id>; NSString * videoID = <your-video-id>; BCOVPlayerSDKManager * manager = [BCOVPlayerSDKManager sharedManager]; [1] id <BCOVPlaybackController> controller = [playbackManager createSidecarSubtitlesPlaybackControllerWithViewStrategy: nil]; [self.view addSubview: controller.view]; BCOVPlaybackService * playbackService = [[BCOVPlaybackService alloc] initWithAccountId: accoundId policyKey: policyKey]; [2] [playbackService findVideoWithVideoID: parámetros de videoID: finalización nula: ^ (BCOVVideo * video, NSDictionary * jsonResponse, NSError * error) { [controller setVideos:@[ video ]]; [controller play]; }];

BCOVSidecarSubtitles agrega algunos métodos de categoría a BCOVPlaybackManager. El primero de estos es -createSidecarSubtitlesPlaybackControllerWithViewStrategy:. Utilice este método para crear su controlador de reproducción. Alternativamente (si está utilizando más de un proveedor de sesión), puede crear un BCOVSSSessionProvider y pasarlo al método de administrador que crea un controlador de reproducción con proveedores de sesión ascendentes. * Si está desarrollando para tvOS, la ViewStrategy pasada a createSidecarSubtitlesPlaybackControllerWithViewStrategy debe ser nulo.

* Tenga en cuenta que BCOVSSSessionProvider debe aparecer antes que cualquier proveedor de sesión de la cadena que se pase al administrador al construir el controlador de reproducción. Este complemento es no compatible con el complemento Widevine.

Si tiene preguntas o necesita ayuda, visite el foro de soporte para los SDK de Brightcove Native Player en https://groups.google.com/forum/#! forum / brightcove-native-player-sdks.

Uso de los controles integrados de PlayerUI

El fragmento de código anterior presenta un reproductor de video sin ningún control. Puede agregar controles de reproducción a su código de esta manera.

Agregue una propiedad para realizar un seguimiento de la BCOVPUIPlayerView:

// Vista del jugador de PlayerUI @property (no atómico) BCOVPUIPlayerView * playerView;

Crea el BCOVPUIBasicControlView , y luego el BCOVPUIPlayerView. Aquí es donde asociamos el controlador de reproducción (y por lo tanto todos los videos que reproduce) con los controles. Configure la vista del reproductor para que coincida con el contenedor de video de su diseño (yourVideoView ) cuando cambia de tamaño.

BCOVPUIBasicControlView * controlView = [BCOVPUIBasicControlView basicControlViewWithVODLayout]; self.playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController: opciones del controlador: nil controlsView: controlView]; // Igualar el tamaño de la vista principal self.playerView.frame = self.yourVideoView.bounds; self.playerView.autoresizingMask = UIViewAutoresizingFlexibleHeight | UIViewAutoresizingFlexibleWidth; // Agrega BCOVPUIPlayerView a tu vista de video. [self.yourVideoView addSubview: self.playerView];

Si usa BCOVPUIPlayerView, también debe eliminar una línea de arriba:

[self.view addSubview: controller.view]; // ya no es necesario cuando se usa PlayerUI

Si desea reutilizar el reproductor con otro controlador de reproducción, simplemente puede realizar una nueva asignación:

self.playerView.playbackController = otroPlaybackController;

La vista del reproductor agregará automáticamente el controlador de reproducción’s vista a su propia jerarquía de vista.

Consulte el SDK del reproductor nativo de Brightcove’s README para obtener más información sobre cómo agregar y acumular controles PlayerUI en su aplicación.

Si tiene preguntas o necesita ayuda, visite el Foro de soporte de Brightcove Native Player SDK.

Completar manualmente los datos de los subtítulos

El complemento BCOVSidecarSubtitle buscará la presencia de una matriz de metadatos de subtítulos en el BCOVVídeo propiedades del objeto, codificadas por kBCOVSSVideoPropertiesKeyTextTracks. Si esta usando BCOVPlaybackService para recuperar videos y esos videos tienen pistas de texto asociadas, esto se completará automáticamente.

Si está proporcionando sus propios videos o es un cliente de Perform, deberá estructurar los datos como se muestra a continuación:

NSArray * subtítulos = @ [@{ kBCOVSSTextTracksKeySource: ..., // required kBCOVSSTextTracksKeySourceLanguage: ..., // required kBCOVSSTextTracksKeyLabel: ..., // required kBCOVSSTextTracksKeyDuration: ..., // required/optional [1] kBCOVSSTextTracksKeyKind: kBCOVSSTextTracksKindSubtitles or kBCOVSSTextTracksKindCaptions, // required kBCOVSSTextTracksKeyDefault: ..., // optional kBCOVSSTextTracksKeyMIMEType: ..., // optional } , @{...} , // segundo diccionario de pista de texto @{...} , // diccionario de la tercera pista de texto]; BCOVVideo * video = [BCOVVideo alloc] initWithSource: <source> cuePoints: <cuepoints> propiedades: @{ kBCOVSSVideoPropertiesKeyTextTracks:subtitles }];

los kBCOVSSTextTracksKeySource La clave contiene la URL de origen de la pista de subtítulos y se puede proporcionar como una URL de WebVTT o una URL de lista de reproducción M3U8.

Los archivos WebVTT deben tener un “.vtt” extensión, y los archivos M3U8 deben tener una “.M3U8” extensión. Si no puede seguir estas convenciones, debe incluir una kBCOVSSTextTracksKeySourceType clave y especifique kBCOVSSTextTracksKeySourceTypeWebVTTURL o kBCOVSSTextTracksKeySourceTypeM3U8URL para indicar el tipo de archivo al que hace referencia la URL.

Si está proporcionando pistas a un video recuperado de Video Cloud, debe agregar sus subtítulos a cualquier pista existente en lugar de sobrescribir ellos. Este código muestra cómo puede agregar pistas a un video existente:

BCOVVideo * updatedVideo = [actualización de video: ^ (id <BCOVMutableVideo> mutableVideo) { // Save the current tracks NSArray *originalTracks = video.properties[kBCOVSSVideoPropertiesKeyTextTracks]; // Create your text track dictionary NSArray *subtitles = @[ @{ kBCOVSSTextTracksKeySource: ..., // required kBCOVSSTextTracksKeySourceLanguage: ..., // required kBCOVSSTextTracksKeyLabel: ..., // required kBCOVSSTextTracksKeyDuration: ..., // required/optional [1] kBCOVSSTextTracksKeyKind: kBCOVSSTextTracksKindSubtitles or kBCOVSSTextTracksKindCaptions, // required kBCOVSSTextTracksKeyDefault: ..., // optional kBCOVSSTextTracksKeyMIMEType: ..., // optional } , @{...} , // segundo diccionario de pista de texto @{...} , // diccionario de la tercera pista de texto]; // Agregar pistas nuevas a las pistas originales, si hay alguna NSArray * mixedTextTracks = ((originalTracks! = Nil)? [OriginalTracks arrayByAddingObjectsFromArray: subtitles]: subtitles); // Actualiza el diccionario actual (no queremos perder las propiedades que ya están ahí) NSMutableDictionary * updatedDictionary = [mutableVideo.properties mutableCopy]; // Almacenar pistas de texto en la propiedad de pistas de texto updatedDictionary [kBCOVSSVideoPropertiesKeyTextTracks] = mixedTextTracks; mutableVideo.properties = updatedDictionary; }];

Notas

  • kBCOVSSTextTracksKeyDuration es una clave obligatoria si utiliza archivos de subtítulos con extensión .vtt. kBCOVSSTextTracksKeyDuration es una clave opcional si está utilizando archivos de subtítulos con una extensión .m3u8.

Consulte la documentación del código en el archivo de encabezado BCOVSSComponent.h para obtener más información sobre el uso de estas claves.

Problemas conocidos

  • Los subtítulos no se mostrarán al ver videos de 360 grados.

  • Actualmente, este complemento no admite la integración con el complemento Widevine para Brightcove Brightcove Player SDK para iOS.

  • Al recuperar sus videos de Brightcove Playback API, sus interpretaciones deben incluir una lista de reproducción maestra M3U8. El complemento Sidecar Subtitle no funciona con listas de reproducción M3U8 de una sola interpretación.

  • Si está proporcionando una lista de reproducción de subtítulos para el complemento, y esa lista de reproducción de subtítulos incluye enlaces a archivos web vtt que responden como 404, la reproducción fallará. Este es un error en Apple’s AVPlayer.
