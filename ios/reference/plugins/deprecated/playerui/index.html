Referencia de BCOVPlayerUI

Complemento BCOVPlayerUI para Brightcove Player SDK para iOS, versión 1.0.4.143

Plataformas compatibles

iOS 7.0 y superior.

Instalación

Brightcove PlayerUI Plugin proporciona dos paquetes de instalación para iOS, un marco de biblioteca estático y un marco dinámico. El destino de la biblioteca estática admite la implementación en iOS 7, mientras que el marco dinámico solo es compatible con iOS 8 y versiones posteriores.

CocoaPods

Puedes usar CocoaPods para agregar el complemento PlayerUI para Brightcove Player SDK a su proyecto. Puedes encontrar lo último Brightcove-Player-SDK-Player-UI podspec aquí.

Ejemplo de marco estático:

pod 'Brightcove-player-SDK-Player-UI'

Ejemplo de marco dinámico:

pod 'Brightcove-player-SDK-reproductor-UI/dinámico'

Manual

Para agregar el complemento PlayerUI para Brightcove Player SDK a su proyecto manualmente:

  1. Instale la última versión del SDK de Brightcove Player.
  2. Descargue la última versión comprimida del complemento de nuestra página de lanzamiento.
  3. Añade el ‘BrightcovePlayerUI.framework’ a su proyecto.
  4. Sobre el “Configuración de compilación” pestaña del destino de su aplicación, asegúrese de que el “Rutas de búsqueda del marco” incluir la ruta al marco. Esto debería haberse hecho automáticamente a menos que el marco esté almacenado en un directorio raíz diferente al de su proyecto.
  5. Sobre el “Fases de construcción” pestaña del destino de su aplicación, agregue lo siguiente a la “Vincular binarios con bibliotecas” fase:
    • BrightcovePlayerUI.framework
  6. (Solo marco dinámico) En el “General” pestaña del destino de su aplicación, agregue ‘BrightcovePlayerUI.framework’ al “Binario incrustado” sección.
  7. (Solo marco dinámico) En el “Fases de construcción” pestaña, agregue un “Ejecutar guión” fase con el comando bash ${BUILT_PRODUCTS_DIR}PS{FRAMEWORKS_FOLDER_PATH}/BrightcovePlayerUI.framework/strip-frameworks.sh. Cheque “Ejecute el script solo al instalar”. Esto eliminará las arquitecturas innecesarias de la compilación, lo cual es importante para el envío de la App Store. ([rdar: // 19209161] [19209161])
  8. (Solo marco estático) En el “Configuración de compilación” pestaña del destino de su aplicación, agregue -ObjC al “Otras banderas del vinculador” entorno de construcción.
  9. (Solo marco estático) Agregar el archivo bcovpuiiconfont.ttf desde el BrightcovePlayerUI.framework agrupe directamente en la lista de su proyecto para que el archivo de fuente se copie en el paquete de la aplicación. Al final, el archivo de fuente debería terminar al mismo nivel que la aplicación.’s Info.plist expediente. El archivo de fuente proporciona algunos de los elementos de la interfaz BrightcovePlayerUI, pero no es necesario que esté incluido en el plist.

Importaciones

El Brightcove Player SDK para iOS se puede importar en código de diferentes formas; @import BrightcovePlayerUI; , #import <BrightcovePlayerUI / BrightcovePlayerUI.h> o #import <BrightcovePlayerUI / [clase específica] .h>.

Inicio rápido

Su uso es relativamente sencillo. Este ejemplo crea un diseño básico adecuado para transmisiones de video bajo demanda generales:

 BCOVPlayerSDKManager * manager = [BCOVPlayerSDKManager sharedManager]; [1] id <BCOVPlaybackController> controller = [manager createPlaybackController]; [2] BCOVPUIPlayerView * playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController: playbackController]; playerView.frame = self.videoContainer.bounds; playerView.autoresizingMask = UIViewAutoresizingFlexibleHeight | UIViewAutoresizingFlexibleWidth; [self.videoContainer addSubview: playerView]; [3] Símbolo NSString *; // (token de Brightcove Media API con acceso URL) NSString * playlistID; // (ID de la lista de reproducción que desea utilizar) BCOVCatalogService * catalog = [[BCOVCatalogService alloc] initWithToken: token]; [catálogo findPlaylistWithPlaylistID: parámetros de playlistID: finalización nula: ^ (BCOVPlaylist * playlist, NSDictionary * jsonResponse, NSError * error) { [controller setVideos:playlist]; [controller play]; }];

Dejar’s divide este código en pasos, para que sea un poco más fácil de digerir:

  1. Crea un controlador de reproducción con su configuración predeterminada.
  2. Crea una vista de reproductor usando el controlador de reproducción. Esto creará una instancia de una vista de jugador con su configuración predeterminada. Configure la vista del reproductor para que coincida con el tamaño de la vista del contenedor de video.
  3. Recupere y reproduzca un video.

Personalización del comportamiento del complemento

El complemento BCOVPlayerUI proporciona una experiencia fácil de usar. De todos modos, eso’Es posible cambiar el comportamiento predeterminado.

BCOVPUIPlayerViewOptions

los BCOVPUIPlayerViewOptions class le permite personalizar algunos comportamientos de BCOVPlayerUI en la inicialización. Puedes personalizar:

  • jumpBackInterval El tiempo en segundos que el jugador saltará hacia atrás cuando se presione el botón de salto hacia atrás.
  • hideControlsInterval El tiempo en segundos después del último evento táctil, antes de que se oculten los controles.
  • hideControlsAnimationDuration El tiempo en segundos que tardan los controles en animarse a ocultarse.
  • showControlsAnimationDuration El tiempo en segundos que tardan los controles en animarse y volverse visibles.
  • presentViewController El controlador de vista que se usa para presentar cosas como modales y pantalla completa.

Las opciones se pueden configurar mediante el siguiente método:

 BCOVPlayerSDKManager * manager = [BCOVPlayerSDKManager sharedManager]; id <BCOVPlaybackController> controlador = [administrador createPlaybackController]; BCOVPUIPlayerViewOptions * opciones = [[BCOVPUIPlayerViewOptions alloc] init]; options.jumpBackInterval = 5; BCOVPUIPlayerView * playerView = [[BCOVPUIPlayerView alloc] initWithPlaybackController: opciones de control de reproducción: opciones];

Diseños suministrados

Se proporcionan tres diseños para admitir diferentes tipos de video:

  • BCOVPUIControlLayout basicVODControlLayout es un diseño básico para transmisiones de video bajo demanda generales.

  • BCOVPUIControlLayout basicLiveControlLayout es un diseño dirigido a videos en vivo.

  • BCOVPUIControlLayout basicLiveDVRControlLayout está dirigido a transmisiones de video en vivo con controles DVR.

Por lo general, establece un nuevo diseño inmediatamente después de su BCOVPUIPlayerView se crea, pero también puede establecer un nuevo diseño en cualquier momento. Por ejemplo, puede establecer un nuevo diseño de VOD como este:

playerview.controlView.layout = [BCOVPUIControlLayout basicVODControlLayout]

Diseños personalizados

Además de los diseños predeterminados, puede crear sus propios diseños altamente personalizados creando una nueva instancia BCOVPUIControlLayout con tu propio diseño.

Primero, cree los controles que irán en su diseño usando BCOVPUIBasicControlView layoutViewWithControlFromTag: ancho: elasticidad:. Cada control está empaquetado en un BCOVPUILayoutView que determina el espaciado de control. Puede configurar el ancho de cada vista de diseño al ancho predeterminado (que se basa en el tipo de control), o puede especificar su propio ancho.

los elasticidad El argumento determina cuánto la vista de diseño que contiene el control redimensiona su ancho para llenar la barra de control. Una elasticidad de cero significa que el tamaño de la vista de diseño será fijo. Un valor de elasticidad mayor que cero determina cuánto crecerá la vista de diseño para llenar el espacio disponible en relación con todas las demás vistas elásticas en esa barra de control. Una vista de diseño con un valor de elasticidad de 2.0 crecerá dos veces más rápido que una vista de diseño con un valor de elasticidad de 1.0. Normalmente, una fila de vistas de diseño tendrá al menos un control con una elasticidad mayor que cero.

A continuación, se muestran ejemplos de cómo crear una variedad de controles básicos.

Tenga en cuenta que también puede crear una vista de diseño vacía, en la que puede colocar su propia vista (logotipo, control, nada, etc.). Este código muestra cómo colocar un logotipo de UIImage dentro del logoLayoutView1 que creamos arriba.

// Cree una imagen de logotipo dentro de una vista de imagen para mostrarla en la barra de control. UIImage * logoImage1 = [UIImage imageNamed: @ "myLogo"]; UIImageView * logoImageView1 = [[UIImageView alloc] initWithImage: logoImage1]; logoImageView1.autoresizingMask = UIViewAutoresizingFlexibleWidth | UIViewAutoresizingFlexibleHeight; logoImageView1.contentMode = UIViewContentModeScaleAspectFit; logoImageView1.frame = logoLayoutView1.frame; // Agrega una vista de imagen a nuestra vista de diseño vacía. [logoLayoutView1 addSubview: logoImageView1];

Ahora que tenemos una colección de controles empaquetados en vistas de diseño, podemos ordenarlos en arreglos, cada uno de los cuales representa una sola línea de controles, es decir, una barra de control. Tenga en cuenta que puede tener diferentes diseños para las orientaciones vertical y horizontal, por lo que normalmente configurará dos matrices diferentes de barras de control.

En este diseño estándar para orientación horizontal, organizaremos nuestros controles en una sola línea y luego colocaremos esa matriz en otra matriz que representa la configuración completa.

 NSArray * standardLayoutLine1 = @ [playbackLayoutView, jumpBackButtonLayoutView, currentTimeLayoutView, timeSeparatorLayoutView, durationLayoutView, progressLayoutView, spacerLayoutView1, logoLayoutView1, spacerLayoutView2, closedCaptionLayoutLayoutLayoutView, externalViewModeRouteView; NSArray * standardLayoutLines = @ [standardLayoutLine1];

En el diseño compacto, para la orientación vertical, creamos dos arreglos de controles, y luego esos dos arreglos se empaquetan en otro arreglo que representa el diseño compacto.

Tenga en cuenta que usamos exactamente los mismos objetos para la mayoría de los controles en cada diseño. Una vez hecho esto, y cambie entre las orientaciones vertical y horizontal, el objeto se moverá a su nueva posición mediante una animación suave.

 NSArray * compactLayoutLine1 = @ [currentTimeLayoutView, progressLayoutView, durationLayoutView]; NSArray * compactLayoutLine2 = @ [playbackLayoutView, jumpBackButtonLayoutView, spacerLayoutView1, closedCaptionLayoutView, screenModeLayoutView, externalRouteLayoutView, logoLayoutView2]; NSArray * compactLayoutLines = @ [compactLayoutLine1, compactLayoutLine2];

Finalmente, ahora que tenemos dos configuraciones de diseño, una para ancho completo y otra para ancho compacto, podemos crear una nueva BCOVPUIControlLayout objeto y ponerlo en el reproductor’s vista de control.

 BCOVPUIControlLayout * customLayout = [[BCOVPUIControlLayout alloc] initWithStandardControls: standardLayoutLines compactControls: compactLayoutLines]; playerView.controlView.layout = customLayout;

Si tiene controles que necesita mostrar u ocultar con frecuencia, puede configurar el remoto propiedad en ese control’s vista de diseño. Cuando haya cambiado sus controles, llame setNeedsLayout en el playerView’s controlView:

 logoLayoutView1.removed = SÍ; [playerView.controlsView setNeedsLayout];

También puede personalizar varios BCOVPUIControlLayout propiedades:

  • controlBarHeight establece el tamaño de cada fila de controles.
  • horizontalItemSpacing establece el espacio entre cada BCOVPUILayoutView en cada barra de control.
  • compactLayoutMaximumWidth determina qué conjunto de controles se utiliza. Si la vista de control es menor que compactLayoutMaximumWidth , se utilizará el conjunto de control compacto; de lo contrario, se utilizarán los controles estándar.

Para cambiar el conjunto de controles que se muestran, debe crear e instalar un nuevo BCOVPUIControlLayout.

Preguntas frecuentes

¿ Funciona esto con todos los complementos de Brightcove Player SDK?

Esta versión admite la reproducción básica de VOD, Live y DVR en vivo sin publicidad. Agregaremos soporte para otros complementos en futuras versiones.

¿ Cómo puedo personalizar propiedades distintas de las mencionadas anteriormente?

Nuestro objetivo es crear algo que’Es fácil de personalizar, sin dejar de ofrecer una gran cantidad de funciones y comportamientos listos para usar. En las próximas versiones, cubriremos más personalización con nuevas API y más documentación. Si hay una función que está buscando, háganoslo saber en nuestro foro.

¿ Habrá cambios importantes en la API en las versiones futuras?

Haremos todo lo posible para no cambiar las cosas a menos que realmente sea necesario. Sin embargo, si necesitamos realizar cambios en la API, proporcionaremos pasos de migración claros en el CHANGELOG.
