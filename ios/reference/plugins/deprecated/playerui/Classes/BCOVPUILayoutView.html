Descripción general

Los objetos de vista BCOVPUILayoutView se utilizan para diseñar controles en un BCOVPUIBasicControlView. Las vistas de diseño se crean normalmente con BCOVPUIBasicControlView’s + layoutViewWithControlFromTag: width: elasticity: method y contienen un elemento estándar de la interfaz de usuario del reproductor, como un botón de reproducción, una etiqueta de código de tiempo o un control deslizante de progreso. También puede crear vistas de diseño vacías en las que agregar sus propias subvistas. Las propiedades de la vista de diseño determinan exactamente cómo se colocan las vistas entre sí en diseños de ancho variable.

  Mínimo Ancho

El ancho más pequeño que se establecerá para la vista cuando se cambie el tamaño para el diseño. Si elasticidad se establece en cero, este será el ancho exacto de la vista.

  elasticidad

La elasticidad determina cuánto se redimensiona cada vista para llenar el ancho disponible de la interfaz de usuario del reproductor’s barra de control relativa a todas las demás vistas. Si hay dos vistas de diseño en una barra de control, y una tiene una elasticidad de 1.0 y la otra tiene una elasticidad de 2.0, esta última crecerá el doble cuando se cambie el tamaño de la vista. Cada fila de un diseño de control normalmente debe contener al menos una vista de diseño con una elasticidad mayor que cero. Una vista con una elasticidad de cero permanecerá en su ancho mínimo.

  remoto

los “remoto” La propiedad se puede establecer en SÍ o NO para ocultar o mostrar rápidamente una vista de diseño en particular. Esto es más eficiente que establecer un nuevo BCOVPUIControlLayout con una vista eliminada o agregada. Call -setNeedsLayout en el BCOVPUIBasicControlView después de configurar esta propiedad para mostrar el nuevo diseño.
