Referencia BCOVWidevine

Complemento Widevine para Brightcove Player SDK para iOS, versión 2.1.4.358

Plataformas compatibles

iOS 8.0 y 9.0. Widevine no se ejecutará en el simulador de iOS.

Importante: tenga en cuenta

El complemento Widevine para Brightcove Player SDK para iOS es obsoleto.

Instalación

El complemento Widevine para Brightcove Player SDK proporciona un marco de biblioteca estático para la instalación.

CocoaPods

Puedes usar Cocoapods para agregar el complemento Widevine para Brightcove Player SDK a su proyecto. Puedes encontrar lo último Brightcove-Player-SDK-Widevine podspec aquí. Para usar esta especificación, agregue lo siguiente al principio de Podfile:fuente 'https://github.com/brightcove/BCOVSpecs.git ' o corre pod repo agregar BCOVSpecs https://github.com/brightcove/BCOVSpecs.git. Se requiere CocoaPods 0.39 o más reciente.

El SDK de Widevine no es incluido en esta vaina. usted debe agregue manualmente la biblioteca Widevine a su proyecto incluso cuando utilice Cocoapods. Sin embargo, el pod agregará todas las bibliotecas requeridas por la biblioteca Widevine, como libc ++ , libxml2 , libresolv , libSystem , y libz. Póngase en contacto con el soporte de Brightcove para obtener acceso a la biblioteca de Widevine.

Para instalar manualmente la biblioteca Widevine:

  1. Solicite la biblioteca de Widevine al soporte de Brightcove.
  2. Agregue el contenido del paquete a su proyecto.
  3. Sobre el “Fases de construcción” pestaña del destino de su aplicación, agregue lo siguiente a la “Vincular binarios con bibliotecas” fase:
    • libWViPhoneAPI.a
  4. Sobre el “Configuración de compilación” pestaña del destino de su aplicación:
    • Asegúrese de que los encabezados de Widevine estén en su aplicación’s “Ruta de búsqueda de encabezado”.
    • Asegurarse de que -ObjC ha sido agregado a la “Otras banderas del vinculador” entorno de construcción.

Ejemplo de marco estático:

fuente 'https://github.com/CocoaPods/Specs.git' source' https://github.com/brightcove/BCOVSpecs.git' pod 'Brightcove-player-SDK-WideVine'

Manual

Para agregar el complemento Widevine para Brightcove Player SDK a su proyecto manualmente:

  1. Instale la última versión del SDK de Brightcove Player.
  2. Descargue la última versión comprimida del complemento desde nuestro página de lanzamiento.
  3. Solicite la biblioteca Widevine del soporte de Brightcove y agréguela a su proyecto.
  4. Añade el ‘BrightcoveWidevine.framework’ a su proyecto.
  5. Sobre el “Configuración de compilación” pestaña del destino de su aplicación, asegúrese de que el “Rutas de búsqueda del marco” incluir la ruta al marco. Esto debería haberse hecho automáticamente a menos que el marco esté almacenado en un directorio raíz diferente al de su proyecto.
  6. Sobre el “Fases de construcción” pestaña del destino de su aplicación, agregue lo siguiente a la “Vincular binarios con bibliotecas” fase:
    • BrightcoveWidevine.framework
    • libWViPhoneAPI.a
    • libc ++
    • libxml2
    • libresolv
    • libSystem
    • libz
  7. Sobre el “Configuración de compilación” pestaña del destino de su aplicación:
    • Asegurarse de que -ObjC ha sido agregado a la “Otras banderas del vinculador” entorno de construcción.
    • Asegúrese de que los encabezados de Widevine estén en su aplicación’s “Ruta de búsqueda de encabezado”.

Inicio rápido

 NSString * token; // (token de Brightcove Media API con acceso URL) NSString * playlistID; // (ID de la lista de reproducción que desea usar) BCOVPlayerSDKManager * manager = [BCOVPlayerSDKManager sharedManager]; [1] id <BCOVPlaybackController> controller = [playbackManager createWidevinePlaybackControllerWithViewStrategy: nil] viewStrategy: nil]; [self.view addSubview: controller.view]; BCOVCatalogService * catalog = [[BCOVCatalogService alloc] initWithToken: token]; [2] [catálogo findWidevinePlaylistWithPlaylistID: parámetros de playlistID: finalización nula: ^ (BCOVPlaylist * playlist, NSDictionary * jsonResponse, NSError * error) { [controller setVideos:playlist]; [controller play]; }];

Dejar’s divide este código en pasos, para que sea un poco más fácil de digerir:

  1. BCOVWidevine agrega algunos métodos de categoría a BCOVPlaybackManager. El primero de estos es -createWidevinePlaybackControllerWithViewStrategy. Utilice este método para crear su controlador de reproducción. Alternativamente (si está utilizando más de un proveedor de sesión), puede crear un BCOVWidevineSessionProvider y pasar eso al método de administrador que crea un controlador de reproducción con proveedores de sesión ascendentes. *
  2. BCOVWidevine agrega algunos métodos de categoría a BCOVPlaybackManager para recuperar representaciones de Widevine del catálogo.

* Tenga en cuenta que BCOVWidevineSessionProvider debe aparecer antes que cualquier proveedor de sesiones de anuncios en la cadena que se transfiera al administrador al construir el controlador de reproducción.

Si tiene preguntas o necesita ayuda, tenemos un foro de soporte para Brightcove’s SDK de reproductor nativos en https://groups.google.com/forum/#! forum / brightcove-native-player-sdks.

Personalización del comportamiento del complemento

Puede personalizar el comportamiento del complemento predeterminado creando una instancia de BCOVWidevineSessionProviderOptions y anulando las propiedades predeterminadas. Para usar un BCOVWideSessionProviderOptions instancia de opciones, necesita crear el BCOVWideSessionProvider utilizando [BCOVSDKManager createWidevineSessionProviderWithOptions:].

 BCOVWidevineSessionProviderOptions * opciones = [[BCOVWidevineSessionProviderOptions alloc] init]; options.widevineSettings = @{ WVPlayerDrivenAdaptationKey: @0 } ; id <BCOVPlaybackSessionProvider> sessionProvider = [playbackManager createWidevineSessionProviderWithOptions: opciones]; id <BCOVPlaybackController> playbackController = [playbackManager createPlaybackControllerWithSessionProvider: sessionProvider viewStrategy: nil];

Selección de fuente (WVM, HTTP / HTTPs)

Brightcove Player SDK para iOS ofrece a los clientes la posibilidad de adjuntar varios tipos de URL y entrega (BCOVSource ) a un solo video (BCOVVídeo). Por ejemplo, si el catálogo o el servicio de reproducción están recuperando sus videos, puede haber una combinación de versiones WVM, HLS o MP4 para un solo video, junto con las versiones HTTP y HTTPs. Cuál de estas fuentes que se seleccionan está determinada por un bloque de selección de fuente. La política de selección de fuente predeterminada utilizada por el complemento Widevine selecciona el primer Widevine BCOVSource en cada BCOVVídeo , independientemente del esquema.

La selección de fuente se puede anular creando un BCOVWidevineSessionProviderOptions y usándolo para crear un BCOVWidevineSessionProvider. Por ejemplo:

BCOVPlayerSDKManager * sdkManager = [BCOVPlayerSDKManager sharedManager]; BCOVWidevineSessionProviderOptions * opciones = [[BCOVWidevineSessionProviderOptions alloc] init]; options.sourceSelectionPolicy = <policy> id <BCOVWidevineSessionProvider> proveedor = [sdkManager createWidevineSessionProviderWithOptions: opciones]; id <BCOVPlaybackController> control de reproducción [sdkManager createPlaybackControllerWithSessionProvider: proveedor viewStrategy: nil];

Si esta política de selección predeterminada no le funciona, existen algunas alternativas para seleccionar una fuente:

  • Si recupera videos de Video Cloud a través del servicio de catálogo o el servicio de reproducción, antes de llamar [BCOVPlaybackController setVideos:] , utilice el método de actualización en el BCOVVídeo para contener solo la fuente que desea (consulte el “Valores” sección para más información).

  • Si prefiere fuentes HTTPS, el complemento Widevine ofrece opciones adicionales para seleccionar fuentes de Widevine con un esquema particular. [BCOVWidevineSourceSelectionPolicy sourceSelectionWidevineWithScheme: kBCOVSourceURLSchemeHTTPS] proporciona una política de selección de fuentes que prefiere las fuentes de Widevine mediante HTTPS. Esto no convertirá las URL que no sean HTTP en URL HTTP. Si elige seleccionar HTTP, asegúrese de que su CDN esté configurado para HTTP. Si la CDN está configurada para HTTP, utilice BCOVPlaybackService en lugar de BCOVCatalogService para recuperar metadatos de video / lista de reproducción.

  • De manera similar a la actualización del objeto de video, también puede implementar su propio bloque de selección de fuente.

     options.sourceSelectionPolicy = ^ BCOVSource * (BCOVVideo * video) {<Ver fuente video.sources>
