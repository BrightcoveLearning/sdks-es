BCOVCatalogService (BCOVWidevineImperativeCallbacks) Referencia de categoría

Declarado en BCOVCatalogService + BCOVWidevineAdditions.h

Descripción general

La funcionalidad de los métodos de búsqueda agregados en estos métodos de categoría es similar a los métodos de búsqueda centrales en BCOVCatalogService, excepto que en lugar de devolver una señal que entrega los resultados (o un error), estos métodos devuelven vacío pero toman un bloque que se llama con los resultados (o un error). Si se siente cómodo usando ReactiveCocoa’s señal API, los métodos de señal ofrecidos por BCOVCatalogService ofrecen una mayor flexibilidad. Estos métodos de categoría están destinados a clientes que no desean utilizar señales directamente.

– findWidevinePlaylistWithPlaylistID: parámetros: finalización:

Recupera una lista BCOVPlaylist del servicio Media API por su ID de lista de reproducción.

Parámetros

playlistID

El ID de la lista de reproducción que se va a buscar.
parámetros

Parámetros de consulta NSString adicionales para agregar a las solicitudes de Media API. Estos valores anularán los valores predeterminados si entran en conflicto.
CompletionHandler

bloque que se invocará cuando finalice la solicitud. La ejecución de deploymentHandler ocurrirá en el hilo principal.

Discusión

Recupera una lista BCOVPlaylist del servicio Media API por su ID de lista de reproducción.

Si Media API devuelve una respuesta de error estándar con formato JSON, la respuesta deserializada será la devolución de llamada.’s error’s Información de usuario.

Si se produce un error de análisis JSON, la respuesta NSData sin procesar se incluirá en la devolución de llamada.’s error’s Información de usuario , con clave de kBCOVCatalogJSONDeserializationErrorRawDataKey.
