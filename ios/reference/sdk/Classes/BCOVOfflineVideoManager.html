- (void) requestVideoDownload: (BCOVVideo *)video mediaSelections: (NSArray <AVMediaSelection *> *)mediaSelections parámetros: (NSDictionary *)parámetros finalización: (void (^) (BCOVOfflineVideoToken offlineVideoToken, NSError * error))CompletionHandler

Parámetros

video

BCOVVídeo para ser descargado. El video debe tener una fuente HLS con un esquema HTTPS y puede usar el cifrado FairPlay.
mediaSelections

Una serie de AVMediaSelections para descargar, además del video principal. Las AVMediaSelections disponibles para un BCOVVideo se pueden recuperar llamando a -urlAssetForVideo: error: seguido de [AVAsset favoriteMediaSelection] o [AVAsset allMediaSelections]. Pasar nil para mediaSelections hará que se recuperen las AVMediaSelections preferidas. Tenga en cuenta que con iOS 13+, todos los activos de video deben descargarse mediante una sola llamada para requestVideoDownload: mediaSelections: parameters: complete :. Las pistas adicionales no se pueden descargar más tarde.
parámetros

NSDiccionario de parámetros utilizados en esta solicitud de descarga. Puede ser nulo. Los parámetros válidos son: kBCOVOfflineVideoManagerRequestedBitrateKey, kBCOVFairPlayLicensePurchaseKey, kBCOVFairPlayLicenseRentalDurationKey.
CompletionHandler

Un bloque que recibe el nuevo offlineVideoToken después de que la solicitud se realiza correctamente. Si hay un error, offlineVideoToken será nulo y el error reportará cualquier NSError.

Discusión

Si está utilizando videos FairPlay con Video Cloud Dynamic Delivery, los certificados de la aplicación FairPlay se descargarán automáticamente y se usarán según sea necesario. Si está utilizando videos heredados de Video Cloud FairPlay, debe agregar los certificados de aplicación necesarios al BCOVOfflineVideoManger antes de iniciar una descarga, usando - [addFairPlayApplicationCertificate: identifier:].

– renovarFairPlayLicense: video: parámetros: finalización:

Inicie una solicitud de una nueva licencia de FairPlay.

- (void) renewFairPlayLicense: (BCOVOfflineVideoToken)offlineVideoToken video: (BCOVVideo *)video parámetros: (NSDictionary *)parámetros finalización: (void (^) (BCOVOfflineVideoToken offlineVideoToken, NSError * error))CompletionHandler

Parámetros

offlineVideoToken

Token de video sin conexión que se usa para identificar el video descargado para el cual adquirir una nueva licencia.
video

BCOVVídeo objeto para un objeto de video en línea recuperado a través de la API de reproducción o el BCOVPlaybackService clase. Este video no debe haber sido creado por ‘-videoObjectFromOfflineVideoToken:’.
parámetros

NSDiccionario de parámetros utilizados en esta solicitud de renovación de licencia. Puede ser nulo. Los parámetros válidos son: kBCOVFairPlayLicensePurchaseKey, kBCOVFairPlayLicenseRentalDurationKey.
CompletionHandler

Un bloque que se llama después de que se ha completado la solicitud de renovación de la licencia. Si la solicitud resultó en un error, se devolverá aquí como un NSError. El parámetro offlineVideoToken será el mismo token de vídeo sin conexión que se pasó como el primer parámetro.

Discusión

Puede solicitar una nueva licencia de FairPlay en cualquier momento después de haber adquirido la licencia inicial.

En los parámetros puede especificar una nueva duración de alquiler o puede convertir una licencia de alquiler en una licencia de compra.

No es necesario volver a descargar el video para usar una nueva licencia de FairPlay, pero sí necesita recuperar un nuevo objeto de video de la API de reproducción (ya sea directamente oa través de la BCOVPlaybackService clase). Si el video al que se hace referencia ya no está disponible a través de la API de reproducción, puede usar un video sustancialmente similar, lo que significa que el video debe ser de la misma cuenta y también tener la misma configuración de FairPlay.

No debe especificar una tasa de bits en los parámetros. Para cambiar la tasa de bits, debe eliminar la descarga actual y luego iniciar una descarga completamente nueva con la nueva tasa de bits.

– addFairPlayApplicationCertificate: identificador:

Este método se utiliza para agregar un certificado de aplicación FairPlay al Administrador de videos sin conexión’s lista de certificados de aplicaciones FairPlay disponibles.

- (vacío) addFairPlayApplicationCertificate: (NSData *)applicationCertificateData identificador: (NSString *)identificador

Parámetros

applicationCertificateData

El certificado de la aplicación FairPlay en un objeto NSData. Si se establece en nil, se eliminará cualquier certificado de aplicación existente para el identificador dado.
identificador

Una cadena que se usa para ubicar el certificado de la aplicación.

Esta cadena no puede ser nula.

Para Dynamic Delivery, el identificador debe ser la URL que se utilizó para recuperar este certificado.

Para las cuentas de Video Cloud heredadas, esto debe establecerse en kBCOVDefaultFairPlayApplicationCertificateIdentifier para que se aplique a todas las cuentas.

Si utiliza varias cuentas de Video Cloud heredadas, establezca este parámetro en el ID de cuenta.

Discusión

Los certificados se conservarán durante la vigencia de la solicitud. Para eliminar un certificado, agregue el mismo identificador con un cero que los datos del certificado.

Si está utilizando Dynamic Delivery, los certificados de la aplicación se recuperan automáticamente mediante el complemento FairPlay, por lo que este método no es necesario. Sin embargo, puede utilizar este método para precargar su certificado de aplicación para acelerar la reproducción del primer video encriptado con FairPlay.

Los certificados se almacenan y reutilizan para videos posteriores.
