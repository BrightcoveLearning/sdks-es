Referencia del protocolo BCOVPlaybackController

De acuerdo a NSObject
Declarado en BCOVPlaybackController.h

Descripción general

Protocolo adoptado por objetos que proporcionan funcionalidad de reproducción.

Las implementaciones de este protocolo formal deben admitir sus operaciones de reproducción estándar, pero pueden extender la API para realizar funciones adicionales específicas para su conjunto de características.

  adsDisabled método requerido

Solo SSAI: Si utiliza otros complementos publicitarios, utilice seekWithoutAds: CompletionHandler: en lugar de.

@property (nonatomic, readwrite) BOOL adsDisabled

Discusión

Solo SSAI: Si utiliza otros complementos publicitarios, utilice seekWithoutAds: CompletionHandler: en lugar de.

Desactiva la reproducción de anuncios que, de lo contrario, podría ocurrir después de un avance -seekTo :. El uso previsto es, por ejemplo, reanudar la reproducción de video cuando una aplicación se cierra y se reinicia; la aplicación puede buscar un tiempo ahorrado y los anuncios que se han visto previamente no juego de nuevo. Establezca adsDisabled en YES, luego llame a uno de los métodos -seekToTime: y finalmente establezca adsDisabled en NO en el controlador de finalización -seekToTime:. El valor predeterminado de adsDisabled es NO.

Tenga en cuenta que a menos que el BCOVPlaybackSessionLifecycleEvent Se ha recibido kBCOVPlaybackSessionLifecycleEventReady, nunca se llamará al controlador -seekTo: complete. Por esa razón, se recomienda que llame a -seekTo: desde -playbackController: playbackSession: didReceiveLifecycleEvent delegar método para el evento kBCOVPlaybackSessionLifecycleEventReady.

– addFairPlayApplicationCertificate: identificador: método requerido

Este método se utiliza para agregar un certificado de aplicación FairPlay al controlador de reproducción’s lista de certificados de aplicación FairPLay disponibles. Los certificados se conservarán durante la vida útil del controlador de reproducción.

- (vacío) addFairPlayApplicationCertificate: (NSData *)applicationCertificateData identificador: (NSString *)identificador

Discusión

Este método se utiliza para agregar un certificado de aplicación FairPlay al controlador de reproducción’s lista de certificados de aplicación FairPLay disponibles. Los certificados se conservarán durante la vida útil del controlador de reproducción.

Si está utilizando Dynamic Delivery, los certificados de la aplicación se recuperan automáticamente mediante el complemento FairPlay, por lo que este método no es necesario. Sin embargo, puede utilizar este método para precargar su certificado de aplicación para acelerar la reproducción del primer video encriptado con FairPlay. Los certificados se almacenan y reutilizan para videos posteriores.

@param applicationCertificateData El certificado de la aplicación FairPlay en un objeto NSData. Si se establece en nil, se eliminará cualquier certificado de aplicación existente para el identificador dado. @param identifier Una cadena que se usa para ubicar el certificado de la aplicación. Esta cadena no puede ser nula. - Para Dynamic Delivery, el identificador debe ser la URL que se utilizó para recuperar este certificado. - Para las cuentas de Video Cloud heredadas, esto debe establecerse en kBCOVDefaultFairPlayApplicationCertificateIdentifier para que se aplique a todas las cuentas. - Si utiliza varias cuentas de Video Cloud heredadas, establezca este parámetro en el ID de cuenta.
