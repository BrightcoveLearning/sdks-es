Declarado en BCOVOfflineVideoManagerDelegate.h

Descripción general

Cumpla con este protocolo para recibir información sobre cada descarga de video y pista de video.

– didCreateSharedBackgroundSesssionConfiguration:

Este método se llama después de que se crea el objeto NSURLSessionConfiguration de fondo, y antes’s utilizados para crear el objeto AVAssetDownloadURLSession compartido que’Se usa para descargar videos.

- (void) didCreateSharedBackgroundSesssionConfiguration: (NSURLSessionConfiguration *)backgroundSessionConfiguration

Discusión

Puede utilizar este objeto de configuración para establecer varias opciones especificadas para NSURLSessionConfiguration en NSURLSession.h, como el discrecional bandera, o la HTTPMaximumConnectionsPerHost configuración.

Debería no selecciona el allowCellularAccess propiedad; que se establece en las tareas de descarga individuales.

– didDownloadStaticImagesWithOfflineVideoToken:

Este método se llama cuando las imágenes estáticas (miniatura y póster) asociadas con un video se descargan al almacenamiento.

- (void) didDownloadStaticImagesWithOfflineVideoToken: (BCOVOfflineVideoToken)offlineVideoToken

Parámetros

offlineVideoToken

Token de video sin conexión utilizado para identificar el video descargado.

Discusión

Las URL de archivo de las imágenes descargadas se almacenan como propiedades en un BCOVVídeo creado a partir del token de video sin conexión usando - [videoObjectFromOfflineVideoToken:] Las propiedades se recuperan usando las claves kBCOVOfflineVideoThumbnailFilePath y kBCOVOfflineVideoPosterFilePath.

Este método se llamará incluso si no se pudieron recuperar imágenes, así que asegúrese de verificar el UIImage que crea a partir del póster y las rutas de las miniaturas. Las rutas de archivo estarán presentes, pero es posible que no haya imágenes almacenadas en esas ubicaciones.

– shouldDeleteVideoPackage:

Este método se llama al limpiar descargas varadas.

- (BOOL) shouldDeleteVideoPackage: (NSString *)videoPackagePath

Parámetros

videoPackagePath

Ruta al directorio del paquete de video que está marcado como varado.

Discusión

Las descargas pueden quedar bloqueadas si fuerza el cierre de una aplicación durante una descarga o si reinicia mientras la aplicación está suspendida. No es necesario que implemente este método a menos que esté iniciando sus propias descargas de video fuera del SDK de Brightcove y no’No quiero que se eliminen tus videos. Se llama a este método cuando se marca un paquete de vídeo para su eliminación, lo que significa que no hay ningún registro del mismo en el SDK de Brightcove.

Si el archivo es uno que reconoce, debe devolver NO. De lo contrario, devolver SÍ hará que se elimine el paquete de video. Si no implementa este método, las descargas perdidas se eliminarán de forma predeterminada.

– offlineVideoStorageDidChange

Este método se llama cuando la aplicación está activada y el almacenamiento de video sin conexión cambió mientras la aplicación no se estaba ejecutando o estaba en segundo plano, por ejemplo, cuando el usuario elimina videos descargados a través de la configuración de almacenamiento del dispositivo en la configuración general de la aplicación Configuración.

- (void) offlineVideoStorageDidChange

Discusión

Cuando se le llame, debe actualizar su interfaz de usuario para reflejar los cambios en el almacenamiento fuera de línea.

La detección de cambios en el almacenamiento de video sin conexión ocurre cuando la aplicación se activa.

Este método se llama en el hilo principal.
