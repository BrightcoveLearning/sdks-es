Declarado en BCOVPlaybackController.h

Descripción general

Cumpla con este protocolo para recibir información de reproducción básica para cada sesión.

– playbackController: didAdvanceToPlaybackSession:

Se llama cuando el controlador avanza a una nueva sesión de reproducción, lo que ocurre cuando -advanceToNext se llama. Cuando se configura un delegado en un controlador de reproducción, este método se llama con la sesión de reproducción avanzada más reciente (cuando corresponda).

- (vacío) control de reproducción: (id <BCOVPlaybackController>)controlador didAdvanceToPlaybackSession: (id <BCOVPlaybackSession>)sesión

Parámetros

controlador

El controlador de reproducción que recibe la nueva sesión.
sesión

La sesión de reproducción que se avanzó.

Discusión

Se llama cuando el controlador avanza a una nueva sesión de reproducción, lo que ocurre cuando -advanceToNext se llama. Cuando se configura un delegado en un controlador de reproducción, este método se llama con la sesión de reproducción avanzada más reciente (cuando corresponda).

cuePointInfo

Un diccionario de información sobre el evento del punto de referencia.

Discusión

Llamado cuando una sesión’La cabeza lectora pasa los puntos de referencia registrados con su vídeo. Esto ocurrirá independientemente de si el cabezal de reproducción pasa el tiempo del punto de referencia para el progreso estándar (reproducción) o si busca (hacia adelante o hacia atrás) a través de los medios. Cuando se configura un delegado en un controlador de reproducción, este método solo se llamará para futuros eventos de puntos de referencia (no se informará de los eventos que ya hayan ocurrido).

Si se registran varios puntos de referencia en un tiempo o tiempos que se encuentran entre los “momento anterior” y “tiempo actual” para un evento de punto de referencia, todos los puntos de referencia posteriores al “momento anterior” y antes o en “tiempo actual” se incluirá en la colección de puntos de referencia. Dicho de otra manera, varios puntos de referencia al mismo tiempo se agregan en un solo evento de punto de referencia cuya colección contendrá todos esos puntos de referencia. El escenario más probable en el que esto sucedería es cuando se busca en un rango de tiempo que incluye múltiples puntos de referencia (potencialmente en diferentes momentos). – esto dará como resultado un único evento de punto de referencia cuya hora anterior es el punto en el que comenzó la búsqueda, cuya hora actual es el destino de la búsqueda y cuyos puntos de referencia son todos los puntos de referencia cuyo tiempo se encontraba dentro de ese rango.

El diccionario cuePointInfo contendrá las siguientes claves y valores para cada evento de punto de referencia:

kBCOVPlaybackSessionEventKeyPreviousTime: el intervalo de progreso que precede inmediatamente a los puntos de referencia para los que se recibió este evento. kBCOVPlaybackSessionEventKeyCurrentTime: el intervalo de progreso en o inmediatamente después de los puntos de referencia para los que se recibió este evento. kBCOVPlaybackSessionEventKeyCuePoints: el BCOVCuePointCollection de los puntos de referencia para los que se recibió este evento.

Discusión
