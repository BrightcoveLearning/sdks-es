Descripción general

¿Algo salió mal o sucedió inesperadamente durante el desarrollo con los SDK de Brightcove Native Player? Háganos saber lo que sucedió para que pueda volver a encarrilarse.

Para acelerar el proceso de notificación, haga lo siguiente antes de enviar un error:

  1. Verifique si se trata de un problema conocido
  2. Aislar el problema a los SDK de Brightcove

Una vez que haya determinado que su problema no es un problema conocido y está aislado en los SDK de Brightcove, estará listo para enviar un error. Siga estos pasos:

  1. Proporcionar información específica del caso
  2. Incluya información adicional útil
  3. Envíe su error

Compruebe si hay un problema conocido

Primero, verifique si su problema ha sido reportado en el archivo del foro de desarrolladores. Aquí, también puede registrarse para ser incluido en la lista de correo para anuncios de lanzamiento.

  • Archivo del foro de desarrolladores de SDK nativos

Luego, verifique si su error es un problema conocido que se indica en la documentación del desarrollador:

  • Problemas conocidos de los SDK nativos

Aislar el problema

Intente reproducir el problema en una de las aplicaciones de muestra proporcionadas. Estas aplicaciones proporcionan una implementación básica y pueden ayudarlo a revelar si el problema está en el SDK o en un factor externo.

Pruebe con una de las siguientes aplicaciones de muestra:

  • SDK nativo para aplicaciones de muestra de iOS
  • SDK nativo para aplicaciones de muestra de Android

Proveer información

Para proporcionar información específica del caso, recopile lo siguiente:

  • Comportamiento de error que experimentas / el comportamiento esperado
  • Detalles específicos del dispositivo (marca, versión del modelo)
  • Versión de SDK utilizada
  • Registros de fallas (logcat para Android, registros de fallas de consola / dispositivo)
  • pasos para reproducir

Información Adicional

Incluya los siguientes detalles si puede:

  • Fragmento de código
  • Aplicación de muestra que muestra un error
  • Lo que ha intentado resolver el problema.

Envíe su error

Use lo siguiente para ingresar su error al equipo de soporte:

  • Crea un ticket de soporte general
