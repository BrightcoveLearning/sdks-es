Descripción general

Después de completar este inicio rápido, debería poder:

  • Cree un proyecto y agregue Brightcove Player SDK para Android con Gradle.
  • Modifique el diseño para incluir un BrightcoveExoPlayerVideoView.
  • Agregar al onCreate() método para reproducir videos. de diferentes fuentes.
Notas:
  • Este ejemplo usa el BrightcoveExoPlayerVideoView , que es la vista recomendada para usar en la mayoría de los casos. Si cree que necesita utilizar una vista diferente, consulte la Elegir un BaseVideoView documento.
  • Este ejemplo usa el API de reproducción de Brightcove. Esta es la API más reciente y recomendada para recuperar contenido de su biblioteca de Video Cloud.

Audiencia

Desarrolladores que usan Android Studio para el desarrollo y están interesados en usar Brightcove Player SDK para Android en una aplicación de Android.

Prerrequisitos

Un conocimiento mínimo del desarrollo de aplicaciones Java y Android.

Prepararse

Prepárese para el desarrollo instalando Android Studio

  1. Descargar el Estudio de Android solicitud.
  2. Siga las instrucciones de las páginas para instalar la aplicación.

Crea un proyecto

Cree un proyecto en Android Studio y luego vincúlelo al SDK del reproductor con Gradle.

Crea un proyecto en Android Studio

  1. Abra Android Studio.
    • Si es la primera vez que abre Android Studio, seleccione Iniciar un nuevo proyecto de Android Studio.
    • Si está viendo los menús, seleccione Archivo> Nuevo> Nuevo proyecto.
  2. En el Elige tu proyecto ventana, seleccione Actividad vacía y haga clic en próximo.

  3. Valores de suministro para el Nombre , Nombre del paquete , Idioma , y Guardar dirección. En este inicio rápido se utilizan los valores que se muestran a continuación:

  4. Haz clic en Finalizar.

  5. Android Studio funcionará por un tiempo y eventualmente mostrará el estado inicial del proyecto.

Utilice Gradle para vincularse con Brightcove Player SDK para Android

Dado que Gradle está integrado con Android Studio, lo usará para agregar el SDK nativo para Android a su proyecto.

  1. Ampliar la Secuencias de comandos de Gradle sección y abra la segunda build.gradle archivo, que está asociado con el módulo actual en su proyecto PlayVideos.

  2. En el build.gradle archivo, haga lo siguiente:

    1. En el android sección, agregue lo siguiente compileOptions valor:
      Para ExoPlayer v2.9.0 +, debe habilitar la compatibilidad con Java 8. Para hacer esto, agregue lo siguiente al android sección de tu aplicación build.gradle expediente. Para obtener más detalles, consulte ExoPlayer de Android documento.
      android {
    compileSdkVersion 28
    defaultConfig {
        applicationId "com.brightcove.playvideos1"
        minSdkVersion 21
        targetSdkVersion 28
        versionCode 1
        versionName "1.0"
        testInstrumentationRunner "androidx.test.runner.AndroidJUnitRunner"
    }
    buildTypes {
        release {
            minifyEnabled false
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
    compileOptions {
        targetCompatibility JavaVersion.VERSION_1_8
    }
}
    2. Bajo la android sección, agregue un repositories sección para el repositorio de Brightcove Maven. 
      repositories {
      maven {
          url 'https://repo.brightcove.com/releases'
      }
    }
    3. En el dependencies sección, incluye la última versión del SDK nativo para Android. Reemplazar el valor 6.10.0 a continuación con la última versión del SDK, que se puede encontrar en el Descripción general: SDK nativo de Brightcove para Android documento. 
      dependencies {
  implementation fileTree(dir: 'libs', include: ['*.jar'])
  implementation 'com.android.support:appcompat-v7:26.1.0'
  implementation 'com.android.support.constraint:constraint-layout:1.0.2'
  testImplementation 'junit:junit:4.12'
  androidTestImplementation 'com.android.support.test:runner:1.0.1'
  androidTestImplementation 'com.android.support.test.espresso:espresso-core:3.0.1'
  implementation "com.brightcove.player:exoplayer2:6.10.0"
}
     

    Usando una versión de dependencia dinámica con el '+' El personaje se desaconseja en la comunidad de Gradle. El uso de versiones dinámicas conlleva un riesgo significativo para su proceso de compilación, ya que las API más nuevas pueden volverse inesperadamente incompatibles con el código fuente de su aplicación.

    Notas:
    • Brightcove recomienda utilizar siempre la última versión con una cadena de versión específica. Por ejemplo, revise el build.gradle y gradle.properties archivos en el Android-player-samples proyecto.
    • La dependencia de ExoPlayer incluye el SDK nativo para Android, por lo que no es necesario que incluya el SDK por separado.
    • ExoPlayer de Google es un reproductor multimedia de código abierto a nivel de aplicación para Android. Tenga en cuenta que estamos utilizando el marco de trabajo ExoPlayer 2. Para obtener más detalles, consulte la Migración a ExoPlayer 2 Framework documento.
  3. Sincronice el proyecto para recoger los cambios del archivo Gradle.

Codifique la aplicación

A continuación, escribirá el código para diseñar la aplicación, crear la lista de videos y reproducir los videos.

Definir el diseño de la aplicación

Incluso con la actividad vacía, obtendrá una simple TextView diseño. Reemplace esto con el BrightcoveExoPlayerVideoView , que define la vista del jugador.

  1. Abre el archivo app/res/layout/activity_main.xml.

  2. Hacer clic Texto en la parte inferior de la pestaña del archivo para ver el XML sin formato.

  3. Eliminar el existente TextView elemento y agregue un BrightcoveExoPlayerVideoView sección para que el XML resultante aparezca de la siguiente manera. Nota para un uso posterior del id de la vista se llama brightcove_video_view.

    <?xml version="1.0" encoding="utf-8"?>
    <android.support.constraint.ConstraintLayout xmlns:android="http://schemas.android.com/apk/res/android"
        xmlns:app="http://schemas.android.com/apk/res-auto"
        xmlns:tools="http://schemas.android.com/tools"
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        tools:context="com.brightcove.playvideos.MainActivity">

    <com.brightcove.player.view.BrightcoveExoPlayerVideoView
        android:id="@+id/brightcove_video_view"
        android:layout_width="match_parent"
        android:layout_height="280dp"
        android:layout_gravity="center_horizontal|top"/>
    </android.support.constraint.ConstraintLayout>

Habilite la aplicación para usar Internet

  1. Abre el app/manifests/AndroidManifest.xml expediente.

  2. Justo después del <application> bloque de código, pero por encima del final </manifest> etiqueta, inserte lo siguiente para habilitar el acceso a Internet.

    <uses-permission android:name="android.permission.INTERNET"/>
  3. Sincronice el proyecto para reconstruirlo y recoger estos cambios.

Obtener y reproducir un video

En esta sección, utilizará el Catalog class para recuperar un solo video del servidor de Video Cloud y luego reproducirlo.

La com.brightcove.player.edge.Catalog La clase proporciona métodos asincrónicos para recuperar videos y listas de reproducción de la API de reproducción de Brightcove. Esta es la API más reciente y recomendada para recuperar contenido de su biblioteca de Video Cloud.

  1. Ya no se necesita algo de código de la aplicación anterior. En el onCreate() método, elimine todo el código después de crear una instancia del brightcoveVideoView .

  2. Confirmar tu onCreate() El método aparece de la siguiente manera:

    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

Recuperar un video del Catálogo

  1. Desde Video Cloud Studio, recopile la siguiente información:
    • ID de la cuenta
    • ID de video
    • Clave de política
     
    Notas:
    • Para acceder a la API de reproducción, necesitará una clave de política. Si no está familiarizado con las claves de política, consulte la Descripción general de la API de políticas.
    • Debido a que los SDK nativos no envían un encabezado de origen, no puede usar una clave de política restringida por dominio. Si prueba esto, obtendrá un 403 Forbidden respuesta cuando intenta obtener contenido de su biblioteca de Video Cloud.

  2. Defina sus valores personalizados en su proyecto. Abre el res/values/strings.xml archivo y actualícelo de la siguiente manera:

    <?xml version="1.0" encoding="utf-8"?>
  <resources>

      <!-- Application name -->
      <string name="app_name">PlayVideos</string>

      <!-- A sample Brightcove Edge Account ID -->
      <string name="account">your account id</string>

      <!-- A sample Brightcove Edge Policy Key -->
      <string name="policy">your policy key</string>

      <!-- A sample Brightcove Video ID -->
      <string name="videoId">your video id</string>

  </resources>

  3. Volver a la MainActivity.java archivo y agregue el siguiente código:

    • Obtener el emisor de eventos del SDK

    • Crea una instancia de com.brightcove.player.edge.Catalog , utilizando sus valores definidos para ID de cuenta y Clave de política del paso anterior.

    // Get the event emitter from the SDK and create a catalog request to fetch a video from the
// Brightcove Edge service, given a video id, an account id and a policy key.
EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));

  4. Utilice el catálogo findVideoByID() método con su ID de video y un VideoListener para la devolución de llamada.

    catalog.findVideoByID(getString(R.string.videoId), new VideoListener() {

  // Add the video found to the queue with add().
  // Start playback of the video with start().
  @Override
    public void onVideo(Video video) {
    }

  @Override
    public void onError(String s) {
    }
});
    los VideoListener identifica los objetos que esperan recibir un objeto de vídeo de una operación asincrónica. En una operación exitosa, un onVideo() se llamará al método. Si hay un error en la operación, un onError() se llamará al método. Android Studio insertó las firmas de los dos métodos necesarios.

  5. En el onVideo() método, agregue el video a brightcoveVideoView , luego inicie el video.

    // Start playback of the video with start().
  @Override
    public void onVideo(Video video) {
      brightcoveVideoView.add(video);
      brightcoveVideoView.start();
    }

  6. En el onError() método lanza la cadena de error.

    @Override
    public void onError(String s) {
      throw new RuntimeException(s);
    }

  7. El código completo para su MainActivity la clase debería ser similar a esto:

    public class MainActivity extends BrightcovePlayer {

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

    EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
    Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));
    catalog.findVideoByID(getString(R.string.videoId), new VideoListener() {

      // Add the video found to the queue with add().
      // Start playback of the video with start().
      @Override
      public void onVideo(Video video) {
        brightcoveVideoView.add(video);
        brightcoveVideoView.start();
      }

      @Override
      public void onError(String s) {
        throw new RuntimeException(s);
      }
    });
  }
}
  8. Ejecute la aplicación para confirmar que se reproduce el video.

Obtén y reproduce una lista de reproducción

En esta sección, utilizará el Catalog class para recuperar una lista de reproducción del servidor de Video Cloud y luego reproducir los videos en la lista de reproducción.

La com.brightcove.player.edge.Catalog La clase proporciona métodos asincrónicos para recuperar videos y listas de reproducción de la API de reproducción de Brightcove. Esta es la API más reciente y recomendada para recuperar contenido de su biblioteca de Video Cloud.

Eliminar el código innecesario

  1. Ya no se necesita algo de código de la aplicación anterior. Eliminar la llamada al catálogo findVideoByID() método, y el asociado VideoListener función de devolución de llamada anónima.

        brightcoveVideoView.add(video);
        brightcoveVideoView.start();
      }

    @Override
      public void onError(String s) {
        throw new RuntimeException(s);
      }
  });

Recuperar una lista de reproducción del catálogo

  1. La instancia de catálogo existente funcionará para recuperar una lista de reproducción, por lo que no son necesarios cambios a estas líneas de código:

    EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));
  2. En Video Cloud Studio's Medios de comunicación módulo, seleccione una lista de reproducción y copie el ID de lista de reproducción.

  3. Utilice el catálogo findPlaylistByID() método usando su ID de lista de reproducción y un PlaylistListener para la devolución de llamada.

    catalog.findPlaylistByID("2764931905001", new PlaylistListener() {
  @Override
  public void onPlaylist(Playlist playlist) {
  }

  @Override
  public void onError(String s) {
  }
});
    los PlaylistListener identifica los objetos que esperan recibir un objeto de lista de reproducción de una operación asincrónica. En una operación exitosa, un onPlaylist() se llamará al método. Si hay un error en la operación, un onError() se llamará al método. Android Studio insertó las firmas de los dos métodos necesarios.

  4. En el onPlaylist() método, recupere los videos de la lista de reproducción, agregue todos los videos a brightcoveVideoView , luego inicie el primer video.

    @Override
    public void onPlaylist(Playlist playlist) {
      brightcoveVideoView.addAll(playlist.getVideos());
      brightcoveVideoView.start();
    }

  5. En el onError() método lanza la cadena de error.

     @Override
    public void onError(String s) {
      throw new RuntimeException(s);
    }

  6. El código completo para su MainActivity la clase debería ser similar a esto:

    public class MainActivity extends BrightcovePlayer {

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

    EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();
    Catalog catalog = new Catalog(eventEmitter, getString(R.string.account), getString(R.string.policy));
    catalog.findPlaylistByID("4845949311001", new PlaylistListener() {
      @Override
      public void onPlaylist(Playlist playlist) {
        brightcoveVideoView.addAll(playlist.getVideos());
        brightcoveVideoView.start();
      }

      @Override
      public void onError(String s) {
        throw new RuntimeException(s);
      }
    });
  }
}
