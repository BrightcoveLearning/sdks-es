    Resumen: SDK nativo de Brightcove para Android

    Bienvenido a la documentación para desarrolladores de Android SDK. Aquí encontrará la información necesaria para desarrollar con el SDK de Android.

    Estructura de documentación

    En Brightcove, nos esforzamos por mantener la documentación estructurada, limpia e intuitiva. Como tal, aprovechamos los ReadMes de github para obtener detalles sobre todos nuestros complementos e integraciones. Para su comodidad, hemos enumerado y vinculado a estos complementos a continuación.

    Notas:

    • El SDK nativo de Brightcove para Android ahora admite bibliotecas Android Jetpack (AndroidX) . Para obtener más información, consulte la sección Descargar el SDK

    Versiones admitidas

    El SDK de Native Player para Android ofrece compatibilidad con las siguientes versiones de Android:

    Soporte activo

    Brightcove proporciona asistencia técnica activa sobre el SDK más reciente para las siguientes versiones del sistema operativo Android:

    • 10.0
    • 9.0
    • 8.0, 8.1
    • 7.0, 7.1
    • 6.0

    Soporte pasivo

    Brightcove proporciona soporte pasivo para las siguientes versiones de Android:

    • 5,0, 5.1

    Entender los niveles de soporte

    El soporte se define en términos de la [versión principal].[ versión menor] de un sistema operativo (por ejemplo: Android 8.1). Para aprovechar las últimas mejoras tecnológicas de Google, ofrecemos dos niveles de soporte:

    • Activo: Brightcove probará y corregirá errores en estas versiones del sistema operativo.
    • Pasivo: Brightcove comprobará errores en esas versiones de sistema operativo, pero solo los corregirá caso por caso.

    Por regla general, Brightcove ofrece servicio técnico activo para las versiones del sistema operativo utilizadas por un 80 % de los usuarios, y servicio pasivo para otras versiones que sean utilizadas por al menos un 5 % de la base de usuarios.

    TV conectada/Fixtop boxes

    El SDK de Native Player para Android ofrece compatibilidad con los siguientes decodificadores basados en Android:

    • Android TV
    • Fire TV (dispositivos FireOS 5.0+)

    Los dispositivos decodificadores con los que probamos incluyen:

    • Jugador Nexus
    • Escudo Nvidia

    El SDK nativo de Brightcove para aplicaciones de ejemplo de Android funcionará en Android TV y Fire TV. Esto significa que sus aplicaciones basadas en TV pueden incluir publicidad, protección de contenido (DRM), reproducción en vivo, controles de interfaz de usuario personalizados, métricas de Brightcove Analytics y QoE.

    Echa un vistazo a la aplicación de ejemplo Controles con estilo para obtener más detalles sobre el estilo de los controles de interfaz de usuario en la aplicación de TV.

    Introducción

    Antes de comenzar el desarrollo con el SDK de Android, hay algunos conceptos sobre los que debe aprender:

    1. Media

      La versión actual del SDK de Native Player admite la reproducción de contenido de su cuenta de Video Cloud a través de la API de reproducción o como activos remotos con URL accesibles a Internet.

    2. Analytics

      Con el SDK de Native Player, los clientes de Video Cloud obtienen automáticamente datos analíticos sólidos reportados en Video Cloud.

    3. Gradle

      Gradle es la base del sistema de compilación de Android SDK y está integrado con Android Studio.

    4. Repo Maven

      El reproductor nativo de Brightcove para Android Maven artefactos (archivos jar) ya están disponibles en Brightcove Maven Repo - https://repo.brightcove.com/releases

      Para obtener más información sobre cómo agregar el repositorio Maven a su proyecto de Android, consulte el documento Crear una aplicación usando el SDK nativo para Android .
    5. Descripción general del SDK de Android

      Para ayudarle a comprender el SDK, descubra cómo interactúan los componentes críticos para manejar las complejidades del control del reproductor de vídeo nativo.

    6. Cree una aplicación usando el SDK nativo para Android

      Cree su primera aplicación con el SDK de Brightcove Player para Android, que reproduce un vídeo desde su propio sistema back-end. Los clientes de Video Cloud también recuperarán y reproducirán un vídeo del servicio de catálogo de Brightcove.

    7. Foro de desarrolladores de SDK

      Únase a nosotros en el foro de desarrolladores de SDK, donde los ingenieros de Brightcove y la comunidad responden a las preguntas relacionadas con SDK. Busque preguntas previamente respondidas o publique una nueva. Como grupo público, puede obtener acceso proporcionando su dirección de correo electrónico.

    8. Capacitación en vídeo bajo demanda

      Vea una serie de vídeos de formación que le guiará a través de los pasos de crear una aplicación en Android Studio que reproduzca un vídeo y una lista de reproducción de su biblioteca de Video Cloud.

    Notas

    Las siguientes notas se refieren a la versión más reciente del SDK nativo:

    • A partir de la versión 6.10.0, el SDK nativo de Brightcove para Android utiliza las bibliotecas Jetpack (androidx.*) en lugar de las bibliotecas de soporte originales. Este cambio es necesario para Exoplayer versión 2.10.0 y superior. Para usar Jetpack, las aplicaciones deben cumplir los siguientes requisitos:

      • Migrar a dependencias de Jetpack (AndroidX)
      • Android Gradle Tools 3.2.0
      • Gradle versión 4.6

      Para obtener más información sobre cómo migrar a JetPack, consulte la guía oficial de migración de Android.

    • La versión mínima del SDK de Android compatible con Brightcove Native SDK para Android es ahora 19.
    • Para ExoPlayer v2.9.0+, debe habilitar la compatibilidad con Java 8. Para hacer esto, agregue lo siguiente a la sección del archivo build.gradle de su aplicación.android Para obtener más información, consulte el documento ExoPlayer de Android . 
      android {
    compileOptions {
      targetCompatibility JavaVersion.VERSION_1_8
    }
  }

      Esto anula la versión predeterminada de Java basada en el compileSdkVersion valor del proyecto.

    Se ha eliminado el soporte para ExoPlayer 1 del SDK nativo para Android. Si aún no lo ha hecho, migre al Framework ExoPlayer 2.

    Instalación de Core SDK

    La instalación del SDK de núcleo de Brightcove es muy sencilla. Hay dos maneras de hacer esto:

    • Uso del sistema de compilación gradle
    • Descarga manual del SDK

    Uso del sistema de compilación gradle

    La forma más fácil de agregar el SDK de Brightcove y sus dependencias es usar el sistema de compilación gradle en Android Studio.

    El uso de una versión de dependencia dinámica con el carácter '+' se desaconseja en la comunidad Gradle. El uso de versiones dinámicas conlleva un riesgo significativo para su proceso de compilación, ya que las API más recientes pueden volverse inesperadamente incompatibles con el código fuente de su aplicación.

    Brightcove recomienda utilizar siempre la última versión con una cadena de versión específica. Por ejemplo, revise los archivos build.gradle y gradle.properties en el proyecto android-player-samples .

    Para obtener más información sobre el uso de gradle, consulte la documentación de Add Build Dependencies de Android.

    Descarga manual del SDK

    También puede descargar manualmente el SDK de Brightcove de la siguiente manera:

    1. Simplemente descargue el SDK y descomprimirlo. Encontrará una serie de carpetas.
    2. Dentro de la carpeta sdk , encontrará el SDK principal con el nombre android-sdk-x.x.x.aar , donde x.x.x es el último número de versión del SDK.
    3. Agregue el archivo android-sdk-x.x.aar a su biblioteca, junto con cualquier archivo de complemento que pueda necesitar.

    Anuncios auxiliares

    Los anuncios complementarios y los anuncios clics no son compatibles con Android TV. Debido a que no hay compatibilidad con el navegador web en Android TV, no hay destino para una URL de clic.

    Trabajar con ProGuard

    El uso de ProGuard le permite crear aplicaciones más seguras, lo que dificulta la ingeniería inversa. ProGuard está integrado en el sistema de compilación Android, por lo que puede elegir habilitarlo o construirlo sin él.

    Si recibe un error cuando ProGuard está habilitado en Gradle, intente agregar las siguientes reglas de exclusión:

      -dontwarn com.google.**
  -dontwarn android.media.**
  -keep class android.media.** { *; }
  -keep class com.google.** { *; }
  -keep interface com.google.** { *; }
  -keep class com.google.ads.interactivemedia.** { *; }
  -keep interface com.google.ads.interactivemedia.** { *; }

    Para obtener una lista completa de las reglas de ProGuard utilizadas por Brightcove Native SDK, descargue y descomprima el SDK nativo actual para Android. Abra la carpeta sdk y vea el proguard.txt archivo.

    Comportamiento de la reproducción

    Es posible que observe algunos de los siguientes comportamientos de reproducción en ciertos dispositivos Android:

    Pausa de vídeo

    La función de pausa inteligente de Android en los dispositivos Samsung pausará un vídeo cuando el usuario mira lejos de la pantalla del dispositivo. Puede deshabilitar esta función si no desea que los vídeos se paren en dispositivos habilitados para Smart Pause.
