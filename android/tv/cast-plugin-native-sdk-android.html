    Complemento de lanzamiento para el SDK nativo para Android

    En este tema, aprenderá a transmitir sus vídeos a un televisor conectado a Chromecast, utilizando el complemento Cast para el SDK nativo de Brightcove para Android.

    Resumen

    Con la tecnología Google Cast, puedes iniciar y controlar el contenido de vídeo retransmitido desde tus dispositivos móviles hasta televisores de alta definición y sistemas de audio doméstico. Desde la aplicación, pulsa el botón Reenviar para transmitir tu contenido en una pantalla grande.

    Para crear una aplicación Cast, necesita los siguientes componentes:

    • Una aplicación de remitente : reside en su dispositivo móvil y detecta dispositivos receptores, establece una conexión segura y refleja su contenido. La aplicación remitente es el reproductor local que se utiliza para enviar contenido a la aplicación del receptor de tu dispositivo Chromecast. Después de enviar a la aplicación del receptor, puedes considerarlo como un mando a distancia para el Chromecast.

      La aplicación remitente es proporcionada por el complemento Cast para Brightcove Native SDK para Android. Aprenderás al respecto en este tema.

    • Una aplicación receptora : esta aplicación se ejecuta en el dispositivo Chromecast. Se puede pensar como una aplicación HTML de una sola página con activos CSS y JavaScript.

      Para realizar pruebas, siga estos pasos:

      1. Comience con el receptor de emisión de muestras de Google
      2. Consulta la documentación de Google Cast Apps

      Para su uso en producción, este documento ilustrará cómo utilizar Brightcove Receiver v2.0.0.

    Notas:

    1. Google no admite el uso de su aplicación de receptor de demostración en un entorno de producción.
    2. El complemento Cast del SDK nativo de Brightcove para Android no proporciona funcionalidad de aplicación de receptor.

    Versión del SDK compatible

    Para utilizar el complemento Cast con el nuevo Receiver v2.0 de Brightcove, debe utilizar el SDK nativo de Brightcove para Android versión 6.16.0 y superior.

    Comprender el plugin Cast

    El plugin Cast está construido sobre la nueva Extensión de Cast ExoPlayer y el Marco de Cast de Google Play Services. Después de agregar la dependencia del complemento de cast, gradle extraerá la dependencia Play Services Cast Framework, ExoPlayer Cast Extension junto con otras dependencias necesarias.

    El complemento Cast se ha rediseñado para minimizar su esfuerzo al integrarse con Video Cloud. Cuando se utiliza Video Cloud, la BrightcoveCastMediaManager clase recopila información de la respuesta de Video Cloud, como Brightcove Video y Source objetos, cada vez que se realiza el EventType.SET_SOURCE evento se emite. Esta información se almacena en caché y está lista para usar cuando el usuario selecciona reproducir para poner en cola el vídeo.

    Integración del plugin Cast

    Hay dos maneras de integrar su aplicación con el complemento Native SDK para Android Cast. Puede integrarse con el Receiver Cast Receiver v2.0 de Brightcove o con Google Cast Demo Receiver.

    Para cualquiera de las integraciones, debe agregar esta dependencia a su proyecto de aplicación:

    implementation "com.brightcove.player:android-cast-plugin:6.16.0"

    Uso del Receiver Cast Receiver v2.0 de Brightcove

    Esta integración está pensada para aquellos clientes de Brightcove que utilizan las API de Brightcove para entregar su contenido.

    Para obtener un ejemplo de código completo, consulte BasicCastBrightCovereCeiverSampleApp.

    Uso del Receiver Demo de Google Cast

    Esta integración está pensada para aquellos clientes de Brightcove que son nuevos en el casting.

    Para obtener un ejemplo de código completo, consulte BasicCastGoogleEeiverSampleApp.

    La aplicación de ejemplo BasicCastGoogleReceeiverSampleApp se conocía anteriormente como BasicCastSampleApp

    Especificación de su propio ID de aplicación de Cast Receiver

    BasicCastGoogleReceeiversampleApp establece un ID de aplicación de Google Demo Receiver, que puede ser útil para comenzar y probar, pero no para la producción.

    Para anular este valor con su aplicación Cast Receiver, defina el siguiente valor de cadena en el archivo strings.xml de la aplicación de ejemplo:

    <string name="cast_receiver_app_id">4F8B3483</string>

    El componente GoogleCastComponent de Brightcove

    La GoogleCastComponent clase es la clase principal del plugin de Brightcove Cast. Instancia el ExoPlayer CastPlayer y establece sus oyentes. Expone algunos métodos esenciales para cargar un vídeo o para añadirlo a la cola. La clase también agrega varios detectores de eventos de Brightcove para gestionar eventos de ciclo de vida de actividad y fragmentos, así como otros detectores de eventos que puede utilizar para emitir información multimedia para cargar un vídeo en el dispositivo Chromecast.GoogleCastComponent

    GoogleCastComponent Ahora utiliza un patrón Builder. En el SDK nativo para Android versiones anteriores a v6.16.0, necesitaba crear una instancia del componente y pasar el Context y EventEmitter al GoogleCastComponent constructor. A continuación, establecería las opciones del componente en una serie de llamadas a métodos independientes.

    A partir del SDK nativo para Android v6.16.0, utilice el patrón Builder para crear una instancia de GoogleCastComponent y establecer sus opciones, todo dentro de una sola cadena de llamadas al método Builder.

    CustomData

    Al igual que con el GoogleCastComponent, la CustomData clase utiliza un patrón Builder para crear una instancia del objeto y agregarle propiedades. Aunque el receptor de Brightcove puede utilizar CustomData para recuperar vídeos del catálogo de Brightcove, no es necesario enviar un CustomData objeto completo, por ejemplo, para emitir un recurso remoto. También es importante tener en cuenta que cuando se usa Google Demo Receiver, el uso de no CustomData es compatible. A los efectos de esta discusión, nos centraremos en CustomData enviar al receptor utilizado para recuperar los datos de vídeo del catálogo de Brightcove.

    ¿Qué es CustomData?

    CustomData es un objeto JSON contenido en el MediaInfo objeto. Su uso previsto es con la aplicación Receiver Cast Receiver v2.0 de Brightcove.

    Datos personalizados con el receptor de Brightcove y los datos del catálogo

    Al integrarse con el receptor de Brightcove, el objeto CustomData JSON adoptará la siguiente forma:

    "customData": {
	"accountId": "1234567890",
	"analyticsParams": {
		"application": "com.brightcove.player.test",
		"user": "abcde1c44b951234"
	},
	"catalogParams": {
		"adConfigId": null,
		"type": "video",
		"bcovAuthToken": null,
		"id": "2345678901",
		"policyKey": "BCpkPolicyKeyObject"
	}
}

    El ejemplo de CustomData objeto anterior contiene todos los elementos de datos necesarios para emitir un vídeo desde el receptor de Brightcove. Estos datos son los mismos independientemente del cifrado, es decir, no hay estructura adicional necesaria para la URL de licencia en el caso de videos DRM.

    También puede encontrar un ejemplo del CustomData objeto en BrightCoveCastBrightCovereCeiversampleApp.

    Datos personalizados con Google Demo Receiver

    Como se indicó anteriormente, no CustomData es compatible con Google Demo Receiver.

    OpcionesProveedor

    A continuación, debe especificar la implementación de OptionsProvider para el marco de Google Cast. La interfaz ayuda a configurar varias opciones necesarias para inicializar la clase CastContext.OptionsProvider Aquí es donde establecerá el ID de la aplicación Cast Receiver. Para obtener más información sobre cómo integrar OptionsProvider, consulte el documento Inicializar el contexto de emisión de Google.

    El complemento Brightcove Cast incluye una DefaultOptionsProvider clase, donde el ID de la aplicación Cast Receiver se establece a través de un conjunto de claves de cadena en el archivo de recursos strings.xml . Para obtener más información y obtener más información sobre cómo sobrescribirlo en la aplicación, consulta la sección Using Your Own Cast Receiver App ID de la aplicación anterior.

    Ya sea que utilice la clase o su propia implementación, debe establecer el nombre de la clase de implementación en el archivo AndroidManifest.xml de su aplicación como metadatos de par clave-valor, como se muestra aquí:DefaultOptionsProviderOptionsProviderOptionsProvider

    <meta-data android:name="com.google.android.gms.cast.framework.OPTIONS_PROVIDER_CLASS_NAME" android:value="com.brightcove.cast.DefaultOptionsProvider" />

    Si utiliza la DefaultOptionsProvider clase, puede configurar ExpandedControllerActivity para activar/desactivar la notificación de lanzamiento estableciendo información de metadatos similar en su AndroidManifest.

    Actividad expandida del controlador

    La ExpandedControllerActivity viene con la biblioteca de Google Cast y te permite controlar el vídeo que se está emitiendo en tu dispositivo Chromecast. Esta clase proporciona cierta flexibilidad de personalización. Por ejemplo, hay cinco ranuras disponibles para mostrar botones, donde la tercera ranura está reservada para el botón Reproducir no configurable. El resto de los botones se pueden configurar como otros botones predefinidos o como botones personalizados.

    El complemento Brightcove Cast proporciona la subclase.DefaultExpandedControllerActivity Hemos habilitado los siguientes botones en el siguiente orden:

    • Subtítulos opcionales
    • Omitir anterior
    • Jugar
    • Saltar Siguiente
    • Alternar silenciar

    Además, la barra de búsqueda establece el mismo dibujable predeterminado que el utilizado en el estándar BrightcoveMediaController:

    • El progreso dibujable: 
      R.drawable.default_scrubber_progress_horizontal
    • El pulgar se puede dibujar: 
      R.drawable.default_scrubber_thumb

    Para obtener información sobre cómo personalizar la barra de búsqueda, consulte SeekBarColorsSampleApp.

    Para establecer el DefaultExpandedControllerActivity o el suyo propio ExpandedControllerActivity , establezca los siguientes metadatos en el archivo AndroidManifest.xml :

    <meta-data  android:name="com.brightcove.cast.DefaultOptionsProvider.EXPANDED_CONTROLLER_ACTIVITY_CLASS_NAME"
android:value="com.brightcove.cast.DefaultExpandedControllerActivity" />

    Notificación de lanzamiento

    Cuando la notificación de envío está activada, la notificación aparecerá cuando emita un vídeo y coloque la aplicación en segundo plano; por ejemplo, después de presionar la tecla de inicio.

    <Para habilitar Notificación de lanzamiento, establezca los siguientes metadatos en el archivo AndroidManifest.xml y proporcione el nombre de la actividad que se iniciará cuando se haga clic en la notificación:

    <meta-data android:name="com.brightcove.cast.DefaultOptionsProvider.NOTIFICATION_TARGET_ACTIVITY_CLASS_NAME"
android:value="com.brightcove.cast.BrightcoveControllerActivity" />

    Si no proporciona com.brightCove.cast.defaulToptionsProvider.Notification_Target_Activity_Class_Name o si el valor tiene un nombre de actividad no válido, se desactivará la notificación de conversión.

    Si bien quitar la etiqueta del archivo AndroidManifest.xml deshabilitará las notificaciones de la aplicación remitente, seguirá siendo posible ver una notificación en el dispositivo remitente que indique que un dispositivo está emitiendo en su red Wi-Fi.meta-data Esta no es una notificación de la aplicación remitente, sino una notificación del dispositivo remitente, independiente de cualquier configuración de la aplicación.

    Botón de lanzamiento

    El botón de conversión te permite seleccionar un dispositivo Chromecast en la misma red que tu dispositivo y te permite conectar y crear una sesión. Para añadir el botón Cast a tu aplicación, sigue Integrate Cast de Google: Agregue un documento de botón de conversión .

    El plugin de Brightcove Cast proporciona un método de utilidad para configurar fácilmente el botón Cast. Esto es útil cuando solo desea agregar el botón Cast al menú Actividad/Fragmento. Consulte el siguiente código para obtener más detalles:

    //Activity
@Override
public boolean onCreateOptionsMenu(Menu menu) {
   super.onCreateOptionsMenu(menu);
   GoogleCastComponent.setUpMediaRouteButton(MainActivity.this, menu);
   return true;
}

    El Mini Controlador

    El Mini Controlador es un Fragmento que se adjunta a su Actividad, generalmente ubicado en la parte inferior del diseño. El Mini Controller te permite reproducir y pausar el vídeo e indica cuándo se está reproduciendo un vídeo en tu dispositivo Chromecast. Cuando se hace clic en el Mini Controller, se iniciará el controlador expandido.

    Para habilitar el Mini Controller, agregue el siguiente código al diseño de su Actividad.

    <fragment
   android:id="@+id/castMiniController"
   android:layout_width="match_parent"
   android:layout_height="wrap_content"
   android:layout_alignParentBottom="true"
   android:visibility="gone"
   class="com.google.android.gms.cast.framework.media.widget.MiniControllerFragment" />

    Para conocer las prácticas recomendadas con el Mini Controlador, consulte la Lista de comprobación de diseño: Documento del mini controlador del remitente. Para obtener detalles sobre la implementación, consulte Integrate Cast: Agregar documento Mini Controller .

    Temas avanzados

    Escuchar la sesión de Cast

    Si desea que su aplicación reaccione cuando se inicie o finalice la conexión de Cast, puede agregar los detectores de eventos o como se muestra aquí:GoogleCastEventType.CAST_SESSION_STARTEDGoogleCastEventType.CAST_SESSION_ENDED

    eventEmitter.on(GoogleCastEventType.CAST_SESSION_STARTED, new EventListener() {
   @Override
   public void processEvent(Event event) {
       // Session Started
   }
});
eventEmitter.on(GoogleCastEventType.CAST_SESSION_ENDED, new EventListener() {
   @Override
   public void processEvent(Event event) {
       // Session Ended
   }
});

    Alternativamente, puede llamar GoogleCastComponent.isSessionAvailable() para comprobar si hay una sesión disponible.

    Lanzamiento de un vídeo

    Para enviar un vídeo a Chromecast después de una conexión correcta, puede utilizar el EventEmitter y emitir la información de Media Info o puede utilizar directamente los GoogleCastComponent métodos.

    Si prefiere emitir la información multimedia, puede utilizar los siguientes eventos:

    • GoogleCastEventType.LOAD_MEDIA_INFO
    • GoogleCastEventType.LOAD_MEDIA_QUEUE_ITEM
    • GoogleCastEventType.ADD_MEDIA_INFO
    • GoogleCastEventType.ADD_MEDIA_QUEUE_ITEM

    En la siguiente tabla se muestran las propiedades esperadas para cada evento:

    Nombre del evento Propiedades
    Clave Tipo de clase de valor
    LOAD_MEDIA_INFO GoogleCastComponent.CAST_MEDIA_INFO
    GoogleCastComponent.CAST_MEDIA_PLAY_POSITION    		 MediaInfo Entero
    LOAD_MEDIA_QUEUE_ITEM GoogleCastComponent.CAST_MEDIA_QUEUE_ITEM MediaQueueItem
    ADD_MEDIA_INFO GoogleCastComponent.CAST_MEDIA_INFO
    GoogleCastComponent.CAST_MEDIA_PLAY_POSITION    		 MediaInfo Entero
    ADD_MEDIA_QUEUE_ITEM GoogleCastComponent.CAST_MEDIA_QUEUE_ITEM MediaQueueItem

    Alternativamente, puede utilizar los siguientes GoogleCastComponent métodos:

    • public void loadMediaInfo(MediaInfo mediaInfo, long positionMs)
    • public void loadMediaInfo(MediaInfo mediaInfo)
    • public PendingResult<RemoteMediaClient.MediaChannelResult> loadItem(MediaQueueItem mediaQueue, int playheadPosition)
    • public PendingResult<RemoteMediaClient.MediaChannelResult> addItems(MediaQueueItem... mediaQueue )

    Cambiar los datos predeterminados de MediaInfo

    De forma predeterminada, el complemento Cast recopila información sobre el objeto Video y Source actual emitido por EventType.SET_SOURCE event. Si desea cambiar o agregar información adicional, como agregar un objeto JSON personalizado a su que el receptor de la aplicación entiende, puede hacerlo sobrescribiendo los métodos y de.MediaInfoloadMediaInfo()addMediaInfo()BrightcoveCastMediaManager Luego, su subclase se pasa como un parámetro constructor a la clase.BrightcoveCastMediaManagerGoogleCastComponent

    Dentro de estos métodos, puede crear sus MediaInfo objetos y emitir los eventos apropiados como se muestra anteriormente. Asegúrese de comprobar com.brightcove.cast.util.CastMediaUtil, ya que proporciona algunos métodos de utilidad para crear una a MediaInfo partir de los objetos de vídeo y fuente.

      Una vez que el se ha recreado con el diseño proporcionado en el, obtendrá acceso a la vista.BrightcoveMediaControllerMediaControllerConfigBrightcoveControlBar Desde aquí, puede acceder a sus vistas de interfaz de usuario, como Botones, donde puede OnClickListeneragregarlas.

    Establecer el MediaControllerConfig

    El es una clase de configuración que puede utilizar para cambiar el diseño de control predeterminado de la clase.MediaControllerConfigBrightcoveMediaController En esta clase, puede establecer el diseño y el OnTouchListener. Una vez creado y configurado, puede emitir este objeto como se muestra a continuación:

    Map<String, Object> properties = new HashMap<>();
properties.put(Event.MEDIA_CONTROLLER_CONFIG, myMediaControllerConfig);
eventEmitter.emit(EventType.SET_MEDIA_CONTROLLER_CONFIG,properties);

    El objeto predeterminado establece el como la presentación con un solo botón Reproducir.MediaControllerConfigR.layout.cast_media_controller Cuando se hace clic, se abrirá un diálogo con dos opciones:

    Problemas conocidos

    Android 9

    Cuando uses Chromecast con Android 9 y superior, debes incluir un FOREGROUND_SERVICE permiso. Esto permite a la aplicación usar notificaciones para controlar una sesión de conversión cuando la aplicación está en segundo plano y se vuelve a primer plano.

    La etiqueta debe agregarse al archivo AndroidManifest.xml de la aplicación, de la siguiente manera:uses-permission

    <uses-permission android:name="android.permission.FOREGROUND_SERVICE"/>

    Servicios de Google Play

    La conexión de transmisión puede no crearse si la versión de los Servicios de Google Play del remitente no está actualizada. Cuando los Servicios de Google Play del remitente, en particular el marco de Servicios de Cast, están desactualizados, es posible que no tengas éxito al crear una conexión de Cast. Esto se soluciona actualizando los Servicios de Google Play del remitente a través de Google Play Store.
