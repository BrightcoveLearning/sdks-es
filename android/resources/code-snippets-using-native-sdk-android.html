    Personalización de controles para Android TV

    Puede agregar botones a la barra de control para dispositivos o Android TV. Los pasos a continuación le guian a través de la adición de un botón de pantalla completa a la barra de control para Android TV:

    1. Abra la aplicación de ejemplo Controles personalizados . Usaremos esto como punto de partida. Revise el archivo README de la aplicación de ejemplo para obtener más información sobre cómo personalizar los controles del reproductor.
    2. Abra el archivo res/layout/land/my_media_controller.xml en Android Studio.
    3. Copie el Button elemento del full_screen botón. El código debería ser similar a esto: 
      <Button
style="@style/BorderlessButton"
android:id="@id/full_screen"
android:padding="4dp"
android:layout_gravity="center_vertical"
android:layout_height="wrap_content"
android:layout_width="wrap_content"
android:visibility="visible"
android:text="@string/brightcove_controls_enter_full_screen"/>
    4. Abra el archivo res/layout/my_tv_media_controller.xml en Android Studio.
    5. Pegue el Button elemento copiado en este archivo de diseño. La posición de los elementos determina su posición en la barra de control.
    6. Desplácese hasta la parte superior del archivo res/layout/my_tv_media_controller.xml . Asegúrese de que la propiedad style está establecida en @style/BrightcoveControlBar.TV.Custom. El código debería verse así: 
      <?xml version="1.0" encoding="utf-8"?>
<com.brightcove.player.mediacontroller.BrightcoveControlBar
xmlns:android="http://schemas.android.com/apk/res/android"
xmlns:bmc="http://schemas.android.com/apk/res-auto"
xmlns:tools="http://schemas.android.com/tools"
tools:ignore="Overdraw, InconsistentLayout"
android:id="@+id/brightcove_control_bar"
android:background="@color/bmc_background"
android:layout_width="match_parent"
android:layout_height="wrap_content"
android:padding="8dp"

android:orientation="horizontal"
android:layout_gravity="bottom"
    style="@style/BrightcoveControlBar.TV.Custom">
    7. Abra el archivo res/values/styles.xml en Android Studio.
    8. Para el style elemento con un name valor de BrightcoveControlBar.TV.Custom , agregue un item elemento para el botón de pantalla completa y configurarlo en true para mostrarlo. 
      <style name="BrightcoveControlBar.TV.Custom">
<item name="brightcove_vr_mode">false</item>
  <item name="brightcove_full_screen">true</item>
</style>
    9. Eso es todo. Ejecute la aplicación de ejemplo en un dispositivo físico o virtual Android TV. Debería ver el botón de pantalla completa en la barra de control.

    Modo de pantalla completa

    Puede administrar el modo de pantalla completa usando código.

    • Aquí está el código para establecer el modo de pantalla completa:

      brightcoveVideoView.getEventEmitter().emit(EventType.ENTER_FULL_SCREEN);

    • Aquí está el código para salir del modo de pantalla completa:

      brightcoveVideoView.getEventEmitter().emit(EventType.EXIT_FULL_SCREEN);

    Obtención de duración para las transmisiones en vivo

    Para obtener la duración de una transmisión en vivo, puede usar el MAX_POSITION. Esto le da la posición más grande que se puede buscar para una transmisión en vivo (es decir, cuanto más lejos se puede arrastrar la barra de desplazamiento). Su código debería verse similar a esto:

    brightcoveVideoView.getEventEmitter().on(EventType.PROGRESS, new EventListener() {
  @Override
  public void processEvent(Event event) {
  int duration = (int) event.properties.get(Event.MAX_POSITION);
  }
});

    Obtener la instancia de ExoPlayer

    Puede haber ocasiones en las que necesite obtener el objeto ExoPlayer para modificar propiedades que no están expuestas a través del SDK nativo de Brightcove. Una instancia de este objeto se crea después de establecer el vídeo. Por lo tanto, debes esperar al DID_SET_VIDEO evento.

    Puede obtener la instancia de ExoPlayer haciendo lo siguiente:

    // Get the ExoPlayer instance
eventEmitter.on(EventType.DID_SET_VIDEO, new EventListener() {
  @Override
  public void processEvent(Event event) {
    VideoDisplayComponent videoDisplayComponent = brightcoveVideoView.getVideoDisplay();
    if (videoDisplayComponent instanceof ExoPlayerVideoDisplayComponent) {
      // Get ExoPlayer
      ExoPlayer exoPlayer = ((ExoPlayerVideoDisplayComponent) videoDisplayComponent).getExoPlayer();
      if (exoPlayer instanceof SimpleExoPlayer) {
        // Get SimpleExoPlayer
        SimpleExoPlayer simpleExoPlayer = (SimpleExoPlayer) exoPlayer;
        Log.v(TAG, "onDID_SET_VIDEO: ExoPlayer = " + simpleExoPlayer);
      }
    }
  }
});

    Para obtener un ejemplo de modificación de la AudioAttribute propiedad para ExoPlayer, consulte la publicación del foro Configuración de AudioAttributes en Exoplayer Instance .

    Obtención de URL para imágenes en miniatura

    Es posible que desee utilizar la miniatura o las imágenes fijas de vídeo de su biblioteca de Brightcove. Utilice el siguiente código para obtener la URL de sus imágenes:

    Para los clientes de Video Cloud que utilizan el catálogo perimetral (com.brightcove.player.edge.Catalog), puede obtener la URL de miniatura de la siguiente manera:

    String thumbnailUrl = video.getProperties().get("thumbnail").toString();

    Si encuentra que la imagen en miniatura es demasiado pequeña, puede obtener la imagen fija del vídeo en su lugar:

    String videoStillUrl = video.getProperties().get("stillImageUri").toString();

    Google Analytics

    Si utiliza el reproductor de Brightcove y la clase de catálogo, los análisis de vídeo se recopilarán automáticamente y aparecerán en el módulo Analytics de Video Cloud. Para obtener métricas adicionales, puedes agregar Google Analytics a tu aplicación.

    Para integrar Google Analytics con tu aplicación, sigue estos pasos:

    1. Añade el complemento Servicios de Google a tu proyecto.
    2. Obtén un archivo de configuración de Google y añádelo a tu proyecto.
    3. Amplíe la aplicación y proporcione un método de ayuda que devuelva el rastreador de aplicaciones. Debería ser similar a la aplicación AnalyticsApplication de google-services.
    4. En una clase separada que amplía BrightCovePlayer, obtenga la instancia de rastreador compartida: 
      // Obtain the shared Tracker instance
AnalyticsApplication application = (AnalyticsApplication) getApplication();
tracker = application.getDefaultTracker();
    5. Sustituir los métodos apropiados para registrar los cambios de pantalla y/o enviar eventos personalizados para el seguimiento.

    Para conocer los pasos detallados, consulta el documento de Google para Agregar Analytics a tu aplicación de Android.

    Cargar un vídeo remoto y una imagen de póster

    Brightcove Player sólo los clientes utilizan activos de vídeo de un servidor remoto. Siga estos pasos para cargar un vídeo y una imagen de póster, ambos residiendo en un servidor remoto:

    1. En el onCreate() método, después de definir la vista de vídeo, cree un objeto de vídeo a partir del vídeo alojado en un servidor remoto. Establezca el para que coincida con el tipo de vídeo que tiene.DeliveryType

      Video video = Video.createVideo("https://learning-services-media.brightcove.com/videos/hls/greatblueheron/greatblueheron.m3u8",
  DeliveryType.HLS);

    2. Cargue una imagen remota para utilizarla como imagen de póster antes de que comience la reproducción de vídeo.

      try {
  java.net.URI myposterImage = new java.net.URI("https://solutions.brightcove.com/bcls/assets/images/Great-Blue-Heron.png");
  video.getProperties().put(Video.Fields.STILL_IMAGE_URI, myposterImage);
} catch (URISyntaxException e) {
  e.printStackTrace();
}

    3. Añada el vídeo a la vista e inicie la reproducción de vídeo.

      brightcoveVideoView.add(video);
brightcoveVideoView.start();

    4. Asegúrate de que tu onCreate() método aparezca de la siguiente manera:

      @Override
  protected void onCreate(Bundle savedInstanceState) {
    super.onCreate(savedInstanceState);
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

    // Optional: For Brightcove Player customers to register their apps
    Analytics analytics = brightcoveVideoView.getAnalytics();
    analytics.setAccount("your account Id");

    // Define a video from a remote server
    Video video = Video.createVideo("https://learning-services-media.brightcove.com/videos/hls/greatblueheron/greatblueheron.m3u8", DeliveryType.HLS);

    // Load a remote poster image
    try {
        java.net.URI myposterImage = new java.net.URI("https://solutions.brightcove.com/bcls/assets/images/Great-Blue-Heron.png");
        video.getProperties().put(Video.Fields.STILL_IMAGE_URI, myposterImage);
    } catch (URISyntaxException e) {
        e.printStackTrace();
    }

    // Add video to the view
    brightcoveVideoView.add(video);
    // Start video playback
    brightcoveVideoView.start();
  }

    Agregar contenido DRM manualmente

    Como cliente de Brightcove Player, es posible que desee utilizar contenido protegido DRM desde su propio servidor. Puede cargar manualmente el contenido DRM de la siguiente manera:

    import com.brightcove.player.display.WidevineMediaDrmCallback;

Video video = Video.createVideo("https://storage.googleapis.com/wvmedia/cenc/h264/tears/tears.mpd");
video.getProperties().put(WidevineMediaDrmCallback.DEFAULT_URL, "https://proxy.uat.widevine.com/proxy?video_id=&provider=widevine_test");
brightcoveVideoView.add(video);
brightcoveVideoView.start();
    No es necesario cargar manualmente contenido DRM en la biblioteca de Brightcove.

    Métodos: sincrónico o asíncrono?

    Estos son algunos de los métodos asincrónicos que se encuentran en el SDK de Native Player para Android:

    • los start() , seekTo() y stopPlayback() Los métodos son asincrónicos, porque emiten un evento para que otros componentes del sistema lo manejen.
    • El clear() método es sincrónico con respecto a la actualización de la lista, pero es asincrónico con respecto a la descarga del video actual.

    Silenciar/dessilenciar el reproductor

    Puede silenciar o anular el silencio mediante programación del reproductor. Brightcove recomienda utilizar BrightCoveExoPlayerVideoView siempre que sea posible.

    Debe esperar a que el evento DID_SET_VIDEO o el DID_PLAY evento antes de ejecutar el código para silenciar el reproductor. De lo contrario, el silenciamiento no surtirá efecto. La razón para esperar estos eventos es que el reproductor se crea cuando se carga el vídeo.

    BrightCoveExoPlayerVideoView

    Cuando utilice BrightCoveExoPlayerVideoView, puede silenciar o anular el silencio mediante programación del reproductor haciendo lo siguiente:

    float volume = 100F;
...
brightcoveVideoView.getEventEmitter().on(EventType.DID_SET_VIDEO, new EventListener() {
  @Override
  public void processEvent(Event event) {
    setMute(true);
    brightcoveVideoView.start();
  }
});
...
private void setMute(boolean mute) {
  volume = mute ? 0F : 100F;
  Map<string, Object> properties = new HashMap<>();
  properties.put(Event.VOLUME, volume);
  brightcoveVideoView.getEventEmitter().emit(EventType.SET_VOLUME, properties);
}

    BrightCoveVideoView

    Brightcove recomienda utilizar BrightCoveExoPlayerVideoView.

    Si necesita usar BrightCoveVideoView, que utiliza MediaPlayer de Android, puede silenciar o anular el silencio mediante programación del reproductor con el siguiente código:

    float volume = 1F;
...
brightcoveVideoView.getEventEmitter().on(EventType.DID_SET_VIDEO, new EventListener() {
  @Override
  public void processEvent(Event event) {
    setMute(true);
    brightcoveVideoView.start();
  }
});
...
private void setMute(boolean mute) {
  volume = mute ? 0F : 1F;
  Map<String, Object> properties = new HashMap<>();
  properties.put(Event.LEFT_VOLUME, volume);
  properties.put(Event.RIGHT_VOLUME, volume);
  brightcoveVideoView.getEventEmitter().emit(EventType.SET_VOLUME, properties);
}

    Paginación con la API de reproducción

    Al recuperar el contenido de Video Cloud desde la API de reproducción, puede implementar la paginación de una lista de reproducción.

    Para realizar una página a través de un conjunto de vídeos de una lista de reproducción, utilice los siguientes parámetros de URL de solicitud:

    • limit - define el número de vídeos que se van a devolver desde la API de reproducción
    • offset - establece el número de vídeos que se van a omitir en una lista de reproducción desde la API de reproducción

    Los parámetros de consulta se pasarán al método Catalog como un Map objeto, como pares clave-valor. Este ejemplo devuelve 6 vídeos comenzando por el décimo vídeo de la lista de reproducción:

    Map<String, String> queryParameters = new HashMap<>();
queryParameters.put("limit", "6");
queryParameters.put("offset", "9");

Catalog catalog = new Catalog(eventEmitter, "myAccount", "myPolicy");
catalog.findPlaylistByID("myPlaylistId", null, queryParameters, myListener);

    Reproducción de vídeos locales

    Si desea reproducir vídeos MP4 almacenados localmente, debe guardarlos en la carpeta de activos de la aplicación.

    Uri video = Uri.parse("file:///android_asset/path/to/video.mp4");
brightcoveVideoView.add(Video.createVideo(video.toString()));
brightcoveVideoView.start();
    Puede obtener un error de excepción si intenta reproducir vídeos MP4 almacenados en la carpeta de recursos de la aplicación (/res/raw).

    Colocar los controles del jugador en la parte inferior

    Por defecto, la barra de control del reproductor se coloca directamente debajo del vídeo, como se muestra a continuación:

    También puede colocar la barra de control del reproductor de manera que se encuentre en la parte inferior de la vista de vídeo. Dado que los controles forman parte de BrightcoveVideoView, al establecer la altura en match_parent, la vista cubrirá toda la pantalla. Cuando los controles aparezcan en la parte inferior de la vista, el vídeo permanecerá en el medio de la vista.

    Hay dos maneras de hacer esto. El primer enfoque sigue:

    1. Abra el archivo main/res/values/style.xml para su proyecto.

    2. Agregue el siguiente código XML:

      <style name="BrightcoveControlBar" parent="BrightcoveControlBarDefault">
  <item name="brightcove_align">false</item>
</style>

    O bien, puede usar el código dentro del archivo MainActivity.java de la siguiente manera:

    1. Abra el archivo MainActivity.java del proyecto.

    2. Después de obtener el emisor de eventos, como se muestra aquí:

      EventEmitter eventEmitter = brightcoveVideoView.getEventEmitter();

      Agregue el siguiente detector de eventos y código. Estamos esperando hasta después del DID_SET_VIDEO evento para asegurarnos de que la barra de control ha sido creada.

      eventEmitter.on(EventType.DID_SET_VIDEO, new EventListener() {
  @Override
  public void processEvent(Event event) {
    BrightcoveControlBar brightcoveControlBar = brightcoveVideoView.getBrightcoveMediaController().getBrightcoveControlBar();
    brightcoveControlBar.setAlign(false);
    }
});

    Precar anuncios de IMA

    Para precargar anuncios mediante el SDK nativo para Android y el complemento IMA, utilice la setEnablePreloading(true) propiedad.

    Tenga en cuenta que esto solo funciona con preolladas individuales de VAST.

    Para utilizar esta propiedad, siga estos pasos:

    1. El punto de partida es la aplicación de ejemplo de IMA.

    2. Localice la private void setupGoogleIMA() función.

    3. Agregue el bloque de código de fragmento antes de la última línea de código de la función.

      ImaSdkSettings imaSdkSettings = sdkFactory.createImaSdkSettings();
AdsRenderingSettings adsRenderingSettings =
ImaSdkFactory.getInstance().createAdsRenderingSettings();
adsRenderingSettings.setEnablePreloading(true);

    4. Añada el adsRenderingSettings parámetro a la última línea de código.

      googleIMAComponent = new GoogleIMAComponent(brightcoveVideoView, eventEmitter, true, adsRenderingSettings);

    Buscar sin anuncios

    Puede utilizar la adsDisabled propiedad VideoPlaybackController para deshabilitar la reproducción de anuncios mientras busca a través de un vídeo.

    Esta función funciona todos los plugins de anuncios de Brightcove, incluyendo SSAI y los para CSAI. Para obtener detalles del complemento, consulte la sección Native SDK para Integraciones de Android .

    Para utilizar esta función, siga estos pasos:

    1. Obtener la VideoPlaybackController instancia de BrightcoveVideoView.

      VideoPlaybackController playbackController = brightcoveVideoView.getPlaybackController();

    2. Desactivar la reproducción de anuncios.

      playbackController.setAdsDisabled(true);
    3. Busque la posición de tiempo deseada en el vídeo actual.

    4. Reanude el comportamiento normal del anuncio.

      playbackController.setAdsDisabled(false);

    Su código debería verse algo como esto:

    final VideoPlaybackController playbackController = brightcoveVideoView.getPlaybackController();
eventEmitter.on(EventType.VIDEO_DURATION_CHANGED, new EventListener() {
  @Override
  public void processEvent(final Event event) {
    playbackController.setAdsDisabled(true);
    brightcoveVideoView.seekTo(10000);
  }
});

eventEmitter.on(EventType.DID_SEEK_TO, new EventListener() {
  @Override
  public void processEvent(final Event event) {
    playbackController.setAdsDisabled(false);
  }
});

    Establecer el tamaño del búfer

    Puede considerar aumentar la longitud del búfer para eliminar el almacenamiento en búfer en el reproductor si la entrega del siguiente segmento se retrasa desde la CDN. Sin embargo, es posible que no pueda hacer nada mejor manualmente que HLS ya hace.

    HLS está diseñado para jugar de inmediato y caer calidad si no puede mantenerse al día. De esta manera, no necesita precargar un búfer. Si no puede mantenerse al día, cargará tanto del vídeo como sea posible con la mejor calidad para evitar interrupciones.

    A partir del SDK nativo para Android v6.3.1, se agregaron las siguientes clases al SDK para la implementación de ExoPlayer 2:

    • LoadControlConfig
    • AllocatorConfig

    Los valores establecidos en estas clases se utilizan para crear el DefaultLoadControl objeto ExoPlayer, que se utiliza para crear la instancia de ExoPlayer. Los desarrolladores pueden controlar el tamaño del búfer de la siguiente manera:

    1. los LoadControlConfig y AllocatorConfig las clases siguen los patrones del constructor. Los valores que se pueden establecer en ambas clases reflejan los valores que puede establecer en sus constructores.

      AllocatorConfig allocatorConfig = new AllocatorConfig.Builder().build();
LoadControlConfig loadControlConfig = new LoadControlConfig.Builder()
.setAllocatorConfig(allocatorConfig)
.setMinBufferMs(X)
.setMaxBufferMs(Y)
.build();

    2. Los valores que no se establezcan en el generador utilizarán valores predeterminados. Utilice el ExoPlayerVideoDisplayerComponent.setLoadControlConfig() método para establecer el LoadControlConfig.

      brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);
...
ExoPlayerVideoDisplayComponent dc = (ExoPlayerVideoDisplayComponent) brightcoveVideoView.getVideoDisplay();
dc.setLoadControlConfig(loadControlConfig);

    Configuración de subtítulos y temas

    Actualmente, el BrightcoveCaptionPropertiesActivity está configurado en el archivo de manifiesto del SDK de Brightcove Player, por lo que los desarrolladores no necesitan especificarlo en sus aplicaciones:

    <?xml version="1.0" encoding="utf-8"?>
<manifest
xmlns:android="http://schemas.android.com/apk/res/android"
package="com.brightcove.player"
android:versionCode="1"
android:versionName="1.0">

<application>
<!-- If we don't register this Activity in the Manifest, apps using the SDK will crash when they try to access it. -->
<!-- During the app's build process, this manifest will be merged with the app-level one. -->
<activity android:name="com.brightcove.player.captioning.BrightcoveCaptionPropertiesActivity"/>
</application>
</manifest>

    El BrightcoveCaptionPropertiesActivity heredará el tema predeterminado de la plataforma, siempre y cuando establezca temas para su aplicación en el <activity> nivel del manifiesto de su aplicación.

    Si establece un solo tema de aplicación en el <application> nivel, BrightcoveCaptionPropertiesActivity heredará las propiedades de este tema de nivel de aplicación. Para algunos temas, esto puede hacer que la actividad se vea extrañamente con un estilo o incluso ilegible.

    En esos casos, debe especificar el BrightcoveCaptionPropertiesActivity en su propio manifiesto y aplicar un tema allí, así:

    <activity
android:name="com.brightcove.player.captioning.BrightcoveCaptionPropertiesActivity"
android:theme="@style/MyCustomCaptionSettingsTheme"/>

    Establecimiento de subtítulos predeterminados

    Si el vídeo utiliza varios subtítulos de idioma, puede establecer un idioma predeterminado mediante programación cuando se inicie la reproducción. Su código debería verse similar a esto:

    brightcoveVideoView.getEventEmitter().once(EventType.CAPTIONS_LANGUAGES, new EventListener() {
  @Override
  public void processEvent(Event event) {
    brightcoveVideoView.setClosedCaptioningEnabled(true);
    brightcoveVideoView.setSubtitleLocale("fr");
  }
});

    Configuración de la velocidad de bits máxima

    Para ayudarle a implementar un selector de velocidad de bits en su reproductor, puede utilizar el siguiente código para establecer la velocidad de bits máxima:

    ((ExoPlayerVideoDisplayComponent) videoView.getVideoDisplay()).setPeakBitrate(bitRate);

    Ajuste de la velocidad de reproducción

    El ExoPlayer admite el ajuste de velocidad de reproducción para vídeos que se reproducen en dispositivos Android. Para obtener más información, consulte Reproducción de velocidad variable de Google con el documento ExoPlayer .

    Puede controlar la velocidad de reproducción con el siguiente código:

    eventEmitter.on(EventType.DID_PLAY, new EventListener() {
  @Override
  public void processEvent(Event event) {

    //Get instance of ExoPlayer2
    ExoPlayerVideoDisplayComponent displayComponent = (ExoPlayerVideoDisplayComponent) brightcoveVideoView.getVideoDisplay();
    ExoPlayer exoPlayer = displayComponent.getExoPlayer();

    //Set playback speed
    PlaybackParameters param = new PlaybackParameters(1.6f, 1.0f);
    exoPlayer.setPlaybackParameters(param);
  }
});
    Para obtener más información sobre cómo crear un botón de control para la velocidad de reproducción, consulte la aplicación de ejemplo de controles personalizados.

    Ajuste del modo de gafas de realidad virtual para vídeos de 360°

    Al reproducir un vídeo de 360°, los usuarios pueden seleccionar el botón Video 360 de la barra de control para cambiar al modo de gafas de realidad virtual. Si está utilizando el BrightcovePlayer o BrightcovePlayerFragment actividad, la orientación de la pantalla cambiará a horizontal cuando el modo VR Goggles esté habilitado.

    Si está utilizando una actividad personalizada, deberá agregar lo siguiente:

    brightcoveVideoView.getEventEmitter().on(EventType.CHANGE_ORIENTATION, new EventListener() {
  @Override
  public void processEvent(Event event) {
    int orientation = event.getIntegerProperty(Event.REQUESTED_ORIENTATION);
    setRequestedOrientation(orientation);
  }
});
    En el método que contiene el código anterior, debe agregar @SuppressWarnings("ResourceType") al método.

    Mostrar/ocultar imágenes fijas de vídeo

    El comportamiento predeterminado al recuperar vídeos de la biblioteca de Video Cloud es mostrar la imagen fija de vídeo hasta que comience la reproducción.

    Mostrar la imagen fija del vídeo

    Puede mostrar el vídeo todavía en cualquier momento emitiendo un SET_VIDEO_STILL evento con una VIDEO_STILL propiedad establecida en el URI del vídeo todavía.

    Ocultar la imagen fija del vídeo

    Si está reproduciendo automáticamente el vídeo, es posible que desee ocultar la imagen fija del vídeo. Para ello, puede escuchar el SET_VIDEO_STILL evento e impedir el comportamiento predeterminado de la siguiente manera:

    brightcoveVideoView.getEventEmitter().on(EventType.SET_VIDEO_STILL, new EventListener() {
  @Override
  public void processEvent(Event event) {
    event.preventDefault();
    event.stopPropagation();
  }
});

    Mostrar controles del jugador

    De forma predeterminada, los controles del jugador desaparecerán después de unos segundos. El usuario puede mostrar los controles pulsando en el vídeo.

    Si desea que los controles del reproductor permanezcan mostrándose hasta que el usuario toque el vídeo, puede desactivar la función de ocultación automática de la siguiente manera: (el código necesario está resaltado)

    public class MainActivity extends BrightcovePlayer {
  private BrightcoveMediaController controller;

  @Override
  protected void onCreate(Bundle savedInstanceState) {
    setContentView(R.layout.activity_main);
    brightcoveVideoView = (BrightcoveExoPlayerVideoView) findViewById(R.id.brightcove_video_view);

    controller = new BrightcoveMediaController(brightcoveVideoView);
    controller.setShowHideTimeout(0);

    super.onCreate(savedInstanceState);

    Iniciar la reproducción en el medio de un vídeo

    A veces, es posible que deba iniciar la reproducción desde algún lugar en el medio del video. Para hacer esto, puede llamar BrightCoveVideoView.seekTo() antes de comenzar la reproducción.

    catalog.findVideoByID(getString(R.string.videoId), new VideoListener() {
  @Override
  public void onVideo(Video video) {
    Log.v(TAG, "onVideo: video = " + video);

    brightcoveVideoView.getEventEmitter().on(EventType.DID_SET_VIDEO, new EventListener() {
    @Override
      public void processEvent(Event event) {
      brightcoveVideoView.seekTo(60000);
      brightcoveVideoView.start();
      }
    });
  brightcoveVideoView.add(video);
  brightcoveVideoView.pause();
  }
});

    Intercambio de vídeos

    Este ejemplo muestra un enfoque para intercambiar vídeos en el reproductor.

    Uso de vídeos individuales

    Dado que el clear() método es asincrónico al descargar el video actual, debe esperar antes de agregar un nuevo video al reproductor. Aquí hay dos opciones.

    • Dado que la hora de buscar un vídeo de Video Cloud debería ser más pequeña para actualizar la lista de reproducción, puede probar algo como esto:

      brightcoveVideoView.clear();

catalog.findVideoByID("123456789", new VideoListener() {
  @Override
  public void onVideo(final Video video) {
    brightcoveVideoView.add(video);
    brightcoveVideoView.start();

  }
});

    • Otro enfoque es esperar un evento:

      catalog.findVideoByID("", new VideoListener() {
@Override
  public void onVideo(final Video video) {
    eventEmitter.once(EventType.DID_CHANGE_LIST, new EventListener() {
    @Override
    public void processEvent(Event event)
    	{ brightcoveVideoView.add(video); brightcoveVideoView.start(); }
  });
  brightcoveVideoView.clear();
  }
});

    Uso de una lista de reproducción

    Si está trabajando con una lista de reproducción, tenga en cuenta que al llamar al clear() método se eliminan todos los elementos de la lista de reproducción. Por lo tanto, puede obtener una IndexOutOfBounds excepción al intentar saltar al primer video de la lista de reproducción.

    En su lugar, puedes probar algo como esto:

    brightcoveVideoView.pause();
brightcoveVideoView.stopPlayback();
brightcoveVideoView.setCurrentIndex(0);

    Cambiar entre vídeos de una lista de reproducción

    Si está utilizando una lista de reproducción, puede utilizar el siguiente código para alternar entre vídeos de la lista de reproducción:

    private void setupControls(List<Video> videos) {
  previousVideoButton = (Button) findViewById(R.id.previous_video_button);
  nextVideoButton = (Button) findViewById(R.id.next_video_button);

  if (videos != null) {
    previousVideoButton.setEnabled(false);
    previousVideoButton.setOnClickListener(new View.OnClickListener() {
    public void onClick(View view) {
    int index = brightcoveVideoView.getCurrentIndex();
    int size = brightcoveVideoView.getList().size();
    previousVideoButton.setEnabled(index > 1);
    nextVideoButton.setEnabled((index + 1) < size);

    if (index > 0) {
    brightcoveVideoView.setCurrentIndex(index - 1);
    }
  }
});

nextVideoButton.setEnabled(videos.size() > 1);
nextVideoButton.setOnClickListener(new View.OnClickListener() {
  public void onClick(View view) {
    int index = brightcoveVideoView.getCurrentIndex();
    int size = brightcoveVideoView.getList().size();
    previousVideoButton.setEnabled(index >= 0);
    nextVideoButton.setEnabled((index + 2) < size);

    if ((index + 1) < size) {
    	brightcoveVideoView.setCurrentIndex(index + 1);
    }
  }
});
}
