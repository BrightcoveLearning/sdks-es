Lanzamiento actual

Versión 6.16.3

1 de julio de 2021

Cambios en el SDK:

  • Se agregó soporte para establecer una propiedad de etiqueta en el BrightcoveCaptionFormat clase para admitir el envío de información de pista de audio y subtítulos seleccionados a un receptor de transmisión
  • Actualizado el VideoUtils class para agregar un método de utilidad para eliminar las fuentes de subtítulos del sidecar del objeto Video
  • Se solucionó un problema por el cual el video_duration El parámetro de URL no se incluyó en las solicitudes al back-end de Brightcove Analytics para la generación de informes.

Cambios en el complemento de transmisión:

  • Se agregó soporte para configurar una URL a un reproductor personalizado para la aplicación del receptor Cast. Consulte la documentación de soporte del SDK de Android para obtener más información.

  • Se agregó soporte para enviar subtítulos seleccionados y la información de la pista de audio a un receptor Cast

    Un problema conocido que no es de regresión se abordará en una próxima versión, donde las selecciones de subtítulos en el manifiesto no se muestran automáticamente en el remitente si la sesión de transmisión se cierra durante la reproducción.

    Una solución para este problema es que la selección realizada antes de abrir la sesión de transmisión se puede volver a seleccionar y mostrar cuando la sesión se vuelve a abrir en el dispositivo remitente.

Cambios en la reproducción sin conexión:

  • Se agregó soporte para configurar propiedades en el ResumenFuera de líneaCatálogo clase. Las propiedades del catálogo sin conexión se pueden establecer de dos formas:

    • Como un Mapa <Cadena, Cadena> de pares clave / valor utilizando el setProperties (Map <String, String>) método
    • A través de getProperties (). put (clave, valor) método
  • Soporte agregado para la creación personalizada NotificationChannels en el Gestor de descargas clase. Consulte la documentación de soporte del SDK de Android para obtener más información.

Cambios en el complemento SSAI:

  • Se agregó compatibilidad de análisis para elementos de extensión VMAP y VAST anidados

    Los elementos de extensión, independientemente de la profundidad de anidamiento, no son ejecutados por el complemento SSAI, por lo que esta compatibilidad solo garantiza que el análisis de VAST y VMAP con elementos de extensión anidados de cualquier profundidad no falle.

  • Soporte de análisis agregado para VAST Verificación de anuncios elementos

    Verificación de anuncios Los elementos no son ejecutados por el complemento SSAI, por lo que este soporte solo garantiza que el análisis de VAST con Verificación de anuncios elementos no falla.

    Además, Verificación de anuncios Los elementos son una función de VAST 4.0 y el SDK de Brightcove para Android aún no es totalmente compatible con VAST 4.x. Este soporte de análisis no debe considerarse como soporte para VAST 4.x en general en Brightcove Native SDK para Android.

Versión 6.11.0

7 febrero 2020

Cambios en el SDK:

  • Se agregó soporte para el servicio de administración de derechos de reproducción de Brightcove.

    Tenga en cuenta que esta nueva función tiene disponibilidad limitada.

  • Rompiendo el cambio: El objeto Lista de reproducción ahora es inmutable.

  • El objeto Lista de reproducción ahora puede contener una combinación de videos con estado Status.OK y Status.ERROR

  • Playlist.getErrorVideos(): Este nuevo método devolverá solo videos con el estado de Status.ERROR

  • Playlist.getAllVideosAndErrors(): Este nuevo método devolverá todos los videos con ambos estados, Status.OK y Status.ERROR , siguiendo el mismo orden que se devolvió en la respuesta del Servicio de gestión de derechos de reproducción

  • Nota: Playlist.getVideos() y Playlist.getCount() se comportan de la misma manera que las versiones anteriores, es decir, solo consideran los videos con estado Status.OK

  • Adicional ErrorListener.onError(List) llamar de vuelta.

  • Obsoleto ErrorListener.onError(String) llamar de vuelta.

  • El objeto Video ahora tiene un código de estado, que puede ser Status.OK o Status.ERROR. Puede recuperar el estado a través de Video.getStatus():Status.

  • Status.OK: El video tiene todas las propiedades esperadas y no tiene Video.Fields.ERRORS en eso.

  • Status.ERROR: El video tiene Video.Fields.ERRORS en sus propiedades en forma de List<CatalogError>.

Complemento VMAP:

  • El complemento VMAP ha quedado obsoleto. Para publicidad del lado del cliente con documentos VMAP y VAST, utilice el complemento IMA.
