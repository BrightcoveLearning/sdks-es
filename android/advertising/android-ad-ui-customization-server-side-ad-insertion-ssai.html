    Personalización de la interfaz de usuario de anuncios de Android con inserción de anuncios en el lado del servidor (SSAI)

    En este tema, aprenderá a administrar y personalizar los elementos de la interfaz de usuario mostrados durante la reproducción de anuncios en el lado del servidor, utilizando el complemento SSAI para Brightcove Native SDK para Android.

    Resumen

    Cuando se trabaja con inserción de anuncios en el lado del servidor (SSAI), Brightcove Native SDK para Android le permite personalizar los elementos de la interfaz de usuario del anuncio. Para obtener más detalles acerca de SSAI, consulte lo siguiente:

    • Implementación de anuncios del lado del servidor con los SDK del reproductor nativo
    • El complemento SSAI para el SDK nativo para Android

    La interfaz de usuario del anuncio se compone de dos componentes principales:

    • Superposición de anuncios (A)
    • Controlador de medios publicitarios (B)

    Superposición de anuncio

    La superposición de anuncios se compone de los siguientes componentes:

    • Cuenta atrás de duración total del pod de anuncios (A)
    • Ver más información (B)
    • Omitir vista de anuncio (C)
    El botón Omitir anuncio solo se admite con la inserción de anuncios en el lado del servidor (SSAI) VOD. Actualmente no es compatible con Live SSAI.

    Controlador de medios publicitarios

    El controlador multimedia de anuncios se compone de estos componentes, de izquierda a derecha:

    • Botón Reproducir (A)
    • Cuenta atrás de número de anuncio (B)
    • Cuenta atrás de duración de anuncio único (C)
    • Botón de pantalla completa (D)

    Administración de componentes de la interfaz de usuario

    Para habilitar o deshabilitar los componentes de la interfaz de usuario de anuncios desde la vista, consulte lo siguiente:

    • Modificación del documento VAST
    • Especificación de la configuración del anuncio

    Modificación del documento VAST

    Las vistas Más información y Omitir anuncio se pueden mostrar u ocultar en función de la definición de anuncio del documento VAST.

    La vista Más información sólo aparecerá cuando la creatividad lineal contenga una ClickThrough URL, como se muestra a continuación:

     <VAST xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Ad id="blue-01-5s">
    <InLine>
      ...
      <Creatives>
        <Creative id="creative-01-5s">
          <Linear>
            ...
            <VideoClicks>
              <ClickThrough id="clickthrough">https://www.brightcove.com/en/</ClickThrough>
              <ClickTracking id="...”>...</ClickTracking>
            </VideoClicks>
          </Linear>
        </Creative>
      </Creatives>
    </InLine>
  </Ad>
</VAST>

    La vista Omitir anuncio solo aparecerá cuando haya un skipoffset valor válido en la creatividad lineal:

     <VAST xmlns:xsd="http://www.w3.org/2001/XMLSchema" xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance">
  <Ad id="blue-01-5s">
    <InLine>
      ...
      <Creatives>
        <Creative id="creative-01-5s">
          <Linear skipoffset="00:00:03">
            ...
          </Linear>
        </Creative>
      </Creatives>
    </InLine>
  </Ad>
</VAST>

    Especificación de la configuración del anuncio

    Al utilizar Mejoras del reproductor para SSAI en vivo, puede habilitar o deshabilitar componentes dentro application_ad_configuration de la configuración de su anuncio. El client_options objeto permite activar o desactivar los componentes de la interfaz de usuario de anuncios.

    La modificación de la interfaz de usuario del anuncio mediante la configuración del anuncio solo se aplica a SSAI en vivo.
    {
  "application_ad_configuration": {
	"ad_configuration_description": "$YOUR_DESCRIPTION",
	"ad_configuration_expected_response_type": "Vast",
	"ad_configuration_headers_for_impressions": false,
	"ad_configuration_strategy": "SingleAdResponse",
	"ad_configuration_transforms": [],
	"ad_configuration_url_format": "$YOUR_AD_SERVER",
     "ad_configuration_client_sdk_enabled": true,
     "client_options": {
        "show_ad_break_remaining_time": true / false,
        "show_ad_remaining_time": true / false,
        "show_number_of_remaining_ads": true / false,
        "client_only_tracking": true / false
     }
  },
  "application_description": "$YOUR_DESCRIPTION"
}
    Si está utilizando algún client_options componente, debe haber ad_configuration_client_sdk_enabled establecido en true.

    Utilice el objeto client_options para activar/deshabilitar los siguientes componentes:

    • Show_ad_break_remaining_time: Cuenta regresiva de duración total de Ad Pod
      • Ejemplo: Tu vídeo se reanudará en 15 segundos
    • Show_ad_remaining_time: La cuenta regresiva de duración del anuncio
      • Ejemplo: ( 0:15)
    • Show_number_of_remaining_ads: La cuenta regresiva del número de anuncio
      • Ejemplo: Anuncio 1 de 3, anuncio 2 de 3, etc.

    Personalización de la superposición de anuncios

    Puede personalizar las vistas y el texto predeterminado asociado a la superposición de anuncios.

    Modificación de la vista

    Para personalizar la superposición de anuncios, puede reemplazar la versión del archivo de diseño ssai_ad_overlay.xml en la carpeta res de la aplicación. Este archivo contiene varios diseños independientes diferentes, como se muestra aquí:

    Por ejemplo, puede anular la vista de duración del Pod de anuncios actualizando el archivo ssai_ad_pod_duration_countdown_view.xml . También puede anular toda la vista de superposición de anuncios reemplazando el archivo ssai_ad_overlay.xml .

    Esta estructura de diseño le permite anular componentes particulares mientras mantiene la compatibilidad con versiones anteriores del complemento SSAI.

    Explora los archivos de superposición de anuncios con Android Studio. A partir de la versión 6.9.0 del SDK nativo para Android, ssai_ad_overlay.xml del complemento SSAI se ve así:

    <?xml version="1.0" encoding="utf-8"?>
<FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:background="@color/white">

   <include layout="@layout/ssai_ad_countdown_and_learn_more_overlay"/>
   <include layout="@layout/ssai_ad_skip_overlay"/>
</FrameLayout>

    Al reemplazar un archivo xml de componente, debe utilizar un TextView con el ID de vista original especificado para evitar resultados no deseados. Para obtener más información, consulte la siguiente tabla

    Nombre del componente Id. de componente Vista de componente
    Vista de duración del pod de anuncios text_ad_countdown TextView
    Ver más información text_ad_learn_more TextView
    Omitir vista de anuncio text_ad_skip TextView

    Modificación del texto predeterminado

    Todos los componentes de superposición de anuncios se TextViewsdefinen en el archivo string.xml de la aplicación. Para cambiar el texto de visualización, invalidarlo creando un elemento o utilizando el mismo ID de cadena utilizado por el SDK nativo para Android.stringplurals

    La vista Duración del Pod de anuncios utiliza las siguientes cadenas:

    <plurals name="ssai_message_ad_break_duration_countdown">
   <item quantity="one">Your video will resume in\n%1$d second</item>
   <item quantity="other">Your video will resume in\n%1$d seconds</item>
</plurals>
<string name="ad_info_now_text">Your video will resume now</string>
<string name="ad_buffering_text">Your ad is buffering …</string>

    La vista Más información utiliza la siguiente cadena:

    <string name="ssai_message_learn_more">Learn More >> </string>

    La vista Omitir anuncio utiliza las siguientes cadenas:

    <plurals name="you_can_skip_text">
   <item quantity="one">You can skip this ad in\n%d second</item>
   <item quantity="other">You can skip this ad in\n%d seconds</item>
</plurals>
<string name="skip_text">Skip Ad</string>

    Personalización del controlador de medios publicitarios

    Puede personalizar las vistas y el texto predeterminado asociado al controlador de medios publicitarios.

    Modificación de la vista

    Para personalizar el controlador de medios publicitarios, puede reemplazar la versión del archivo de diseño default_ssai_ad_media_controller.xml en la carpeta res de la aplicación.

    Esto anula el diseño predeterminado utilizado en el plugin SSAI. Cada vez que se detecta un anuncio, el complemento SSAI cambiará a este diseño y, a continuación, volverá al controlador predeterminado cuando finalice el anuncio.

    Cuando trabaje con su propio diseño, utilice el mismo tipo de vista e identificadores de vista que se especificaron originalmente para evitar resultados no deseados. Explore el diseño de Ad Media Controller con Android Studio. Para la versión 6.9.0 del complemento SSAI, el default_ssai_ad_media_controller.xml se ve así:

    <com.brightcove.player.mediacontroller.BrightcoveControlBar
   xmlns:android="http://schemas.android.com/apk/res/android"
   xmlns:tools="http://schemas.android.com/tools"
   android:id="@+id/brightcove_control_bar"
   style="@style/SSAIAdControlBar"
   android:layout_width="match_parent"
   android:layout_height="wrap_content"
   android:layout_gravity="bottom"
   android:background="@color/bmc_background"
   android:orientation="vertical"
   android:padding="8dp">

   <LinearLayout
       android:layout_width="match_parent"
       android:layout_height="wrap_content"
       android:gravity="bottom"
       android:orientation="horizontal">

       <Button style="@style/BorderlessButton"
           android:id="@id/play"
           android:layout_width="wrap_content"
           android:layout_height="wrap_content"
           android:text="@string/brightcove_controls_play"
           android:visibility="gone"
           tools:visibility="visible"/>

       <include layout="@layout/ssai_ad_number_countdown_view"/>

       <include layout="@layout/ssai_single_ad_duration_countdown_view"/>

       <View
           android:id="@id/two_line_spacer"
           android:layout_width="0dp"
           android:layout_height="0dp"
           android:layout_weight="1"
           android:visibility="gone"
           tools:visibility="visible"/>

       <Button style="@style/BorderlessButton"
           android:id="@id/full_screen"
           android:layout_width="wrap_content"
           android:layout_height="wrap_content"
           android:padding="4dp"
           android:text="@string/brightcove_controls_enter_full_screen"
           android:visibility="gone"
           tools:visibility="visible"/>
   </LinearLayout>

</com.brightcove.player.mediacontroller.BrightcoveControlBar>

    Los tipos de vista y los identificadores de vista de componentes SSAI Ad Media Controller se describen aquí:

    Nombre del componente Id. de componente Vista de componente
    Botón Reproducir play Button
    Botón Pantalla completa full_screen Button
    Cuenta atrás de número de anuncio text_ad_number_countdown TextView
    Cuenta regresiva de duración de anuncio único text_single_ad_duration_countdown TextView

    Echa un vistazo a la aplicación de ejemplo de controles personalizados, donde el controlador de medios predeterminado está personalizado para agregar nuevos botones. Puede aplicar el mismo enfoque para personalizar su controlador de medios publicitarios SSAI.

    Modificación del texto predeterminado

    Todos los componentes de Ad Media Controller TextViewsse definen en el archivo string.xml de su aplicación. Para cambiar el texto de visualización, invalidarlo creando un elemento o utilizando el mismo ID de cadena utilizado por el SDK nativo para Android.stringplurals

    La vista de cuenta regresiva del número de anuncio utiliza la siguiente cadena:

    <string name="ssai_message_ad_number_countdown">Ad %1$d of %2$d</string>

    La vista de cuenta atrás de duración de anuncio único utiliza la siguiente cadena:

    <string name="ssai_message_ad_duration_countdown">(%1$s)</string>

    Resumen

    Tiene algunas opciones para personalizar los elementos de la interfaz de usuario de SSAI. El nivel de complejidad dependerá de lo que desee lograr. Desde bajo hasta alto nivel de complejidad, puede:

    • Configure u oculte ciertos componentes a través de la configuración de VAST
    • Mostrar u ocultar componentes utilizando el client_options objeto en la configuración del anuncio (actualmente sólo SSAI en vivo)
    • Cambiar el valor de cadena mostrado
    • Anular el diseño de la interfaz de usuario

    Comparta cualquier idea o comentario que tenga en el foro de SDK de Brightcove Native Player.
