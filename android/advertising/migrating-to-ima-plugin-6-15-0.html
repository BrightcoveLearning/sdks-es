    Migración al complemento IMA de Brightcove 6.15.0

    En este tema, aprenderá cómo migrar su aplicación para usar el complemento IMA de Brightcove para el SDK nativo para Android 6.15.0.

    Resumen

    El SDK nativo de Brightcove para Android 6.15.0 o posterior es compatible con la versión 3.19.4 del SDK de IMA de Google. Si está utilizando la publicidad de IMA, hay algunas cosas que debe tener en cuenta para utilizar esta versión del complemento IMA. Entre ellos se incluyen los siguientes:

    • Cambios en el SDK de Google IMA
    • Cambios en el complemento IMA de Brightcove
    • Migración al complemento IMA 6.15.0 < /a

    Cambios en el SDK de Google IMA

    El SDK nativo de Brightcove actualizó el SDK de Google IMA de la versión 3.11.2 a la versión 3.19.4. Los cambios más relevantes incluyen:

    • SdkFactory.createAdDisplayContainer() está obsoleto
    • AdDisplayContainer.setPlayer(VideoAdPlayer) está obsoleto
    • AdDisplayContainer.setAdContainer(ViewGroup) está obsoleto
    • AdsRequest.setAdDisplayContainer(AdDisplayContainer) se ha eliminado

    Para ver la versión completa, consulte el documento del historial de versiones de Google IMA Android SDK .

    Cambios en el complemento IMA de Brightcove

    El complemento IMA de Brightcove realizó varios cambios para ser totalmente compatible con la nueva versión del SDK de Google IMA. A saber:

    • Uso del Generador de GoogleIMAComponent
    • Creación de AdDisplayContainer
    • Métodos obsoleto en GoogleImavideoAdPlayer

    Uso del Generador de GoogleIMAComponent

    Se recomienda utilizar el Builder para crear la GoogleIMAComponent instancia. Esto facilita la configuración de GoogleIMAComponent.

    googleIMAComponent = new GoogleIMAComponent.Builder(brightcoveVideoView, eventEmitter)
  .setUseAdRules(true)
  .setImaSdkSettings(customIMASDKSettings)
  .setAdsRenderingSettings(customAdsRenderingSettings)
  .setAdDisplayContainerFactory(customAdDisplayContainerFactory)
  .build();
}

    Creación de AdDisplayContainer

    Debido a que se eliminó el SDK de IMA de Google AdsRequest.setAdDisplayContainer(...), el complemento IMA de Brightcove ahora crea esto automáticamente, utilizando el GoogleIMAVideoAdPlayer y BaseVideoView.

    El complemento requiere un AdDisplayContainerFactory para crear el archivo AdDisplayContainer.

    public interface AdDisplayContainerFactory {
  /**
  * Creates the AdDisplayContainer with the provided GoogleIMAVideoAdPlayer
  * and the ViewGroup retrieved with getViewContainer()
  *
  * @param googleIMAVideoAdPlayer the Brightcove Ad Player for Google IMA
  */
  AdDisplayContainer createAdDisplayContainer(GoogleIMAVideoAdPlayer googleIMAVideoAdPlayer);

  /**
  * Returns the ViewGroup container used for both,
  * the creation of the GoogleIMAVideoAdPlayer and the AdDisplayContainer
  */
  ViewGroup getViewContainer();
}

    Bajo el capó, el plugin utiliza el DefaultAdDisplayContainerFactory, an AdDisplayContainerFactory, para crear el AdDisplayContainer. El getViewContainer() devuelve el BaseVideoView pasado en su constructor, y el createAdDisplayContainer método devuelve:

    ImaSdkFactory.createAdDisplayContainer(getViewContainer(), googleIMAVideoAdPlayer);

    Añadir su AdDisplayContainerFactory

    Si necesita una implementación diferente, debe pasar la suya propia AdDisplayContainerFactory al GoogleIMAComponent Builder:

    googleIMAComponent = new GoogleIMAComponent.Builder(brightcoveVideoView, eventEmitter)
  .setAdDisplayContainerFactory(customAdDisplayContainerFactory)
  .build();

    Si desea reproducir los Anuncios en una vista diferente a la BaseVideoView, puede proporcionar el DefaultAdDisplayContainerFactory con su ViewGroup.

    googleIMAComponent = new GoogleIMAComponent.Builder(brightcoveVideoView, eventEmitter)
  .setAdDisplayContainerFactory(new DefaultAdDisplayContainerFactory(myViewGroup))
  .build();

    Métodos obsoleto en GoogleImavideoAdPlayer

    Los siguientes métodos están en desuso en la GoogleIMAVideoAdPlayer clase:

    Método obsoleto Use en su lugar
    playAd() playAd(AdMediaInfo)
    loadAd(String) loadAd(AdMediaInfo, AdPodInfo)
    stopAd() stopAd(AdMediaInfo)
    resumeAd() playAd(AdMediaInfo)
    pauseAd() pauseAd(AdMediaInfo)

    Observe que los métodos de reemplazo ahora requieren AdMediaInfo. Puede recuperar el AdMediaInfo objeto actual llamando a GoogleIMAVideoAdPlayer.getCurrentAdMediaInfo(). Esto devolverá el cargado AdMediaInfo, o Null.

    Migración al complemento IMA 6.15.0

    Para migrar al complemento IMA de Brightcove 6.15.0, siga los pasos siguientes:

    • Actualizar versiones de dependencia
    • Eliminar métodos obsoleto

    Actualizar versiones de dependencia

    Actualice su aplicación para usar las siguientes versiones de dependencia:

    • Todas las dependencias del SDK de Brightcove establecidas en la versión 6.15.0
    • Usar Google IMA SDK versión 3.19.4
    • Usar los Servicios de Google Play versión 19.2.0

    Aquí hay un ejemplo del archivo build.gradle :

    //build.gradle
dependencies {
 //Brightcove SDK dependencies
 implementation "com.brightcove.player:android-sdk6.15.0:"
 implementation "com.brightcove.player:exoplayer2:6.15.0"
 implementation "com.brightcove.player:android-ima-plugin:6.15.0"

 //Google IMA SDK
 implementation "com.google.ads.interactivemedia.v3:interactivemedia:3.19.4"

 //Google Play Services
 implementation "com.google.android.gms:play-services-ads:19.2.0"
 ...
}

    Eliminar métodos obsoleto

    La configuración actual del IMA de Google podría tener un aspecto similar a este:

    AdDisplayContainer container = sdkFactory.createAdDisplayContainer();
container.setPlayer(googleIMAComponent.getVideoAdPlayer());
container.setAdContainer(brightcoveVideoView);

    Estos tres métodos están obsoleto y desaparecen.

    Ahora debe llamar googleIMAComponent.getAdDisplayContainer() y devolverá el AdDisplayContainer asociado con el GoogleIMAVideoAdPlayer , o null si no está disponible.

    Para obtener más información sobre cómo crear el AdDisplayContainer, consulte la sección Creación de AdDisplayContainer.

    El siguiente método también se ha eliminado del SDK de IMA de Google:

    adsRequest.setAdDisplayContainer(container);

    En su lugar, utilice el AdDisplayContainer para crear una instancia del IMA de Google AdsLoader.

    Para obtener más información sobre los métodos obsoleto y sus reemplazos, consulte la sección Métodos obsoleto en GoogleImavideoAdPlayer .

    Más información

    Para obtener más información sobre los cambios de IMA, consulte la documentación del historial de versiones de Google IMA Android SDK .

    Para cualquier pregunta relacionada, no dude en ponerse en contacto con el equipo de Brightcove SDK a través del foro de Brightcove SDK.
